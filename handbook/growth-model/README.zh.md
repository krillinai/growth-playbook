# 增长模型

> 增长模型明确解释一个已定义客户如何获得价值、留存、产生收入或其他持续结果，并创造下一轮增长所需的输入。

**[English](README.md)｜[简体中文](README.zh.md)**

## 定义与边界

| 模型 | 核心问题 | 单独无法解释的内容 |
| --- | --- | --- |
| 用户生命周期 | 客户会经历哪些阶段？ | 业务为什么增长，以及哪个机制具有因果作用 |
| 漏斗 | 用户在哪里转化或流失？ | 输出如何创造未来输入，以及经济性如何复利 |
| 循环 | 一轮行为如何创造下一项输入？ | 完整业务、价值、留存与商业化系统 |
| 增长模型 | 价值、行为、分发、留存、商业化、扩张与再投资如何相互作用？ | 没有证据时，它不能证明这些假设成立 |

生命周期是一张地图，漏斗用于诊断转化，循环是一种重复机制。增长模型把相关部分连接成适合具体业务的因果系统，并识别当前主要约束。

## 核心结构

```text
已定义客户与问题
-> 获客输入与承诺
-> 激活与首次价值
-> 重复价值与留存
-> 商业化或持续结果
-> 推荐、扩张、内容、数据、销售能力或再投资
-> 下一项增长输入
```

并非所有业务都使用所有机制。Sales-led 企业产品、消费订阅、Marketplace、广告产品、按量 API 和社区产品需要不同模型。没有匹配客户行为、产品价值、分发与经济性就复制其他公司的图，只会制造虚假精确。

## 模型组件

| 组件 | 诊断问题 | 证据 |
| --- | --- | --- |
| 客户 | 哪个分群存在问题，并具备获得价值的条件？ | ICP 证据、紧迫性、替代方案、可触达性 |
| 价值 | 什么结果推动采用与重复使用？ | 定性价值、激活、留存、切换或支付意愿 |
| 获客 | 哪些输入以正确承诺触达客户？ | 增量合格需求、留存 CAC、渠道匹配 |
| 激活 | 客户获得可信首次价值前必须发生什么？ | 合格激活、Time to Value、下游留存 |
| 留存 | 价值为什么以及多久重复一次？ | Cohort 曲线、价值事件频率、流失原因、召回 |
| 商业化 | 如何可持续获取价值？ | 转化、计价单位、毛利、LTV、回本、续约、扩张 |
| 传播与扩张 | 哪些输出创造未来分发或更深价值？ | 推荐、协作、内容、网络密度、交叉销售、销售信号 |
| 再投资 | 哪些留存资源支持或改善下一轮？ | 毛利、数据、品牌、内容、供给、产品学习、销售能力 |
| 约束 | 哪个因素当前最限制系统？ | 敏感性、瓶颈证据、容量、经济性、控制力 |

## 增长公式

公式用于暴露假设，而不是制造确定性。一个简单客户模型可以从以下公式开始：

```text
留存客户
= 新增激活客户
+ 留存回访客户
+ 召回客户
- 流失客户
```

收入模型可以增加：

```text
留存收入
= 新收入
+ 留存重复收入
+ 扩张收入
- 收缩收入
- 流失收入
```

再连接获客和激活：

```text
新增留存客户
= 合格触达
x 获客转化
x 激活到价值转化
x 留存质量
```

乘法只是一种概念拆解。定义、时间窗口、成熟度、归因、增量与分群组合决定数字是否适合决策。

## Four Fits

使用 Four Fits 检查模型能否作为连接系统运作：

| Fit | 问题 | 失败信号 |
| --- | --- | --- |
| Market-Product Fit | 产品是否为已定义市场解决重要问题？ | 留存价值弱或需要特殊人工救援 |
| Product-Channel Fit | 产品是否匹配渠道触达和转化需求的方式？ | 流量强但激活弱，或高度依赖单一渠道 |
| Channel-Model Fit | 渠道经济性能否支撑收入和毛利模型？ | 边际 CAC、销售投入或回本超过价值 |
| Model-Market Fit | 市场规模、价格、频率与经济性能否支撑目标规模？ | 局部执行良好，但市场不足或经济性不成立 |

改变一种 Fit 可能削弱另一种。进入更高端市场会改变买家、产品要求、销售周期、价格、支持与留存模型；降低价格可能扩大触达，却让 Sales-led 分发失去经济性。

## 约束诊断

1. 使用已观察的客户流与现金或持续价值流画出当前模型。
2. 把每一条连接标记为 `已验证`、`推断` 或 `缺失`。
3. 按分群与成熟 Cohort 衡量转化。
4. 识别最弱因果连接，而不是简单选择数值最低的指标。
5. 按影响、证据、控制力、学习速度、经济性与风险比较潜在约束。
6. 选择一个主要约束，并保留可信备选解释。
7. 测试能够在明确窗口内改变决策的干预。
8. 根据结果更新模型，包括反面证据。

如果改善低转化只会带来低价值客户，它就未必是主要约束。获客量很高，但留存泄漏更快时也无法形成增长。即使留存强，如果市场、渠道、价格或服务能力过于有限，增长仍不能成立。

## 循环与再投资

只有输出能够稳定创造下一项有用输入时，循环才应进入增长模型。

```text
留存价值
-> 输出或资源
-> 合格下一项输入
-> 客户价值
-> 留存价值
```

输出可以是邀请、共享成果、内容、供给、声誉、数据、毛利、品牌、合作伙伴能力或销售证明。衡量资格、转化、接收者或参与者价值、留存质量、周期、增量、成本与护栏。

当留存毛利能够购买增量且盈利的需求时，付费获客也可以形成再投资循环。它并不天然低于病毒或自然增长。模型应该暴露边际经济性与渠道集中度，而不是奖励某一种标签。

## 模型验证

| 判断 | 验证方式 |
| --- | --- |
| 客户获得价值 | 访谈、任务完成、激活深度关系、留存使用 |
| 价值能够重复 | 成熟 Cohort、频率分布、续约、重复购买、召回 |
| 渠道带来正确客户 | 增量、按来源激活与留存、边际 CAC、回本 |
| 商业化与价值一致 | 支付意愿、转化、扩张、毛利、可预测计价单位 |
| 循环能够闭合 | 合格下一项输入、接收者价值、留存调整传播、周期 |
| 模型能够规模化 | 市场容量、运营负担、服务成本、饱和、治理、风险 |

优化组件前先进行敏感性分析：哪个假设对输出影响最大，团队能否控制或学习它。再复杂的 Spreadsheet 也不能补偿未经验证的价值或留存关系。

## 运营节奏

1. 在制定战略和资源计划时审查模型。
2. 监控少量与因果连接对应的结果、输入和护栏指标。
3. 使用漏斗和 Cohort 调查转化，而不是让它们替代模型。
4. 围绕当前主要约束进行实验或证据工作。
5. 记录模型发生了什么变化，以及下一项决策。
6. 分群、产品、渠道、价格、销售模式、技术或市场发生实质变化时重建模型。

## 常见错误

- 画出通用生命周期就称为公司增长模型。
- 罗列指标，却不说明因果关系。
- 把所有增长都理解成获客。
- 复制知名公司的循环、飞轮或组织图。
- 没有留存价值与经济性证据就优化最低漏斗转化。
- 把 Dashboard 预测当成因果证明。
- 在一种价值与留存机制成立前加入大量循环。
- 忽略服务容量、信任、监管、欺诈、平台依赖或渠道饱和。
- 改变客户分群、价格、产品或进入市场方式后仍使用旧模型。

## 证据与局限

- 增长模型是可证伪的决策工具，不是完整因果识别方法。
- 公司结果不能归因于一个框架、运营者或可见机制。
- 客户组合、竞争、渠道、技术和行为变化时，历史系数会衰减。
- 持续保留不确定性、备选解释、运营约束与外部性。

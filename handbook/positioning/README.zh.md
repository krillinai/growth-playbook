# 定位

> 定位定义了一个竞争语境，说明特定产品为什么能为特定最佳客户提供差异化价值。

**[English](README.md)｜[简体中文](README.zh.md)**

## 定义与边界

| 概念 | 核心问题 | 边界 |
| --- | --- | --- |
| 定位 | 在当前情境中，为什么这个产品是这类客户的最佳选择？ | 定位是市场假设，不是口号。 |
| Messaging | 应该如何面向特定受众和时刻表达定位？ | 渠道信息可以变化，但底层定位需要保持一致。 |
| 价值主张 | 为谁承诺什么结果，为什么可信？ | 价值主张是定位的一部分，不能定义完整竞争语境。 |
| 品牌 | 公司或产品积累了什么预期、记忆、身份与信任？ | 品牌可以强化定位，不能替代客户与竞争证据。 |
| 战略 | 公司在哪里竞争、如何分配资源并建立优势？ | 战略选择行动，定位解释当前选择依据。 |
| 愿景 | 公司希望创造什么未来？ | 愿景可以指导方向，但不一定解释客户现在为什么选择。 |
| 品类 | 哪种市场框架能帮助客户理解替代方案与评估标准？ | 品类是一项定位决策，不是定位的全部。 |
| 客户分群与 ICP（理想客户画像） | 哪些客户差异应该改变产品或增长决策？ | 分群定义匹配条件，定位解释这些条件下的差异化价值。 |

定位位于产品市场匹配证据与增长执行之间：

```text
产品、客户与竞争证据
-> 定位假设
-> 信息、叙事、Offer 与资格判断
-> 获客、销售、激活、定价与留存行为
-> 更新证据与定位
```

定位不能制造需求、产品质量、留存或支付意愿，但能让这些假设清晰到可以验证。

## 定位单位、范围与版本

开始定位工作前先确定单位。同一家公司可以针对不同产品、分群、购买情境、地区或市场路径拥有不同且有效的定位。

| 范围字段 | 决策内容 |
| --- | --- |
| 产品单位 | 产品、版本、模块、平台、服务或组合 |
| 客户单位 | 用户、团队、账户、购买者、行业、使用场景或市场层级 |
| 购买情境 | 触发事件、任务、紧迫性、当前替代方案与决策标准 |
| 地区与语言 | 市场条件、监管、文化、本地替代方案与术语 |
| 市场路径 | Self-serve、Sales-led、Partner-led、Marketplace 或 Hybrid |
| 时间范围 | 下一个运营周期的当前定位，而不是永久身份 |
| 负责人和版本 | 决策负责人、证据日期、生效版本与下次复盘 |

有效定位要足够具体，可以指导决策；也要足够稳定，能覆盖目标客户旅程。如果一句话必须覆盖互不兼容的买家或产品，应先拆分范围，再简化表达。

## 竞争替代方案

先确认如果产品不存在，客户会怎么办。替代方案包括：

- 直接竞争对手；
- 相邻品类或通用产品；
- 表格、文档、邮件或内部工具；
- 人工流程或外包服务；
- 现有流程、行为或预算分配；
- 延迟行动、什么都不做或接受问题。

使用 Win/Loss 访谈、替换行为、搜索与销售语言、采购比较、流失去向和真实工作流。竞争功能表可以作为辅助材料，但不能成为主要证据。

按购买情境梳理替代方案，不要假设所有客户拥有同一组竞争对象：

| 替代方案 | 客户为什么选择 | 优势情境 | 失效位置 | 证据状态 |
| --- | --- | --- | --- | --- |
| 现状 | 没有切换成本和新增风险 | 流程与责任已经熟悉 | 原有成本、延迟、错误或约束继续存在 | Verified、inferred 或 missing |
| 直接竞争对手 | 品类与评估标准已经熟悉 | 能力和证据容易理解 | 在目标情境下存在相关缺口 | Verified、inferred 或 missing |
| 通用工具 | 灵活且已经可用 | 新增成本较低 | 设置、治理、专业化或规模受限 | Verified、inferred 或 missing |
| 服务或人工流程 | 具有人类判断和定制能力 | 复杂或低频任务 | 成本、速度、一致性或容量受限 | Verified、inferred 或 missing |

不行动经常是最强替代方案。如果客户感知的改变成本高于问题成本，对竞争产品的差异化并不会自动产生紧迫性。

## 独特能力、差异化价值与证据

定位需要从真实差异连接到客户结果：

```text
独特能力
-> 运营或行为后果
-> 差异化客户价值
-> 证据
-> 成立条件与边界
```

能力可能来自产品设计、专有数据、工作流集成、网络结构、分发、服务、运营、信任、合规、生态、速度、成本结构，或者竞争对手难以在相同条件下复制的组合。

| 层级 | 检验问题 |
| --- | --- |
| 能力 | 产品现在是否具备，替代方案能否在相同条件下提供？ |
| 后果 | 这项能力会改变客户的什么工作或行为？ |
| 价值 | 这种改变是否影响收入、成本、时间、风险、质量、可达性、控制或战略灵活性？ |
| 证据 | 哪种产品演示、客户结果、技术证明、Benchmark 或运营记录能够支持？ |
| 边界 | 对哪些客户、工作流、规模、地区或前提条件不再成立？ |

不要把 Roadmap 当作当前差异，也不要依赖所有竞争对手都能使用的“简单”“现代”“强大”或“AI-powered”等利益。更强的证据链会说明价值在什么条件下可以被观察。

## 最佳客户与客户分群

最佳客户通常：

- 以足够紧迫或高频的方式面对相关问题；
- 使用的替代方案存在能让差异变得重要的限制；
- 具备获得价值所需的工作流、数据、权限、协作者、成熟度或基础设施；
- 能够识别或学习相关评估标准；
- 可以采用、留存，并在经济上支持 Offer；
- 不需要会消除产品优势的特殊例外。

客户分群与定位相互作用：

```text
客户差异
-> 不同价值条件或替代方案
-> 分群假设
-> 面向该分群的定位
-> 获客、采用、留存与经济证据
-> 修正分群与定位
```

不要只用 Firmographic 属性定义最佳客户。应使用问题、替代方案、场景、准备度、行为、购买流程、价值实现、经济性与市场路径证据。分群定义与治理参见[客户分群与 ICP](../customer-segmentation/README.zh.md)。

## 品类选择与重构

品类为客户提供理解捷径：产品是什么、替代什么、如何评估，以及可能属于哪个预算或负责人。

| 选择 | 适用情境 | 主要成本 |
| --- | --- | --- |
| 现有品类 | 客户已经使用有效标准，产品优势能在其中被看见 | 强势现有公司与继承的预期 |
| 子品类或限定词 | 原有框架基本有效，但一项条件会改变最佳选择 | 限定词可能被复制或忽略 |
| 重构品类 | 客户理解组成部分，但需要不同的评估优先级 | 教育、证明与销售一致性 |
| 新品类 | 现有框架持续掩盖价值，公司能够长期教育市场 | 时间、资金与注意力成本高，采用结果不确定 |

创建品类不是命名工作。它需要持续教育、可信示例、分析师与生态理解、客户证据、销售赋能，以及足够的市场需求让新框架持续存在。

验证品类理解、替代方案回忆、评估标准、销售推进与采用预期。一个只增加关注却吸引错误客户的流行标签，会削弱定位。

## 定位架构

### 单产品

当买家、替代方案、价值、品类和证据一致时，使用一个定位。不同分群的信息仍可以变化，不一定需要创建独立定位。

### 平台与产品组合

有意识地选择：

- 独立产品定位；
- 公司统一定位加产品专属定位；
- 一个由模块支撑的平台定位；
- 以集成价值为核心的产品套件定位；
- 基于共享能力的分群定位。

架构应该解释购买入口、共享价值、Cross-sell 逻辑、证据与产品边界。当统一定位变得空泛，或要求每位买家理解不需要的能力时，组合定位就会失效。

### 分群与市场定位

当不同分群的替代方案、价值优先级、评估标准、证据或购买流程存在实质差异时，可以使用不同定位。翻译或 Campaign 变化本身不能证明需要新定位。

### 多市场定位

国际扩张需要重新验证本地替代方案、术语、品类理解、渠道、信任、监管、价格预期与证据。核心优势仍成立时应保持一致，但不能把全球表达强加给购买语境不同的市场。

## 定位顺序

按依赖顺序工作：

1. 定义产品、客户、购买情境、市场、路径、时间范围与负责人。
2. 找出竞争替代方案，包括现状与不行动。
3. 列出当前真实存在的独特能力。
4. 把能力转化成差异化价值与证据。
5. 识别最重视并能获得该价值的客户。
6. 选择最容易解释价值的品类或重构方式。
7. 只在趋势能加强紧迫性或相关性时加入趋势。
8. 把定位转化成信息、叙事、资格判断、Offer 与旅程决策。
9. 验证行为、保留反面证据，并管理定位版本。

如果从口号、品类、宽泛 Persona 或趋势开始，团队可能在理解客户选择原因之前锁定错误假设。

## 定位证据账本

| 主张 | 支持证据 | 反面证据 | 状态 |
| --- | --- | --- | --- |
| 客户把我们与 X 比较 | Win/Loss、替换行为、销售与搜索语言 | 客户持续使用另一种框架 | Verified、inferred 或 missing |
| 能力 Y 具有实质差异 | 产品、技术、运营或生态证据 | 替代方案在相关情境下可以提供 | Verified、inferred 或 missing |
| 差异 Y 创造价值 Z | 结果数据、客户语言、避免成本、行为 | 有兴趣但没有实现或持续价值 | Verified、inferred 或 missing |
| 分群 A 最匹配 | 紧迫性、转化、激活、留存、支付意愿 | 服务负担高、留存弱、难以触达 | Verified、inferred 或 missing |
| 品类 C 能解释决策 | 理解度、正确替代方案、评估行为 | 困惑、错误预期、不合格需求 | Verified、inferred 或 missing |
| 证据 P 可信 | 可重复演示、客户 Reference、审计、Benchmark、结果 | 选择偏差、无法验证、情境不兼容 | Verified、inferred 或 missing |

记录证据日期、单位、分群、来源类型、负责人和不确定性。定位不只要记录支持故事的内容，也应明确缺少什么。

## 定位工作坊

让掌握不同证据的产品、销售、客户成功、增长、市场接触团队和管理层共同参与。会前收集近期 Win/Loss、流失、客户语言、产品行为、定价、支持、竞争与市场证据。

工作坊应该产出：

- 一项范围明确的定位假设；
- 替代方案地图；
- 能力、价值与证据链；
- 最佳匹配与排除条件；
- 品类决策与教育成本；
- 未解决分歧与证据缺口；
- 负责人、验证工作、版本与复盘日期。

不要用会议协商口号。不同职能存在分歧时，明确哪项观察能够解决问题，并分配验证责任。

## 信息层级

Messaging 把定位转化成面向特定受众与时刻的表达。

```text
定位假设
-> 品类与问题框架
-> 差异化价值支柱
-> 能力与证据
-> 受众和阶段信息
-> 渠道执行
```

维护统一层级，避免网站、产品、销售、生命周期沟通、合作伙伴与客户成功创造互相冲突的评估标准。不同版本可以调整重点、证据、术语与细节，但不应该创造不同的产品承诺。

## 销售叙事

销售叙事应该先帮助客户理解决策，再展示产品：

1. 市场洞察或变化。
2. 现有替代方案及其限制。
3. 理想未来与决策标准。
4. 差异化价值。
5. 产品能力与可信证据。
6. 异议、权衡、实施与下一步。

叙事只能教育证据能够支持的内容。戏剧化市场故事无法补偿薄弱价值，同一叙事也不应该被强加给互不兼容的分群。

## 客户资格判断与 Discovery

从定位建立 Qualification，判断潜在客户是否：

- 具有相关问题、触发事件与当前替代方案；
- 面临产品所解决的约束；
- 重视差异化结果；
- 具备实现价值的前提条件；
- 具有可接受的购买、实施、治理与经济路径；
- 能够理解可信证据与权衡。

Discovery 应保留反面证据，并主动排除不匹配需求。按分群和承诺场景衡量合格客户推进、Win/Loss、周期、折扣、实施成功、激活、续费与扩张。

## 转化到增长系统

| 领域 | 定位决策 | 需要观察的证据 |
| --- | --- | --- |
| 获客 | 受众、问题框架、品类、信息、证据与渠道语境 | 合格响应、来源质量、信息与分群匹配、下游留存 |
| 定价与套餐 | 价值指标、套餐边界、支付意愿、证据与权衡 | 价格接受、折扣、结构、毛利、扩张、服务成本 |
| 销售 | 资格判断、Discovery、叙事、异议、Reference 与竞争策略 | 推进、Win/Loss、周期、证据使用、不决策比例 |
| 激活 | 产品承诺所设定的首次价值路径与预期 | Time to Value、预期场景、设置成功、早期失败 |
| 留存 | 定位隐含的重复价值与客户条件 | 场景留存、续费、扩张、收缩与流失原因 |
| 产品 | 能力优先级与能够产生证据的体验 | 采用、结果质量、可靠性与客户证据 |

如果获客表现良好，但激活或留存薄弱，应检查信息是否吸引错误客户或承诺错误价值。如果留存很强但销售薄弱，应先检查品类理解、证据、资格判断与购买路径，再改变产品假设。

## AI 定位、信任与证据

“AI-powered”是许多产品共有的机制主张。更强的 AI 定位需要说明：

- 替代或改善什么工作流与方案；
- 这项能力对哪种任务和客户重要；
- 输出质量、可靠性、控制、延迟与成本条件；
- 专有语境、数据、工作流集成、反馈或网络优势；
- 隐私、安全、合规、审计与人工复核边界；
- 在精心筛选演示之外仍然可信的证明与结果；
- 当模型和竞争对手进步时，定位如何变化。

信任属于价值链的一部分。分别证明模型能力、产品可靠性、工作流匹配、治理与客户结果。一个 Benchmark 可以支持其中一环，但不能证明完整定位。

## 定位测试系统

通过测试组合验证定位，不要只依赖一个 Headline 实验。

| 测试层级 | 方法 | 失败信号 |
| --- | --- | --- |
| 替代方案 | 访谈、Win/Loss、流失去向、销售对话编码 | 团队与客户的竞争对象完全不同 |
| 理解度 | First-click、Five-second Test、信息访谈、品类回忆 | 正确受众无法解释产品替代什么 |
| 价值 | 问题访谈、证据评审、支付意愿研究 | 兴趣无法转化成有意义优先级 |
| 资格判断 | Lead 复盘、Discovery 审计、分群推进 | 数量上升，但匹配与下游价值下降 |
| 销售 | 叙事测试、异议分析、竞争结果 | 更多教育只延长周期，没有改善 Win |
| 产品 | 承诺场景激活与 Cohort（同期群） | 客户因为不同原因激活，或无法获得承诺证据 |
| 经济性 | 价格、套餐、折扣、服务与贡献分析 | 最容易理解的定位吸引了不经济的需求 |

信息测试只是辅助证据。完整验证还需要合格获客、购买行为、价值实现、留存与经济性。

## 衡量系统

| 层级 | 指标 |
| --- | --- |
| 受众匹配 | 最佳客户在合格需求中的比例、分群覆盖、来源结构、排除准确度 |
| 理解度 | 正确问题、替代方案、品类、价值与证据回忆 |
| 购买行为 | 合格推进、不决策、Win/Loss、销售周期、异议、竞争替换 |
| 价值实现 | 承诺场景激活、Time to Value、实施成功、结果证据 |
| 商业匹配 | 价格接受、折扣、套餐结构、贡献、扩张、Payback |
| 持续匹配 | 留存、续费、收缩、流失原因、按承诺场景划分的推荐 |
| 系统质量 | 证据新鲜度、主张状态、信息一致性、赋能采用、复盘完成度 |

按定位版本和分群同时报告比例与绝对贡献。汇总改善可能掩盖只对很小或不经济客户群有效的定位。

## 诊断地图

| 观察到的模式 | 下一步检查 | 可能响应 |
| --- | --- | --- |
| 高流量、低合格响应 | 受众、问题框架、品类与替代方案 | 收窄定位或改变获客语境。 |
| Lead 数量强、激活弱 | 承诺、资格判断、前提与首次价值路径 | 让信息和新用户引导与可实现价值一致。 |
| 客户理解功能，但没有紧迫性 | 现状成本、触发事件、差异化结果与证据 | 强化价值链，或接受更长教育路径。 |
| 经常因为“我们已经在用 X”失败 | 真实替代方案、切换成本与迁移证明 | 围绕真实决策重建比较。 |
| 兴趣强，但反复不决策 | 不行动、权限、风险、证据与实施负担 | 降低不确定性，或优先更紧迫情境。 |
| 留存强、销售转化弱 | 品类、叙事、证据可达性与购买角色 | 改善决策框架，再考虑改变产品价值。 |
| 销售成交、续费弱 | 过度承诺、分群匹配、实施与价值实现 | 收窄主张，并把流失证据反馈到资格判断。 |
| 同一定位在不同地区失效 | 本地替代方案、术语、监管、渠道与信任 | 差异实质存在时建立市场专属版本。 |
| 产品组合信息空泛 | 购买入口、共享价值、产品边界、Cross-sell 逻辑 | 拆分产品定位或明确平台架构。 |
| AI 演示带来 Trial，但没有留存 | 工作流匹配、可靠性、控制与重复价值 | 用已证明的场景价值替代通用能力主张。 |

这些模式用于指导调查，不能单独建立因果结论。

## 成熟度模型

| 阶段 | 特征 | 下一项约束 |
| --- | --- | --- |
| 1. 无范围主张 | 受众宽泛、功能列表、通用利益、口号驱动 | 定义单位、替代方案与证据状态。 |
| 2. 证据支持的假设 | 替代方案、能力、价值、最佳客户与品类明确 | 转化成证据、叙事、资格判断与旅程决策。 |
| 3. 旅程一致执行 | 获客、销售、产品、价格、激活与留存使用共同定位 | 按分群衡量，并保留反面证据。 |
| 4. 受治理的组合 | 版本、负责人、架构、证据账本与复盘触发器共享 | 在不失去一致性的情况下管理本地与产品组合差异。 |
| 5. 自适应定位系统 | 定位随着产品、市场、竞争、模式与客户证据演化 | 在变化中保持可比性、信任与战略聚焦。 |

成熟度不取决于信息、Persona 文档或品牌规范的数量，而取决于团队能否基于证据做出并修正一致的产品与增长决策。

## 建设顺序

1. 选择一个重要产品、分群、购买情境与运营周期。
2. 指定决策负责人，并收集客户、产品、销售、定价、留存与竞争证据。
3. 梳理直接、间接、人工、现状与不行动替代方案。
4. 建立能力、后果、价值与证据链，并注明边界。
5. 通过分群系统定义最佳匹配条件与排除项。
6. 选择品类，并记录教育成本和被否决的框架。
7. 写出一个定位假设和对应证据账本。
8. 转化成信息层级、销售叙事、资格判断、定价、激活与留存预期。
9. 使用护栏执行理解、行为、价值与经济性测试。
10. 批准版本、发布赋能材料、分配证据工作并确定下次复盘。

## 复盘节奏与治理

按固定运营节奏复盘定位，并在以下触发器发生时增加复盘：

- 新产品、套餐、平台、分群、市场或市场路径；
- 产品能力、竞争、监管、价格或客户行为变化；
- 重复出现的 Win/Loss、不决策、激活、留存或流失模式；
- 品类变化或新技术改变客户预期；
- 产品组合扩张、收购、整合或品牌架构变化。

复盘需要回答：

- 哪些主张获得了新支持或反面证据？
- 最佳客户与替代方案是否仍然正确？
- 品类是否产生了预期评估标准？
- 信息、销售、定价、产品、激活与留存是否一致？
- 每个范围正在使用哪个版本，谁负责例外？
- 哪些内容应该修改、验证、保留或退出？

保留决策日志。不要让 Campaign、销售 Deck、网站文案或功能发布在没有治理的情况下悄悄改变定位。

## 护栏

- 不制造缺少产品或运营证据的差异化。
- 不通过重新定义品类回避真实竞争对手。
- 不把趋势、技术、功能、口号或品牌人格误认为客户价值。
- 不面向无法获得承诺价值的客户。
- 不允许局部信息创造互不兼容的产品承诺。
- 没有实验或可信比较时，不把公司增长归因于定位。
- 不把公司结果归因于一名顾问、高管或单一团队。

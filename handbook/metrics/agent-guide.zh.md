# AI Agent 指标使用指南

本指南定义 AI Agent 如何读取、选择和表达增长指标。通用指标的定义、公式和业务用途请参阅 [指标体系](README.zh.md)。

## 基本规则

1. **先明确业务目标。** 不根据现有报表字段倒推目标。
2. **每条建议连接一个主指标。** 必须说明建议希望改变什么结果。
3. **同时选择领先指标。** 用可更快观测的行为信号判断动作是否生效。
4. **同时选择护栏指标。** 防止局部指标改善却损害毛利、留存、品牌或线索质量。
5. **说明观察周期和分群。** 不混合不同市场、渠道、用户群或时间窗口的数据。
6. **区分事实、推断和建议。** 数据不足时明确指出缺口，不伪造基线或提升幅度。

## Agent 与指标映射

| Agent | 主要读取指标 | 主要影响指标 | 典型判断 |
| --- | --- | --- | --- |
| Market Research Agent | PMF 信号、Persona、Segmentation、竞品渠道表现 | 合格流量、激活率、转化率 | 市场、人群和定位是否匹配 |
| SEO/GEO Agent | SEO、SEM、UV、PV、CTR、AI 答案可见性 | 自然流量、品牌可见性、高意图点击 | 用户是否能在搜索和 AI 答案里找到你 |
| Content Agent | UGC、KOL、内容互动、捕获率、辅助转化 | CTR、CVR、内容辅助收入 | 内容是否连接用户问题和购买决策 |
| CRO Agent | Funnel、CVR、TTV、CPA、退款率 | 转化率、每访问者收入、线索质量 | 页面、Offer、CTA 和信任模块哪里拖后腿 |
| Retention Agent | Retention、Churn、DAU、MAU、LTV、复购率 | LTV、留存率、复购/续费率 | 成交后是否有生命周期运营 |
| Experiment Agent | A/B Test、Lift、主指标、护栏指标 | 实验速度、学习复用率、单实验影响 | 哪些想法值得测，结果是否可决策 |
| Analytics Agent | LTV、CAC、ROI、ROAS、ARPU、Cohort | 决策质量、复盘速度 | 增长变化由哪个驱动因素造成 |

## 指标选择顺序

Agent 应按以下顺序选择指标：

```text
业务目标
→ 北极星指标或商业结果指标
→ 当前漏斗阶段的主指标
→ 能快速反馈的领先指标
→ 防止副作用的护栏指标
→ 分群、基线和观察周期
```

如果无法确认业务目标、基线或数据口径，Agent 应先提出数据需求，不直接给出确定性结论。

## 输出规范

每次分析或建议至少包含以下字段：

| 字段 | 要求 |
| --- | --- |
| 业务目标 | 当前希望改善的用户价值或商业结果 |
| 观察对象 | 市场、渠道、用户分群、产品或页面 |
| 主指标 | 用于判断结果是否改善的核心指标 |
| 领先指标 | 比主指标更早变化的行为信号 |
| 护栏指标 | 用于判断是否产生负面副作用的指标 |
| 当前基线 | 已知数值、时间窗口和数据来源；未知时明确标记缺失 |
| 建议动作 | 与指标变化机制直接相关的具体动作 |
| 验证方式 | 观察周期、实验方法和成功条件 |
| 风险与假设 | 结论依赖的假设、数据缺口和潜在副作用 |

## 输出示例

```text
业务目标：提高新注册用户首次体验到核心价值的比例
观察对象：过去 30 天来自自然搜索的新注册用户
主指标：Activation Rate
领先指标：完成首次关键行为的用户比例、TTV
护栏指标：7 日留存率、客服工单率
当前基线：Activation Rate 28%，TTV 中位数 18 分钟
建议动作：将首次关键行为前的必填步骤从 5 个减少到 3 个
验证方式：运行两周 A/B 测试，主指标相对提升 10%，护栏指标不得显著下降
风险与假设：假设埋点完整；需要排除内部测试账号和重复注册
```

Agent 的任务不是罗列尽可能多的指标，而是选择足以支持下一步决策的最小指标组合。

# 指标体系

指标把增长目标变成可运营模型。增长营销系统里的指标，不只是报表字段，而是团队判断问题、生成建议、设计实验和复盘结果的共同语言。

**[English](README.md)｜[简体中文](README.zh.md)**

## 定义与边界

指标是对行为、价值、质量、成本、风险或系统状态的定量定义。衡量是把事件和记录转化为可信决策输入的完整过程。

| 术语 | 含义 | 边界 |
| --- | --- | --- |
| 事件 | 注册、支付、完成任务或错误等被记录的发生 | 在资格与聚合得到定义前，事件只是原始证据，不是指标。 |
| 指标 | 拥有实体、人群、公式与窗口的定量定义 | 指标不会因为可以获得就自动重要。 |
| KPI | 为特定目标与负责人选出的指标 | KPI 地位取决于决策，而不是名称。 |
| 目标 | 一段时间内指标期望值或区间 | 目标不能证明数值可实现或健康。 |
| Benchmark | 来自另一时期、分群或公司的参考分布 | 迁移需要可比较定义、环境与成熟度。 |
| 维度 | 市场、来源、设备、角色或 Cohort 等拆分 | 过多拆分会制造噪音与 False Discovery。 |
| 模型输出 | LTV、风险、Propensity 或归因功劳等预测 | 需要校准与不确定性，并与已观察结果分开。 |

指标描述系统，但不能单独解释因果。Dashboard 可以揭示调查位置，实验、因果设计与客户证据用于检验变化为何发生。

## 使用原则

- **先定义业务目标，再选择指标。** 不要为了看起来专业而堆指标。
- **每个指标都要有决策用途。** 如果一个指标不会改变下一步动作，它就不是核心指标。
- **主指标必须配护栏指标。** 转化率提升但退款率上升，不能算健康增长。
- **结果指标要拆成领先指标。** LTV、收入、GMV 太滞后，需要拆到流量、转化、留存、复购和实验动作。
- **Agent 输出必须连接指标。** 每个分析、内容、CRO、实验或留存建议，都应该说明影响哪个指标。

## 指标定义规范

每个核心指标都应该拥有可复现、带版本的定义。

```text
指标名称与用途：
决策与负责人：
实体与分析单位：
合格人群：
起始事件与合格事件：
分子与分母：
聚合与公式：
观察与成熟窗口：
时区与货币：
身份与去重规则：
排除与缺失数据处理：
分群与 Cohort 切分：
来源表或事件：
护栏与相关指标：
已知局限：
版本与生效日期：
```

定义变化时，保留旧版本或一致地重算历史数据。静默改变分母、身份、归因或窗口会制造人工增长。

## 指标分层

```text
北极星指标
├── 商业健康指标: LTV, CAC, LTV/CAC, ROI, ROAS, ARPU
├── 漏斗指标: UV, CTR, CVR, Activation Rate, TTV
├── 留存指标: Retention, Churn, DAU, MAU, DAU/MAU
├── 渠道指标: CPC, CPM, CPA, CPL, SEO, SEM, UGC, KOL
├── 实验指标: A/B Test, Lift, 主指标, 护栏指标
└── 增长结构: AARRR, Growth Loop, Flywheel, PMF, NSM
```

## 北极星指标原则

一个有用的北极星指标应该：

- 代表已经交付的用户价值。
- 能连接收入或毛利。
- 能被增长系统影响。
- 能拆解成领先指标。
- 不奖励低质量获客。

GMV 或收入有时有用，但通常太宽，难以指导每周动作。

## 指标星座

North Star 应该位于一套精简的输出、输入与 Trade-off Metrics 系统中。

| 指标角色 | 作用 | 示例 |
| --- | --- | --- |
| 输出指标 | 表示持续客户或业务结果 | Weekly Active Teams、留存收入 |
| 输入指标 | 表示团队可以直接改变的行为 | 完成协作工作流的团队数 |
| Trade-off Metric | 检测改善其他指标时产生的伤害 | 通知退订、内容新鲜度、毛利 |

输出指标通常是滞后计分板。团队需要持续拆解，直到获得足够接近产品或渠道行为、能够支持决策的输入指标。输入仍然是假设：如果实验推动输入却没有改善输出，就应该放弃或替换它。

当留存、参与、商业化、供给或质量可能向相反方向变化时，不要强行把它们合并成一个数字。精简的指标星座既保持聚焦，也让这些关系保持可见。

## 指标树与因果关系

指标树把长期结果拆解为团队可以观察或改变的驱动因素。算术拆解不自动代表因果。

```text
长期客户与业务结果
├── 价值交付与留存使用
├── 合格获客与激活
├── 商业化与贡献
├── 扩张或网络健康
└── 质量、信任、风险与容量护栏
```

对每条边记录假设机制、证据、负责人、延迟与潜在取舍。输入指标仍是假设，只有干预推动输入后，预期结果也随之改善且没有不可接受伤害时，关系才得到更强支持。

只有组成项相互排斥且完整时，才能使用加法恒等式；只有分母和资格保持一致时，才能使用乘法漏斗。不要把来自不同 Cohort 或窗口的平均值相乘。

## 实体、分母、窗口与 Cohort

同一个指标名称会因为实体和分母不同而描述完全不同的系统。

| 设计选择 | 核心问题 | 常见失败 |
| --- | --- | --- |
| 实体 | 用户、账户、Workspace、交易、Listing、产品、市场还是家庭？ | 在同一比例中混合用户与账户。 |
| 资格 | 谁真正拥有完成行为的机会？ | 把遇到摩擦的用户移出分母。 |
| 起始事件 | 计时从何时开始？ | 比较注册 Cohort 与首次使用 Cohort。 |
| 合格事件 | 什么准确价值或状态才算完成？ | 把界面完成当成客户价值。 |
| 窗口 | Session、日、周、月、合同还是机会周期？ | 比较未成熟与成熟 Cohort。 |
| 身份 | 如何处理设备、账户、匿名用户与合并？ | 重复计数或处理组穿越。 |
| Cohort | 哪个共同起点或状态使比较有效？ | 混合日历总量与获客 Cohort。 |
| 分群 | 哪项切分能够改变决策？ | 搜索大量切分直到出现有趣结果。 |

同时报告比例与绝对贡献。保留原始资格和 Cohort 归属直到下游读数。

## 已观察与预测指标

已观察指标总结被记录的结果；预测指标估计未来或未观察状态，例如 LTV、流失风险、Propensity、归因功劳、需求或作弊。

| 层级 | 必要证据 |
| --- | --- |
| 已观察结果 | 稳定事件、身份、分母、窗口与数据质量 |
| 预测 | 训练人群、特征、目标、周期、不确定性与校准 |
| 决策策略 | 预测如何改变处理或资源分配 |
| 结果验证 | 成熟已观察结果、分群表现与伤害 |
| 监控 | Drift、缺失、反馈循环、偏差与 Override 行为 |

模型可能在平均意义上准确，却在重要市场或分群失败。按决策相关 Cohort 报告校准与误差。不能仅为了证明更多投入合理而延长 LTV 周期。

## 按产品结构设计衡量体系

### 消费与习惯型产品

使用自然频率上的价值行为、Cohort 留存、参与分布、召回、信任与商业化。只有活动代表价值时，DAU 或 MAU 才有意义。

### B2B 与销售驱动产品

分开用户、Workspace、账户、买方与收入实体。连接产品激活、实施、活跃席位、工作流采用、账户留存、Pipeline、贡献、回本与支持负担。

### 协作产品

分别衡量个人首次价值与多用户价值。邀请、席位与共享产物需要协作者激活、重复工作流、权限、账户留存与非活跃席位护栏。

### Marketplace 与网络产品

分别衡量每一方和每个本地网络单元，包括供给可用性、需求、密度、匹配或履约、Time to Value、质量、双边留存、参与者经济性、激励与治理。

### 多产品与产品组合业务

保留产品级结果，同时衡量产品组合增量。互导、共享身份、激励与预算分配需要蚕食、重叠、贡献与客户总价值检查。

### AI 产品

使用量、Prompt、Token 或生成输出不能建立价值。衡量成功任务、结果质量、验证、修改、延迟、成本、信任、重复工作流完成与客户留存。

## 常见北极星指标

### DTC 品牌

推荐候选：

- 贡献毛利为正的首单。
- 复购用户数。
- 来自优先市场的合格购买数。

指标树：

```text
贡献毛利
= 流量 x 转化率 x AOV x 毛利率
- CAC
- 履约和运营成本
```

### 订阅型品牌

推荐候选：

- 活跃且有利润的订阅用户。
- 净收入留存。
- 90 天后仍留存的订阅 cohort。

指标树：

```text
订阅用户价值
= 新订阅用户 + 留存订阅用户 + 扩展收入
- 流失
- 折扣和客服成本
```

### 平台或 marketplace 电商

推荐候选：

- 来自优先 listing 的有利润订单。
- 按品类统计的复购买家。
- 搜索可见且转化健康的 SKU。

指标树：

```text
Listing 增长
= 曝光 x 点击率 x 转化率 x 评论质量
```

### 内容驱动品牌

推荐候选：

- 捕获到的高意图受众。
- 内容辅助购买数。
- 完成产品教育节点的合格订阅用户。

指标树：

```text
内容增长
= 合格触达 x 互动深度 x 捕获率 x 辅助转化
```

## 指标库

### 一、核心商业指标

| 缩写/名称 | 英文 | 中文 | 用途 | 常见公式或判断 |
| --- | --- | --- | --- | --- |
| LTV | Lifetime Value | 用户生命周期价值 | 衡量用户在整个生命周期贡献的总收入或毛利 | 平均收入 x 毛利率 x 留存周期 |
| CAC | Customer Acquisition Cost | 获客成本 | 衡量获取一个新用户的平均成本 | 获客总成本 / 新客户数 |
| LTV/CAC | LTV to CAC Ratio | 生命周期价值比 | 判断留存贡献能否支持获客 | 不存在通用健康比例；需要明确贡献口径、周期、不确定性、回本与现金约束 |
| ROI | Return on Investment | 投资回报率 | 衡量投入与产出的收益比例 | (收益 - 成本) / 成本 |
| ROAS | Return on Ad Spend | 广告投入回报 | 衡量广告花费带来的收入回报 | 广告收入 / 广告花费 |
| ARPU | Average Revenue Per User | 每用户平均收入 | 衡量所有用户平均收入贡献 | 总收入 / 总用户数 |
| ARPPU | Average Revenue Per Paying User | 每付费用户收入 | 衡量付费用户平均收入水平 | 总收入 / 付费用户数 |
| GMV | Gross Merchandise Volume | 成交总额 | 衡量平台或交易型业务规模 | 成交金额总和，不等于收入 |

### 二、用户获取

| 缩写/名称 | 英文 | 中文 | 用途 | 常见公式或判断 |
| --- | --- | --- | --- | --- |
| UV | Unique Visitors | 独立访客数 | 衡量去重后的访问用户数量 | 去重访客数 |
| PV | Page Views | 页面浏览量 | 衡量页面被浏览的总次数 | 页面访问次数总和 |
| CTR | Click Through Rate | 点击率 | 衡量曝光转化为点击的比例 | 点击数 / 曝光数 |
| CPC | Cost Per Click | 单次点击成本 | 衡量每次点击成本 | 广告花费 / 点击数 |
| CPM | Cost Per Mille | 千次展示成本 | 衡量每 1000 次曝光成本 | 广告花费 / 曝光数 x 1000 |
| CPI | Cost Per Install | 单次安装成本 | 衡量 App 安装获客成本 | 广告花费 / 安装数 |
| CPA | Cost Per Action | 单次转化成本 | 衡量注册、购买等目标行为成本 | 广告花费 / 目标行为数 |
| CPL | Cost Per Lead | 单条线索成本 | 衡量潜在客户获取成本 | 获客花费 / 线索数 |

### 三、激活与转化

| 缩写/名称 | 英文 | 中文 | 用途 | 常见公式或判断 |
| --- | --- | --- | --- | --- |
| CVR | Conversion Rate | 转化率 | 衡量完成目标行为的用户比例 | 转化人数 / 访问或触达人群 |
| Funnel | Funnel | 转化漏斗 | 分析用户路径各阶段流失 | 入口、激活、转化、成交、留存 |
| Activation Rate | Activation Rate | 激活率 | 衡量首次价值达成比例 | 完成关键行为用户 / 新用户 |
| TTV | Time to Value | 价值实现时间 | 衡量用户感知价值所需时间 | 从首次触达到首次价值的时间 |

### 四、留存与流失

| 缩写/名称 | 英文 | 中文 | 用途 | 常见公式或判断 |
| --- | --- | --- | --- | --- |
| Retention Rate | Retention Rate | 留存率 | 衡量某周期后仍然活跃的用户比例 | 留存用户 / 初始 cohort 用户 |
| Churn Rate | Churn Rate | 流失率 | 衡量一定周期内离开的用户比例 | 流失用户 / 周期初用户 |
| DAU | Daily Active Users | 日活跃用户 | 衡量每日活跃规模 | 当日活跃用户数 |

## 使用频率分布

DAU/MAU 等混合黏性指标可能隐藏差异很大的用户群。Power User Curve 展示用户在固定周期内活跃 1 天、2 天直到全部天数的数量或占比。

```text
L7 = 一周内分别活跃 1、2、...、7 天的用户分布
L30 = 一个月内分别活跃 1、2、...、30 天的用户分布
```

尽量使用承载价值的行为，而不只是打开应用。时间窗口应匹配产品节奏，并按 Cohort、分群、市场和时期对比。健康产品不一定需要每日使用或形成微笑曲线。

## Net Promoter Score

```text
NPS = 推荐者占比（9-10）- 贬损者占比（0-6）
```

NPS 是用户自述推荐意愿的调查信号，不是已观察到的推荐、留存或收入。需要收集评分原因，使用有代表性的样本，控制参与度与用户年限偏差，并把反复出现的主题连接到产品决策。

## CAC 与 CPA

- **CAC** 衡量获得一个明确定义的付费客户所需成本。
- **CPA** 衡量获得 Lead、注册、试用或已激活免费用户等其他状态的成本。

Fully Loaded CAC 应定义客户，按照转化延迟匹配投入，区分新增和回流客户，并纳入相关人员、工具、管理费用与支持成本。应同时报告单渠道、边际和 Blended CAC。

## 漏斗指标

- **获客：** 合格流量、UV、PV、CTR、CPC、CPM、CPA、CPL、CAC、达人 ROI、自然可见性。
- **激活：** Activation Rate、TTV、关键行为完成率、email/SMS 订阅、加购。
- **转化：** CVR、Funnel 各阶段转化、结账完成率、每访问者收入、首单毛利。
- **留存：** Retention Rate、Churn Rate、DAU、MAU、DAU/MAU、复购率、cohort LTV。
- **收入：** GMV、MRR、ARR、ARPU、ARPPU、LTV、LTV/CAC、ROI、ROAS。
- **裂变：** K-factor、Virality、Referral Rate、评论率、UGC 量、推荐收入。

## 护栏指标

增长指标必须搭配护栏：

- 毛利率。
- 退款和退货率。
- 每订单客服工单数。
- 退订和 spam complaint。
- 折扣依赖。
- 评论质量。
- 品牌信任信号。
- 低质量线索比例。
- 付费回本周期。

## 诊断地图

| 观察到的模式 | 下一步检查 | 可能的处理 |
| --- | --- | --- |
| North Star 上升而留存下降 | 实体构成、输入质量、Cohort 成熟度 | 重新定义结果指标或加强留存护栏。 |
| 漏斗转化提升但绝对增长没有变化 | Eligible Volume、归因重叠、下游贡献 | 同时读取绝对变化与增量。 |
| 收入增长但贡献变差 | 折扣、服务成本、结构、毛利、获客成本 | 把决策指标从收入改为留存贡献。 |
| 聚合结果稳定但关键市场下降 | 市场、分群、来源、角色与 Cohort 构成 | 区分结构性 Mix Change 与分群内部变化。 |
| 预测 LTV 上升但成熟 Cohort 没有改善 | 模型周期、校准、特征漂移、选择偏差 | 重新校准、缩短决策周期或限制资源分配。 |
| Marketplace 注册增长但履约变差 | 本地供给、密度、时间覆盖、参与者质量 | 衡量 Atomic Network 与成功互动。 |
| 活跃席位增长但协作没有增加 | 席位资格、角色、邀请质量、工作流采用 | 衡量承载价值的活跃席位与账户结果。 |
| Dashboard 数字不一致 | 定义版本、身份、时区、来源、排除条件 | 暂停决策并对齐指标契约。 |

这些模式用于缩小调查范围，不能单独建立因果关系。

## 成熟度模型

| 阶段 | 特征 | 下一项约束 |
| --- | --- | --- |
| 1. 报表 | 团队收集可获得的总量与渠道指标 | 定义实体、决策、分母、窗口与负责人。 |
| 2. 指标分层 | 结果、输入、诊断与护栏明确 | 连接 Cohort、分群、经济性与数据质量。 |
| 3. 决策系统 | 指标定义可版本化，复盘能够改变行动 | 加入因果证据、预测校准与产品结构视图。 |
| 4. 受治理的自助分析 | 共享语义定义支持本地分析与实验 | 控制漂移、例外、权限与模型反馈循环。 |
| 5. 自适应衡量 | 指标与模型随产品、客户和约束变化 | 保持可比性、简洁、信任与人工判断。 |

成熟度不取决于指标、Dashboard 或模型数量。定义稳定、决策清晰的小型系统，通常强于责任模糊的大型指标目录。

## 建设顺序

1. 定义客户价值、业务结果、指标实体与决策负责人。
2. 选择一个结果指标，以及最小可用的输入、诊断、经济与护栏指标集合。
3. 写清指标定义、分母、事件、窗口、身份、排除条件与版本。
4. 建立指标树，并标记关系属于恒等式、相关、预测还是因果假设。
5. 验证事件、源数据、新鲜度、对账与可复现性。
6. 使用比例与绝对贡献，对比成熟度一致的 Cohort 和决策相关分群。
7. 用后续已观察结果校准代理与预测指标。
8. 使用实验或最强可行设计检验重要因果边。
9. 建立能够产生决策、负责人和后续行动的复盘节奏。
10. 退役无人使用的指标，并在产品、定价、市场、身份或战略变化时重新验证定义。

## 周度增长复盘

每周复盘应该回答：

- 发生了什么变化？
- 是哪个驱动因素造成的？
- 哪个市场、分群、渠道或产品解释了变化？
- 我们测试了什么？
- 我们学到了什么？
- 下一步要做什么？

# 激励增长

> 激励增长使用金钱、功能、社会或保护型奖励改变行为。只有留存增量价值高于成本、作弊与蚕食时，系统才可持续。

**[English](README.md)｜[简体中文](README.zh.md)**

## 激励模型

```text
合格分群 -> 目标行为 -> 奖励 -> 即时提升
         -> 产品价值 -> 激励后行为
         -> 成本、作弊、蚕食、信任
```

奖励可以是现金、积分、优惠券、权限、身份、进度保护、便利或社会认可。奖励必须与行为价值和用户动机匹配。

## 指标体系

| 指标 | 定义或公式 | 决策 |
| --- | --- | --- |
| 合格用户行动率 | `采取目标行为的用户 / 合格用户` | 即时行为变化 |
| 增量提升 | `实验组行动率 - Holdout 行动率` | 项目真正造成的行为 |
| 单次增量行动成本 | `奖励与运营总成本 / 增量行动数` | 效率 |
| 激励后留存 | 奖励结束后仍保留的目标行为 | 持续性 |
| 作弊损失率 | `无效奖励成本 / 奖励总成本` | 滥用风险 |
| 蚕食 | 为本来会自然发生的行为支付奖励 | 被替代的自然价值 |

## 问题诊断

| 模式 | 下一步检查 |
| --- | --- |
| 即时提升高，激励后留存弱 | 只追逐奖励，没有产品价值 |
| 总响应强，增量弱 | 自然用户获得了不必要奖励 |
| 成本增长快于行动 | 竞价、重复资格、边际响应下降 |
| 新分群增长，原分群下降 | 产品矩阵或渠道蚕食 |
| 投诉或作弊增加 | 资格公平性、规则清晰度、风控 |

## 操作方法

1. 定义分群、目标行为、价值假设、资格、奖励、周期、预算与停止条件。
2. 使用随机或可信 Holdout 估计增量。
3. 把奖励行为与自然产品信号分开。
4. 按产品自然周期衡量激励后留存。
5. 监控作弊、蚕食、毛利、信任与运营成本。
6. 增量价值不再高于完整成本时减少或停止奖励。

## 常见错误

- 奖励没有持续客户价值的行为。
- 把总行动数当成增量提升。
- 没有实时风控就上线现金奖励。
- 让长期经济性依赖临时补贴。

# 激励系统

> 激励系统通过金钱、功能、社会、保护、Marketplace 或结构性奖励改变参与者行为。只有留存增量价值高于完整成本、作弊、蚕食和信任损害时，它才创造可持续增长。

**[English](README.md)｜[简体中文](README.zh.md)**

## 定义与边界

激励会改变一个行动的预期收益、成本、身份、权限或风险。它可以帮助参与者跨过暂时障碍、协调 Marketplace、发现产品价值、邀请其他人、维持进度，或让企业收益与客户结果对齐。

激励不能创造产品市场匹配。它可以加速有价值行为，也可能购买在奖励结束后消失的活动、扭曲产品信号、吸引作弊，或让长期经济性依赖补贴。

```text
合格参与者
-> 摩擦、协调缺口或动机问题
-> 激励与目标行为
-> 即时响应
-> 体验产品或网络价值
-> 激励后行为
-> 留存增量价值减去完整成本与伤害
```

价格与套餐参见[商业化](../monetization/README.zh.md)，推荐机制与传播参见[推荐与扩张](../referral-and-loops/README.zh.md)。本模块关注奖励如何改变行为，以及这种改变在经济和伦理上是否成立。

## 激励分类

| 类型 | 示例 | 适用情况 | 主要风险 |
| --- | --- | --- | --- |
| 现金 | 直接付款、返现、提现 | 目标行为具有可衡量经济价值 | 作弊、奖励驱动行为与高运营成本 |
| Credit 与折扣 | 优惠券、余额、免费用量、免手续费 | 价值可以在产品内部实现 | 提前透支自然需求或掩盖弱支付意愿 |
| 产品容量 | 存储、额度、Seat、使用量 | 奖励强化产品核心价值 | 成本增长与低质量推荐 |
| 权限与便利 | Early Access、优先级、更快服务、解锁能力 | 摩擦或稀缺阻碍价值发现 | 不公平、特权感与服务退化 |
| 进度保护 | Streak Freeze、恢复、保险、宽限期 | 损失厌恶可能中断有价值习惯 | 强迫感或削弱底层行为 |
| 身份与社会认可 | Badge、排名、里程碑、曝光 | 身份和社区强化有意义贡献 | 刷身份、排斥与表演性活动 |
| Marketplace 补贴 | 司机保底、乘客优惠、卖家 Credit | 供需需要暂时协调 | 永久补贴、逆向选择与本地失衡 |
| 推荐奖励 | 发送者、接收者或双边收益 | 现有用户能够介绍合格接收者 | Spam、串谋、归因盗窃与社交信任损害 |
| 结构性对齐 | 客户成功时企业也获得回报的商业模式 | 客户与企业结果可以共享经济机制 | 只有对齐主张，却没有可衡量实现价值 |

同一种产物可能服务不同机制。优惠券可以降低试用风险、召回休眠客户、改变购买时间，也可能补贴本来就会发生的行为。分类从行为与反事实开始，而不是从奖励形式开始。

## 生命周期作用

| 增长环节 | 激励任务 | 示例决策 | 必须检查的下游结果 |
| --- | --- | --- | --- |
| 获客 | 推动合格到达或形成供给 | 推荐奖励或新市场补贴 | 激活、留存质量、CAC（客户获取成本）与作弊 |
| 激活 | 在首次价值前降低成本或恐惧 | 试用 Credit、新用户权益、风险逆转 | Time to Value 与不持续奖励后的行为 |
| 留存 | 支持重复行为或保护累计进度 | Streak 恢复、忠诚权益、服务补偿 | 自然回访、客户价值与疲劳 |
| 商业化 | 推动购买、升级或承诺 | 折扣、套餐 Credit、年付权益 | Margin、续费、支付意愿与提前透支 |
| 推荐与扩张 | 奖励创造价值的传播或更深采用 | 双边推荐或团队容量 | 接收者价值、增量、留存与滥用 |
| Marketplace 流动性 | 协调 Hard Side 并减少交易失败 | 供给保底或需求优惠 | 匹配质量、利用率、跨边留存与补贴后流动性 |

一个项目可能影响多个环节，但必须有一个主要决策和能够显示取舍的指标树。

## 行为模型

选择奖励前，先识别行为为什么没有发生。

| 障碍 | 激励假设 | 其他解释 |
| --- | --- | --- |
| 客户尚未发现价值 | 临时 Credit 降低试用风险 | 产品解释或新用户引导薄弱 |
| 行动具有真实参与成本 | 补偿使贡献值得进行 | 价值交换在结构上缺乏吸引力 |
| Marketplace 缺少协调 | 临时补贴形成足够本地流动性 | 供需匹配、密度或频率不足 |
| 有价值习惯较脆弱 | 进度保护避免一次中断造成流失 | 重复产品价值不足 |
| 推荐存在努力或社交风险 | 与接收者价值一致的奖励抵消摩擦 | 用户不相信产品能帮助其他人 |
| 客户与企业激励冲突 | 商业模式重构使结果对齐 | 定位主张强于实际经济性 |

激励在消除通往真实价值的暂时障碍时最合理，在持续付费让用户忍受薄弱产品时最不合理。

## 激励经济性

### 完整项目成本

```text
支付或授予的奖励价值
+ 支付与履约成本
+ 媒体、销售与合作伙伴成本
+ 工程与运营
+ 支持与争议处理
+ 作弊与滥用损失
+ Margin 损失
+ 被蚕食的自然价值
+ 风险与合规成本
= 完整项目成本
```

产品容量和虚拟货币并非免费。需要计入服务成本、支持、负债、兑换行为和授予权益的机会成本。

### 增量价值

```text
增量参与者或行动
x 转化为客户价值的留存比例
x 决策周期内的贡献价值
- 下游服务与风险成本
= 留存增量价值
```

可变成本显著时使用贡献而不是收入。明确周期、成熟假设、折现与不确定性。不能用乐观 LTV（客户生命周期价值）预测为实际 Cohort（同期群）薄弱的项目辩护。

### 决策规则

根据边际留存增量价值决定继续、扩大、重构、递减或停止，而不是只看历史平均响应。整体看似盈利的项目，最新分群、市场或奖励档位可能正在摧毁价值。

## 增量与反事实

总兑换、行程、邀请、课程或购买无法说明激励实际造成了什么。

在法律、伦理和运营允许时，优先随机分配资格或奖励金额。其他方案包括地域 Holdout、Switchback、分阶段上线、阈值设计、匹配对照或带明确假设的中断时间序列。

衡量：

- 合格人群与实际曝光；
- 相对 Holdout 的行动提升；
- 奖励替代的自然行为；
- 时间移动或提前透支；
- 跨渠道与跨产品移动；
- 奖励变化或结束后的留存行为；
- 作弊、支持、投诉与 Margin 影响。

归因与增量必须分开。推荐码或 Promotion ID 可以识别谁获得 Credit，却不能证明奖励造成了行动。

## 激励后留存

最关键的问题是奖励减弱或停止后还剩下什么。

```text
奖励 Cohort
-> 激励期间目标行为
-> 体验产品或网络价值
-> 奖励递减或移除
-> 与自然 Cohort 和 Holdout 比较留存行为
```

按照产品自然频率评估。每日学习、每月财务运营、低频旅行和司机供给需要不同窗口。

按照奖励曝光、是否到达价值、获客来源、市场、参与者侧和作弊风险分群。到达首次价值的用户可能留存良好，而只追逐奖励的参与者可能完全失败。

## Marketplace 激励

Marketplace 激励应针对明确本地约束：供给不足、可用性低、需求弱、履约差、取消率高或利用率低。

| 决策层级 | 示例指标 |
| --- | --- |
| 供给 | 活跃供给者、可用时长、覆盖、接受率 |
| 需求 | 合格请求、频率、交易意愿 |
| 流动性 | 匹配率、Fill Rate、匹配时间、完成率 |
| 参与者经济性 | 收入、利用率、成本、Margin、回本 |
| 质量 | 取消、支持、安全、信任与重复行为 |
| 持续性 | 激励递减后的流动性与留存 |

Marketplace 总增长会掩盖本地失败。按照具有不同流动性经济性的最小地域、品类、时间窗口或参与者分群设计和递减激励。

只补贴受约束的转化，而不是所有参与者。如果需求频率或供给方经济性无法支撑自然可用性，激励可能只是推迟结构性问题。

## 推荐奖励

推荐系统至少有两位客户：发送者和接收者。奖励必须保护接收者价值与社交信任。

```text
合格且满意的发送者
-> 带有明确接收者价值的邀请
-> 合格接收者到达
-> 激活与留存使用
-> 奖励验证与履约
-> 根据增量质量重复或停止
```

衡量发送者参与、每位发送者邀请数、接收者转化、接收者激活、留存质量、传播周期、奖励成本、Spam、投诉与滥用。缺少接收者留存和增量的 K-factor 可能奖励低质量传播。

当双方都获得与产品价值一致的收益时，双边奖励可能有效，但不一定总是更优；新增成本必须产生额外合格行为。

## 进度、身份与游戏化

进度系统通过 Streak、里程碑、等级、League、恢复和庆祝，让有价值的重复行为可见。即使没有现金，这些机制仍属于激励。

```text
有意义的客户行动
-> 可见进度
-> 认可或保护
-> 回到有意义行动
-> 持续客户结果
```

需要防止刷积分、无意义重复、强迫性使用损失厌恶、通知疲劳、身份排斥，以及通过用户害怕失去累计进度来收费。当奖励行为不再产生产品价值时，Engagement 不是有效结果。

## 作弊与滥用

现金和可转让价值会吸引主动作弊。非现金系统也可能为了身份、权限、曝光或算法优势被操纵。

| 攻击面 | 示例 | 控制 |
| --- | --- | --- |
| 身份 | 多账户、合成身份、账户转售 | 验证、账户年龄、图关系与支付信号 |
| 设备与网络 | 设备农场、模拟器、Proxy 轮换 | 设备完整性、网络模式、速度限制 |
| 推荐图 | 串谋网络、自我推荐、循环邀请 | 图分析、关系规则、延迟奖励 |
| 归因 | Code 盗取、Last-touch 劫持、合作伙伴作弊 | 签名事件、去重、Holdout、对账 |
| 交易 | 虚假订单、取消、退款滥用 | 延迟结算、完成规则、Risk Scoring |
| 进度与身份 | 自动化活动、低价值重复 | 质量阈值、异常检测、人工审核 |

风险控制应覆盖资格前、行动中、履约前和结算后。即时奖励改善体验但降低追回能力；延迟奖励减少损失却可能减弱动机，需要明确选择。

不能在不标记激励曝光的情况下，用奖励行为训练个性化系统，否则激励会污染偏好和价值信号。

## 蚕食与行为扭曲

蚕食包括：

- 为本来会自然行动的用户付费；
- 只把购买提前，没有增加总价值；
- 把客户从全价产品转移到补贴版本；
- 在内部渠道或产品之间移动活动；
- 从更健康的 Marketplace 区域抽走供给；
- 用奖励依赖替代内在动机；
- 教会客户等待折扣。

尽可能使用客户级和产品矩阵级 Holdout。比较总贡献、产品组合、留存和后续行为，而不是只评估被推广的界面。

## 资格、公平与信任

资格规则会传达产品重视谁。隐藏差异可能造成不公平感、歧视、合作伙伴冲突或监管风险。

记录：

- 合格与排除人群；
- 差异化处理原因；
- 数据与模型输入；
- 奖励可见性与条款；
- 过期、提现、追回与争议规则；
- 无障碍与市场特定限制；
- 投诉、申诉与补救路径。

避免 Dark Pattern、不清晰提现条件、欺骗性倒计时，以及主要为强迫付费而设计的进度损失。

## 实验与递减设计

上线前明确：

```text
决策与目标行为：
合格人群与排除项：
奖励类型、金额、支付者与接收者：
自然基线与 Holdout：
主要增量结果：
激活与留存价值指标：
成本与 Margin 周期：
作弊与蚕食检查：
信任与公平护栏：
预算与曝光上限：
递减与停止规则：
负责人和复盘节奏：
```

测试机制，而不只是金额。比较奖励与无奖励、不同接收者、产品价值奖励与通用价值、即时与延迟履约、固定与可变奖励，以及不同递减计划。

以下情况应停止或重构：

- 边际留存增量价值低于完整边际成本；
- 激励后留存持续低于要求；
- 作弊或运营损失超过风险上限；
- 自然行为或其他产品被明显蚕食；
- 质量、信任、公平或合规护栏失败；
- 维持相同响应需要不断提高奖励；
- 原始协调障碍已经不存在。

## 治理与运营

| 控制 | 必须明确的决策 |
| --- | --- |
| 预算 | 按分群、市场和周期限制总曝光与边际曝光 |
| 资格 | 规则、版本、负责人、例外与审计记录 |
| 履约 | 时机、结算、追回、负债与客户支持 |
| 风险 | 实时阻止、审核阈值、准备金与事故响应 |
| 经济性 | 批准的价值周期、成本口径、Margin 底线与校准 |
| 实验 | Holdout 完整性、排除、干扰与决策规则 |
| 沟通 | 清晰条款、过期、税务或法律影响与争议路径 |
| 产品矩阵 | 跨产品流量、渠道冲突与蚕食审核 |

增长、财务、风险、产品、法务、支持和 Marketplace 运营可能由不同团队负责，但必须有一位结果负责人整合这些约束。

## 成熟度模型

| 阶段 | 特征 | 下一项约束 |
| --- | --- | --- |
| 1. 临时奖励 | Campaign Code、手工履约、只报告总响应 | 定义资格、完整成本与自然基线 |
| 2. 可衡量项目 | 统一事件追踪、基础 Holdout、成本与作弊报告 | 增加激励后留存与分群经济性 |
| 3. 受治理组合 | 标准规则、风控、决策复盘、跨项目比较 | 改善边际分配与蚕食衡量 |
| 4. 自适应激励 | 受限个性化、自动资格、实时风控与递减 | 控制模型偏差、公平、漂移与战略依赖 |
| 5. 结构性对齐 | 激励和商业模式持续强化客户结果 | 验证实际价值并防止复杂度和隐藏冲突 |

更高成熟度不一定更好。经济性清晰的固定推荐权益，可能优于个性化奖励平台。

## 指标体系

### 行为与增量

- 合格用户行动率；
- 增量行动与参与者提升；
- 激活与 Time to Value；
- 增量交易、课程、邀请或供给时长；
- 置信区间与实际效应阈值。

### 持续性与质量

- 激励后留存与重复行为；
- 奖励移除前后到达的价值；
- 接收者、交易、学习或供给质量；
- 奖励依赖与递减响应；
- 跨边或产品矩阵留存。

### 经济性

- 完整与边际项目成本；
- 单位增量激活或留存参与者成本；
- 增量贡献与回本；
- 奖励兑换与负债；
- 自然行为替代、提前透支与蚕食。

### 风险与信任

- 作弊损失与阻止尝试率；
- 误判审核与申诉结果；
- 投诉、Spam、退订、退款与争议；
- 政策、隐私、公平与合规事故；
- 支持与运营负担。

## 诊断地图

| 症状 | 优先检查的问题 | 第一份证据 |
| --- | --- | --- |
| 总响应很高但提升很小 | 多少行为本来就会自然发生？ | 随机 Holdout 与基线比较 |
| 奖励停止后活动崩溃 | 用户是否到达真实产品或网络价值？ | 激励后 Cohort 与价值到达分析 |
| 成本增长快于增长 | 边际分群是否更弱，或奖励是否被竞争者推高？ | 分群级边际经济性 |
| 新产品增长但产品矩阵价值下降 | 项目是否只在移动现有用户？ | 客户级跨产品 Holdout |
| 推荐增长但投诉增加 | 发送者动机是否损害接收者价值或信任？ | 接收者质量、Spam 与投诉分析 |
| Marketplace 交易量增长但服务变差 | 补贴是否吸引了错误一侧、时段或地域？ | 本地流动性、取消、利用率与质量 |
| 风控阻止有价值用户 | 风险阈值是否经过校准并允许申诉？ | 按分群审核误判 |
| Engagement 增长但客户结果不变 | 游戏机制是否在奖励 Proxy 行为？ | 行动到价值与长期结果分析 |
| 客户等待促销 | 折扣是否替代了支付意愿？ | 购买时间、全价转化与 Cohort Margin |

## 建设顺序

1. 定义目标行为应创造的客户或网络价值。
2. 识别行为障碍，并先测试非激励解释。
3. 选择最小合格分群和一个主要目标行为。
4. 定义自然基线、Holdout、完整成本、风险上限和激励后窗口。
5. 选择与产品价值和参与者动机一致的奖励。
6. 扩大前建设资格、曝光、履约、归因和作弊埋点。
7. 在预算上限内试点，并预先定义递减与停止规则。
8. 复盘增量激活、留存价值、经济性、作弊、蚕食、公平与信任。
9. 只扩大边际留存价值为正的分群，下线或重构其余部分。

## 常见错误

- 在诊断行为障碍前选择奖励。
- 把兑换、行程、邀请或 Engagement 当成增量价值。
- 奖励无法预测客户价值的上游行为。
- 忽略激励后行为与 Margin。
- 把虚拟货币、容量或折扣视为免费。
- 缺少身份与作弊控制就上线可转让奖励。
- 优化 Marketplace 一侧，却伤害另一侧。
- 缺少公平、解释与申诉控制就个性化资格。
- 用游戏化最大化脱离产品价值的注意力。
- 默认让临时补贴变成长期商业模式。

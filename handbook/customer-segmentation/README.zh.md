# 客户分群与 ICP

> 客户分群只在差异应该改变决策时组织客户。ICP（理想客户画像）识别客户最有可能获得、持续使用并在经济上支撑目标价值的可观察条件。

**[English](README.md)｜[简体中文](README.zh.md)**

## 定义与边界

| 概念 | 用途 | 边界 |
| --- | --- | --- |
| 市场 | 广泛需求空间、品类或替代方案集合 | 市场过于宽泛，无法直接规定一条产品或增长路径。 |
| 分群 | 在问题、价值条件、行为、经济性或决策需求上存在实质差异的客户 | 分群必须能够改变产品、定位、渠道、销售、服务或商业化决策。 |
| ICP | 与强价值实现和价值获取相关的可观察客户条件 | 它是带版本的运营假设，不是理想 Logo 或特征清单。 |
| Persona | 一个分群或账户中的角色、动机、工作流或决策情境 | Persona 不能证明账户匹配、市场吸引力或经济性。 |
| Cohort（同期群） | 按开始时间、曝光、行为或产品状态分组衡量 | Cohort 用于诊断表现，不会自动成为长期分群。 |
| 生命周期状态 | 新增、激活、留存、风险或扩张等可变化状态 | 状态特定处理不等于市场分群。 |
| Propensity Score | 对转化、流失、购买或响应可能性的模型估计 | 预测不能解释客户价值，也不能证明处理的因果效果。 |

人口属性和企业属性可以帮助识别客户，但很少能单独解释匹配。有效分群需要把可观察差异连接到不同客户任务、价值路径、约束、支付意愿、购买流程、留存或经济性。

## 决策优先的分群

从决策开始，而不是从数据开始。

    决策
    -> 可能改变决策的客户差异
    -> 可观察分群规则
    -> 分群证据
    -> 差异化行动
    -> 下游结果与护栏

常见决策包括：

- 优先处理哪个客户问题与产品路径；
- 使用哪种定位、证明或品类；
- 推荐哪种新用户引导、模板或工作流；
- 使用哪个渠道、销售动作、合作伙伴或市场；
- 提供哪种套餐、Price Metric、服务级别或合同；
- 辅助、扩张、留存或排除哪些客户。

如果一个群体不能改变行动，它只是分析切片，不是运营分群。

## 分群单位与层级

比较群体前先选择单位。

| 层级 | 示例 | 核心问题 |
| --- | --- | --- |
| 市场 | 协作工作软件 | 存在哪些广泛需求与替代方案？ |
| 分群 | 协调重复工作流的运营团队 | 哪些问题与条件存在实质差异？ |
| 账户或家庭 | 公司、Workspace、团队、家庭或本地网络 | 价值在哪里被采用、付费或治理？ |
| 角色或参与者 | 用户、管理员、买方、审批者、供给方、创作者、学习者 | 谁获得价值、阻碍采用或作出决策？ |
| 使用场景或任务 | 活动规划、费用控制、语言学习 | 客户想取得什么进展？ |
| 生命周期状态 | 新增、激活、留存、休眠、扩张 | 当前适合什么处理？ |
| Cohort | 五月通过合作渠道开始使用 | 哪种曝光或起始条件支持有效比较？ |

不能混合用户、账户、交易、市场与角色分母。B2B（企业对企业）产品可以在永远不会采用、续约或扩张的账户中拥有高活跃用户。

## 分群维度

| 维度 | 问题 | 证据 |
| --- | --- | --- |
| 问题与任务 | 客户想完成什么进展？紧迫性与频率如何？ | 访谈、替代行为、工作流观察、搜索与销售语言 |
| 当前替代方案 | 如果产品不存在，客户会怎么做？ | Win/Loss、切换、表格、内部流程、服务、竞品、不行动 |
| 获得价值的能力 | 客户是否具备数据、权限、协作者、集成、频率或成熟度？ | 激活深度、实施、Time to Value、前提、支持需求 |
| 行为 | 哪些动作、频率、广度或深度说明重复价值？ | 产品事件、Cohort、留存、扩张、协作、功能采用 |
| 价值与经济性 | 创造并获取了多少价值？ | 支付意愿、转化、留存、毛利、服务成本、LTV（客户生命周期价值）、回本 |
| 购买流程 | 谁发现、评估、批准、使用、治理并续约？ | 销售电话、采购、安全审核、角色地图、周期长度 |
| 进入市场方式 | 哪个渠道和动作能可持续触达并服务分群？ | 渠道转化、销售效率、留存 CAC（客户获取成本）、合作伙伴匹配、地区 |
| 环境 | 哪个市场、设备、语言、监管、基础设施或运营条件改变采用？ | 本地留存、设备能力、支付可用性、合规、竞争环境 |
| 风险与约束 | 哪些信任、技术、组织或切换条件改变价值？ | 安全、迁移成本、数据敏感度、依赖、支持与失败模式 |

不能因为 Dashboard 可以拆分就创建分群。只有差异能够改变决策，并且证据与样本足以评估时才应拆分。

## 分群定义规范

每个运营分群都需要可复现定义。

    分群名称与用途：
    它改变的决策：
    分析单位：
    客户任务与替代方案：
    资格条件：
    可观察识别规则：
    价值机制：
    必要产品或客户条件：
    角色与购买结构：
    进入市场方式：
    经济假设：
    排除与边界情况：
    证据与反证：
    负责人、版本与复盘日期：

使用决策发生时可以获得的标准。通过未来留存定义的分群可以支持分析，但如果没有经过验证的早期代理，就不能直接路由新客户。

## ICP Contract

ICP 是针对特定产品、动作与时期的最高优先级分群假设。它应该回答：

1. **需求：**存在什么重要问题或任务？
2. **匹配：**客户在什么条件下能够获得差异化价值？
3. **行为：**哪些可观察动作说明价值正在重复发生？
4. **经济性：**公司能否可持续获客、服务、留存并扩张客户？
5. **可触达性：**公司能否稳定识别并触达更多相似客户？
6. **准备度：**客户是否具备采用所需的权限、数据、工作流、紧迫性与能力？
7. **排除：**哪些看起来相似的客户会反复失败、流失或需要特殊服务？

ICP 不是永久定义。产品、市场、渠道、价格、竞争、客户成熟度或服务容量变化时，最佳匹配条件也会变化。

## 证据账本

使用证据账本，而不是一个综合 ICP 分数。

| 判断 | 支持证据 | 反面证据 | 状态 |
| --- | --- | --- | --- |
| 分群存在紧迫问题 | 重复客户语言、高成本替代、快速开始行动 | 只有好奇，没有实施 | 已验证、推断或缺失 |
| 产品创造差异化价值 | 更好结果、留存使用、愿意切换 | 通用功能兴趣、依赖大量人工救援 | 已验证、推断或缺失 |
| 分群能够激活并留存 | 成熟 Cohort、重复工作流、续约、广度或深度增长 | 使用集中、依赖激励、高流失 | 已验证、推断或缺失 |
| 购买流程可行 | 明确 Champion、权限、采购路径、可接受周期 | 无负责人、安全审核受阻、预算不清 | 已验证、推断或缺失 |
| 进入市场方式可持续 | 留存转化、销售效率、回本、可触达人群 | 服务成本高、渠道饱和、周期无法支撑 | 已验证、推断或缺失 |
| 经济性支持优先级 | 贡献、扩张、服务能力、低失败成本 | 折扣、负毛利、支持负担、集中度 | 已验证、推断或缺失 |

定性解释与行为、留存、收入和成本证据一致时，ICP 才更可靠。知名客户、大合同或热情访谈不能单独证明可复制匹配。

## 定性发现

访谈不同结果的客户，而不只覆盖倡导者：

- 留存并扩张的客户；
- 刚刚激活的客户；
- 设置或实施失败的客户；
- 流失与收缩客户；
- 未转化的合格潜在客户；
- 同一账户中的买方、用户、管理员和阻碍者；
- 需要特殊人工服务的客户。

记录问题触发、替代方案、工作流、价值、异议、准备度、购买流程、失败条件、语言，以及客户选择或拒绝产品的原因。

把证据聚合成少量假设。不能把每项访谈差异都变成分群。

## 行为与留存证据

行为用于验证一个分群是否实现价值。

    合格分群
    -> 到达合格首次价值
    -> 重复承载价值的行为
    -> 留存或续约
    -> 在无不可接受成本或伤害时扩张

比较：

- 激活与 Time to Value；
- 承载价值的频率、广度、深度与协作；
- 用户、账户、收入、供给或需求留存；
- 续约、扩张、收缩与召回；
- 支持、错误、失败与放弃；
- 比率与绝对留存贡献；
- 成熟度相同的 Cohort 与一致定义。

避免只用需要预测的成功行为定义分群。这会产生 Leakage，并让规则无法用于前置决策。

## 价值与经济分群

看起来相似的客户可能获得不同价值、拥有不同购买方式，或产生不同服务成本。

有用差异包括：

- 问题重要性与结果大小；
- 支付意愿与预算来源；
- 偏好的套餐与 Price Metric；
- 实施、推理、履约或支持成本；
- 销售周期、采购、安全与法律负担；
- 留存、扩张、毛利、回本与现金特征；
- 集中度、风险与战略依赖。

价值分群应影响产品范围、套餐、服务与进入市场方式，不能成为不透明或歧视性定价的借口。

## 账户、角色与购买委员会

B2B 匹配同时存在于账户条件与参与角色中。

| 角色 | 核心问题 |
| --- | --- |
| 终端用户 | 工作流是否创造重复个人价值？ |
| Champion | 是否有人愿意投入精力与组织影响力推动采用？ |
| 管理者 | 产品是否改善团队结果与可见性？ |
| 经济买方 | 价值是否足够大且可信，值得投入预算？ |
| 管理员或安全团队 | 产品能否被治理、集成并信任？ |
| 采购或法务 | 商务、隐私与合同条件能否被接受？ |

不能把用户称为 ICP。一个账户可能高度匹配，但某个角色并不匹配；活跃用户也可能存在于永远不会购买或续约的账户里。

## 消费产品与 Marketplace 分群

消费产品通常使用“最佳匹配分群”而不是 ICP，但逻辑相同：识别与重复价值及可行路径相关的条件。

当差异能够改变处理时，可以按用户意图、使用场景、频率、市场、设备、产品状态、支付意愿或价值行为分群。

Marketplace 至少需要考虑：

- 参与方；
- Atomic Geography 或品类；
- 供给与需求质量；
- 交易频率与紧迫性；
- 流动性与匹配时间；
- Multi-homing 与参与者经济性；
- 监管、信任与运营负担。

公司级平均值可能同时隐藏一方的强分群与另一方破损的本地网络。

## 动态状态与 Propensity Model

一些有用群体会快速变化：新增、激活、留存、风险、高意图、Sales-ready 或可能扩张。

状态与 Propensity Model 应定义：

- 预测目标与决策；
- 合格人群与观察窗口；
- 处理前可获得的特征；
- 校准、Precision、Recall 与分群误差；
- 处理容量与预期增量价值；
- Drift、反馈循环、Override 与公平性；
- 用于验证的后续已观察结果。

高转化 Propensity 可能只识别原本就会购买的客户。当决策是“谁应该接受处理”时，需要实验或因果设计。

## 分群发现与验证

1. 定义决策与单位。
2. 收集定性、行为、经济、购买与进入市场证据。
3. 使用可观察规则形成少量分群假设。
4. 检查样本量、缺失、选择、成熟度与重叠。
5. 比较首次价值、留存、经济性与失败模式。
6. 在每项假设中访谈支持和反对结论的客户。
7. 测试差异化行动是否改善下游结果。
8. 根据决策价值保留、合并、拆分或退役分群。

探索性聚类可以发现模式，但统计分离不会自动让群体具有战略意义。

## 何时创建不同路径

当不同分群需要实质不同的以下内容时，才适合建立不同产品或进入市场路径：

- 客户任务、设置、数据、协作者或 Time to Value；
- 定位、证明、品类或异议；
- 渠道、销售协助、采购或服务；
- 套餐、Price Metric、支付意愿或支持级别；
- 安全、合规、本地化、集成或设备要求。

如果答案不能改变下一步，就不要增加 Onboarding 问题、套餐、销售交接或内容版本。

## 路径架构

优先使用最少且有用的路径。

    一个共享默认路径
    + 有证据支持时的条件化引导
    + 面向高复杂度或高价值情况的辅助路径
    + 无法交付价值时的明确排除或 Waitlist

每条路径都需要入口规则、客户承诺、负责人、体验、衡量窗口、退出规则与 Fallback。当差异不再足以支持运营复杂度时，应重新合并路径。

## 指标体系

| 层级 | 指标 |
| --- | --- |
| 触达 | 可服务账户或用户、可识别比例、合格流量、目标账户覆盖 |
| 价值入口 | 合格激活、Time to Value、设置成功、实施完成 |
| 重复价值 | Cohort 留存、有效频率、广度、深度、协作、续约 |
| 商业匹配 | 转化、支付意愿、ARPA（每账户平均收入）或 ACV（年度合同价值）、扩张、毛利、服务成本 |
| 分发匹配 | 留存 CAC、回本、销售周期、渠道集中度、合作伙伴贡献 |
| 分类质量 | 覆盖率、Precision、Recall、未知比例、稳定性、Drift、人工 Override |
| 护栏 | 支持负担、投诉、排除伤害、隐私、公平、合规、集中风险 |

同时报告比率与绝对数量。小分群可能拥有很好比率，却无法支撑目标业务规模；大分群也可能产生收入，却隐藏薄弱价值与高服务成本。

## 诊断地图

| 观察到的模式 | 下一步检查 | 可能的处理 |
| --- | --- | --- |
| Lead 量高，激活低 | 意图、问题严重度、来源、资格、承诺 | 收窄资格或改变价值路径。 |
| 用户活跃强，账户续约弱 | 买方价值、Champion、管理员、账户采用、工作流广度 | 增加账户级证据与角色特定路径。 |
| ACV 高，贡献弱 | 实施、支持、折扣、销售周期、失败成本 | 重新定价、限制服务或降低分群优先级。 |
| 一个分群只在聚合层表现强 | Cohort 成熟度、集中度、异常值、分母 | 读取分布与绝对贡献。 |
| 个性化引导增加摩擦 | 问题用途、路径差异、未知处理 | 删除不能改变体验的问题。 |
| 模型分数上升，但处理 Lift 没有提升 | 选择、校准、因果效果、容量 | 分开 Propensity 与 Treatment Effect。 |
| 国际分群表现弱 | 本地问题、支付、设备、语言、监管、渠道 | 调整 Offer 与路径，而不只是翻译。 |
| 分群表现恶化 | 产品、市场、价格、竞争、Mix、定义漂移 | 重新验证或退役分群与 ICP 版本。 |

这些模式用于指导调查，不能单独建立因果关系。

## 成熟度模型

| 阶段 | 特征 | 下一项约束 |
| --- | --- | --- |
| 1. 描述性群体 | 按人口、企业、渠道或客户清单切分 | 把群体连接到具体决策与价值假设。 |
| 2. 证据支持的假设 | 使用任务、条件、行为与经济性定义少量候选分群 | 建立可复现规则并比较成熟结果。 |
| 3. 差异化路径 | 产品或 GTM（市场进入策略）处理随已验证分群变化 | 衡量增量收益与运营复杂度。 |
| 4. 受治理的运营系统 | 定义、负责人、版本、数据与复盘共享 | 校准模型并管理重叠、公平与 Drift。 |
| 5. 自适应组合 | 分群与 ICP 随产品、市场、容量与战略变化 | 在演进时保持可比性与客户理解。 |

成熟度不取决于分群、Persona、Dashboard 或模型特征数量。能够改变决策的小型系统，强于没有团队使用的复杂分类体系。

## 建设顺序

1. 选择一项重要产品、GTM、商业化或服务决策。
2. 定义市场、单位、角色结构、客户任务与当前替代方案。
3. 写出少量可观察分群假设与明确排除条件。
4. 建立分群定义规范与 ICP Contract，指定负责人和版本。
5. 覆盖成功、失败、流失与未转化客户收集定性证据。
6. 对比成熟度一致的激活、留存、扩张、经济与失败证据。
7. 只使用决策时可获得的信息验证识别规则。
8. 使用下游护栏，对比一条差异化路径与共享默认路径。
9. 记录决策、不确定性、反证、运营成本与复盘日期。
10. 产品、价格、渠道、市场、客户 Mix、容量或监管变化时重新验证。

## 分群复盘

周期性复盘应该回答：

- 每个分群改变哪项决策？
- 定义是否可观察、可复现并能在决策时获得？
- 哪些客户未知、重叠或被错误分类？
- 激活、留存、经济性与定性证据在哪里冲突？
- 差异化路径是否创造增量下游价值？
- 分群带来哪些复杂度、排除、隐私或公平成本？
- 哪个分群或 ICP 应该收窄、合并、扩展或退役？

## 隐私、公平与治理

只收集合法决策所需的必要属性。敏感属性与近似代理需要法律、隐私、公平与伦理审查。

治理：

- 目的与合法基础；
- 数据最小化与保留周期；
- 权限、安全与审计；
- 代理歧视与差异影响；
- 在重要决策中提供客户解释与申诉；
- 模型 Drift、Override 与事件响应；
- 分群负责人、版本与退役。

分群应提高相关性与价值，不能隐藏排除或利用客户脆弱性。

## 常见错误

- 只按公司规模与行业定义 ICP。
- 把当前客户名单当成未来最佳分群的证明。
- 把每一个分析 Cohort 都叫作市场分群。
- 使用 Persona 代替行为与经济证据。
- 选择最大市场，而不是最重视差异化价值的客户。
- 使用决策时不可获得的未来结果定义分群。
- 联系所有活跃免费用户，而不是使用产品与账户信号。
- 在证据与运营不能支持前创建大量路径。
- 忽略流失、实施失败或需要特殊人工服务的客户。
- 在没有必要性、治理或公平审查时使用敏感属性或代理。

## 证据与局限

- 分群表现取决于定义、成熟窗口、样本量、选择、缺失数据与变化中的客户 Mix。
- 产品、渠道、价格、市场、容量或竞争变化后，历史匹配不保证未来匹配。
- 公开公司材料通常缺少完整分群规则、Confusion Matrix、Cohort、经济性、实验与失败路径。
- 敏感属性需要隐私、公平、法律与伦理审查，只能使用合法决策所需的最少信息。
- 分群是一种决策模型，不代表同一群体中的客户完全相同。

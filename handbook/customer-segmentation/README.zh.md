# 客户分群与 ICP

> 客户分群把市场或客户划分为需要不同决策的群体。理想客户画像（ICP）识别最有可能获得、持续使用并支撑目标价值的客户。

**[English](README.md)｜[简体中文](README.zh.md)**

## 定义与边界

| 概念 | 用途 | 边界 |
| --- | --- | --- |
| 市场分群 | 按需求、条件、行为、价值或经济性的实质差异组织客户 | 分群必须能够改变产品、定位、渠道、销售、服务或商业化决策 |
| ICP | 描述产品能够交付并获取强价值的客户条件 | 它是基于证据的运营假设，不是理想客户 Logo 清单 |
| Persona | 表示一个分群或账户中的角色、动机、工作流与决策情境 | Persona 不能证明市场吸引力或账户匹配 |
| Cohort | 按开始时间、曝光、行为或产品状态分组衡量 | Cohort 用于诊断表现，不会自动成为长期市场分群 |

人口属性和企业属性可以帮助定位客户，但很少能单独解释匹配。有效分群需要把可观察差异连接到不同客户任务、价值路径、约束、支付意愿、购买流程、留存或经济性。

## 分群维度

| 维度 | 问题 | 证据 |
| --- | --- | --- |
| 问题与任务 | 客户想完成什么结果？紧迫程度如何？ | 访谈、替代行为、工作流观察、搜索与销售语言 |
| 当前替代方案 | 如果产品不存在，客户会怎么做？ | Win/Loss、切换、表格、内部流程、服务、直接竞品或不行动 |
| 获得价值的能力 | 客户是否具备数据、权限、协作者、集成、频率或运营成熟度？ | 激活深度、实施、Time to Value、使用前提、支持需求 |
| 行为 | 哪些动作、频率、广度或深度预示重复价值？ | 产品事件、Cohort、留存、扩张、协作、功能采用 |
| 价值与经济性 | 创造并获取了多少价值？ | 支付意愿、转化、留存、毛利、服务成本、LTV 与回本 |
| 购买流程 | 谁发现、评估、批准、使用并续约？ | 销售电话、采购、安全审核、角色地图、销售周期 |
| 进入市场方式 | 哪种渠道和模式能可持续触达并服务该分群？ | 渠道转化、销售效率、留存 CAC、合作伙伴匹配、地区 |
| 风险与约束 | 哪些监管、信任、技术、组织或切换条件影响采用？ | 安全要求、合规、迁移成本、数据敏感度、依赖关系 |

不能因为 Dashboard 可以拆分就创建分群。只有差异能够改变决策，并且证据与样本足以评估时才应拆分。

## ICP 证据体系

ICP 应回答五个问题：

1. **需求：**存在什么重要问题或任务？
2. **匹配：**客户在什么条件下能够获得差异化价值？
3. **行为：**哪些可观察动作说明价值正在重复发生？
4. **经济性：**公司能否可持续获客、服务、留存并扩张客户？
5. **可触达性：**公司能否稳定识别并触达更多相似客户？

使用证据账本，而不是一个综合评分：

| 判断 | 支持证据 | 反面证据 | 状态 |
| --- | --- | --- | --- |
| 分群存在紧迫问题 | 重复客户语言、替代行为、快速开始关键动作 | 只有好奇，没有实施 | 已验证、推断或缺失 |
| 产品创造差异化价值 | 更快或更好的结果、留存使用、愿意切换 | 只对通用功能感兴趣、依赖大量人工救援 | 已验证、推断或缺失 |
| 分群能够留存和扩张 | 成熟 Cohort、重复使用、续约、广度或深度增长 | 使用高度集中、依赖激励、高流失 | 已验证、推断或缺失 |
| 进入市场方式可持续 | 留存转化、销售效率、回本、可触达人群 | 服务成本高、渠道饱和、销售周期无法支撑 | 已验证、推断或缺失 |

定性解释与行为、留存、收入和成本证据一致时，ICP 才更可靠。知名客户、高合同金额或热情访谈都不能单独证明可复制匹配。

## 分群流程

1. 定义本次决策：产品路径、定位、获客、销售覆盖、定价、服务或扩张。
2. 从客户任务、替代方案和获得价值所需条件开始。
3. 使用可观察标准形成少量分群假设。
4. 按分群比较激活、Time to Value、留存、商业化、服务成本和获客质量。
5. 访谈支持结论、反对结论、已流失和未转化的客户。
6. 选择最佳客户分群，并明确记录排除或次要分群。
7. 只有分群差异足够实质时才改变体验或运营决策。
8. 产品、市场、价格、渠道或客户组合变化时重新验证。

## 何时创建不同路径

当不同分群需要实质不同的以下内容时，才适合建立不同产品或进入市场路径：

- 客户任务、设置、数据、协作者或 Time to Value；
- 定位、证明、品类或异议处理；
- 渠道、销售协助、采购或服务；
- 套餐、计价单位、支付意愿或支持级别；
- 安全、合规、本地化或集成要求。

如果答案不能改变下一步，就不要增加 Onboarding 问题、套餐、销售交接或内容版本。只增加摩擦却不能改善下游价值的分群只是分类工作，不是增长。

## 指标体系

| 层级 | 指标 |
| --- | --- |
| 触达 | 可服务账户或用户、可识别比例、合格流量、目标账户覆盖 |
| 价值入口 | 合格激活、Time to Value、设置成功、实施完成 |
| 重复价值 | Cohort 留存、有效频率、广度、深度、协作、续约 |
| 商业匹配 | 转化、支付意愿、ARPA 或 ACV、扩张、毛利、服务成本 |
| 分发匹配 | 留存 CAC、回本、销售周期、渠道集中度、合作伙伴贡献 |
| 护栏 | 支持负担、投诉、排除伤害、隐私、公平、合规、集中风险 |

同时报告比率和绝对数量。小分群可能拥有很好比率，却无法支撑目标业务规模；大分群也可能产生收入，却隐藏弱价值和高服务成本。

## 常见错误

- 只按公司规模和行业定义 ICP。
- 把当前客户名单当成未来最佳分群的证明。
- 把每一个分析 Cohort 都叫作市场分群。
- 使用 Persona 代替行为与经济性证据。
- 选择最大市场，而不是最重视差异化价值的客户。
- 联系所有活跃免费用户，而不是使用产品与账户信号。
- 在样本、运营和证据尚不能支持前创建大量路径。
- 忽略流失、实施失败或需要特殊人工服务的客户。

## 证据与局限

- 分群表现取决于定义、成熟窗口、样本量、选择偏差和不断变化的客户组合。
- 产品、渠道、价格、市场或竞争变化后，历史匹配不保证未来匹配。
- 敏感属性需要隐私、公平、法律与伦理审查，只能收集合法决策所需的最少信息。
- 分群是一种决策模型，不代表同一群体中的客户完全相同。

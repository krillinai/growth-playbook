# 激活

> 激活是新用户已经获得有意义产品价值的第一项可靠证据。它是需要通过后续价值验证的行为假设，而不是完成注册或新手引导。

**[English](README.md)｜[简体中文](README.zh.md)**

## 定义与边界

激活回答：**新用户是否已经获得足够的价值，从而更有可能继续使用产品？** 有效定义需要包含可观察的价值事件、明确的时间窗口，以及能够验证该事件的下游结果。

| 概念 | 衡量对象 | 为什么不能单独代表激活 |
| --- | --- | --- |
| 注册 | 创建账户或进入产品 | 获得访问权限不等于获得价值。 |
| 完成新手引导 | 走完产品设计的流程 | 用户可能完成界面操作，却没有得到有用结果。 |
| 参与度 | 活动频率或数量 | 活动可能很浅，或者与价值无关。 |
| 留存 | 一段时间内持续获得价值 | 留存能够验证激活，但发生得更晚。 |

不存在通用激活事件。事件和窗口取决于客户任务、产品频率、用户角色与价值交付方式。

## 激活模型

| 阶段 | 核心问题 | 证据 | 常见失败 |
| --- | --- | --- | --- |
| **1. 用户意图** | 用户为什么来到这里？ | 来源、用途、角色、导入的上下文 | 不同意图进入同一条通用路径。 |
| **2. 首次价值** | 用户是否完成有意义的任务？ | 成功发生的价值事件 | 完成设置取代了客户结果。 |
| **3. 价值强化** | 用户是否加深、重复或分享价值？ | 重复使用、更丰富成果、协作 | 第一次成功没有创造回访理由。 |
| **4. 留存验证** | 该事件能否预测持续价值？ | Cohort 留存、扩展、重复任务 | 方便统计的事件被误认为因果节点。 |

产品不需要设计四个独立界面，但衡量系统需要区分意图、首次成功、价值强化和下游验证。

## 指标树

```text
持续客户价值
└── 留存激活
    ├── 激活结果：激活率、首次价值时间
    ├── 过程诊断：步骤转化、放弃节点、步骤间耗时
    ├── 激活质量：深度、广度、重复价值、协作
    └── 护栏：错误、支持成本、信任、留存、经济性
```

- **结果指标**说明用户是否到达候选价值节点。
- **过程指标**定位摩擦，但不能证明价值。
- **质量指标**区分表面完成与有意义的成功。
- **护栏指标**防止局部转化提升损害持续结果。

## 指标字典

每个指标都必须记录合格用户、起算事件、价值事件、窗口、排除条件、分群与负责人。

| 指标 | 定义与公式 | 窗口与有效拆分 | 解读方式 |
| --- | --- | --- | --- |
| **激活率** | `W 时间内完成价值事件的用户数 / Cohort 中符合条件的新用户数` | 会话、24 小时、7 天；来源、意图、角色、设备 | 核心结果；不能悄悄排除遭遇摩擦的用户。 |
| **首次价值时间** | `首次价值事件时间 - 激活起算时间`；报告中位数与分位数 | 与激活一致；意图、来源、设备 | 只有事件代表真实价值时，更快才有意义。 |
| **步骤转化率** | `完成步骤 n 的用户数 / 符合步骤 n 条件的用户数` | 会话与 Cohort；路径、设备、实验版本 | 定位摩擦；流程完成率高不代表激活强。 |
| **步骤放弃率** | `未进入 n+1 的用户数 / 到达步骤 n 的用户数` | 会话超时加回访窗口 | 说明在哪里停止，不能单独说明原因。 |
| **留存激活率** | `R 周期重复价值的激活用户数 / 符合回访条件的激活用户数` | 符合自然日、周或月频率 | 检查激活质量与重复价值。 |
| **激活带来的留存差值** | `激活用户留存率 - 可比未激活用户留存率` | 固定留存周期；在 Cohort 内比较 | 是预测证据，不是因果证明；仍有选择效应。 |
| **价值深度 / 广度** | 首次成功后获得价值的程度或范围 | 早期窗口与下游 Cohort | 更多活动不一定代表更多价值。 |
| **错误率** | `相关任务失败次数 / 总尝试次数` | 步骤、平台、集成、地区、版本 | 必须说明如何计算重试。 |
| **支持成本** | `激活相关咨询次数 / 符合条件的新用户数` | Cohort 与发布周期；问题类型 | 依赖过多人工协助的提升可能无法规模化。 |

### 计算示例

如果 1,000 个符合条件的新工作区中，有 420 个在 7 天内完成价值事件：

```text
7 日激活率 = 420 / 1,000 = 42%
```

如果其中 210 个在第四周重复价值事件：

```text
第 4 周留存激活率 = 210 / 420 = 50%
```

这些数字只是计算示例，不是行业基准。只有定义稳定，并确认价值事件能够预测持续结果时，它们才有意义。

## 分群与 Cohort 分析

整体激活率可能因为用户构成变化而变化。只比较成熟度相同的 Cohort，并只按能够改变决策的维度拆分。

| 维度 | 诊断问题 |
| --- | --- |
| 来源或 Campaign | 获客承诺是否与产品体验一致？ |
| 意图或用途 | 不同任务是否被迫走同一条路径？ |
| 角色或账户类型 | 用户是否具备所需的数据、权限或协作者？ |
| 设备、平台或版本 | 摩擦来自体验还是技术问题？ |
| 市场、语言或地区 | 内容、集成和价值假设能否迁移？ |
| 新用户、受邀用户、回访用户 | 不同起始上下文是否被混在一起？ |

同时报告比率和绝对人数。不能把已经完整观察 7 天的 Cohort 与昨天刚进入的用户比较。

## 激活问题诊断

| 观察模式 | 下一步检查 | 可能的行动 |
| --- | --- | --- |
| 注册多，第一步启动率低 | 按来源与意图看 Cohort、入口行为、访谈 | 对齐承诺、预填上下文或按意图分流。 |
| 引导完成率高，价值事件完成率低 | 对比流程与留存用户的成功路径 | 删除设置，把产生价值的操作提前。 |
| 首次价值不错，但耗时很长 | 按步骤、角色、平台与错误查看分位数 | 使用模板、渐进设置、默认值或异步准备。 |
| 激活率高，留存激活率低 | 按激活深度看留存与重复任务 | 重定义价值阈值并强化重复价值。 |
| 整体强，关键分群弱 | 检查来源、意图、角色、市场与设备构成 | 只有需求确实不同时才建立独立路径。 |
| 转化提升，但错误或咨询增加 | 错误、咨询、取消与信任信号 | 回滚、限制覆盖或在护栏下重做。 |

观察模式只能缩小调查范围，不能直接证明根因。

## 实验与验证方法

1. 写明客户任务与候选价值事件。
2. 固定合格条件、分子、分母、起算事件、窗口、排除条件、分群和护栏。
3. 用成熟 Cohort 检查该事件是否先于更强留存或重复价值。
4. 结合过程数据与定性证据定位主要约束。
5. 测试一项与诊断直接相关的机制。
6. 同时读取激活、质量、留存激活、错误、支持、信任与经济性。
7. 随产品与获客结构变化持续重新验证。

实验可以估计干预效果，但不能挽救薄弱的价值定义。观察关系可以用于预测，却不一定具有因果性。

## 操作定义

```text
用户与意图：
客户任务：
合格用户：
激活起算事件：
首次价值事件：
观察窗口：
价值强化信号：
留存验证周期：
主要分群：
质量指标：
护栏指标：
决策负责人：
现有证据与置信度：
```

当客户任务或价值路径存在实质差异时，保留不同激活定义。产品、目标客户、获客结构或自然使用频率变化时，需要重新审查定义。

## 漏斗

案例证据：[Airtable、Sean Ellis 与 Brian Balfour](../../case-library/themes/activation/README.zh.md#概念证据地图)。

关于状态设计、绝对贡献、约束类型、Cohort 成熟度与决策规则，参见[漏斗与约束模型](../funnel-and-constraint/README.zh.md)。

漏斗是符合条件的用户在明确状态之间移动的诊断模型。每一步都需要资格规则、事件、分母、窗口与决策负责人。

```text
合格进入 -> 首次行动 -> 价值事件 -> 留存价值
```

步骤转化与端到端转化需要同时报告。中间步骤提升可能只是把摩擦转移到下游。

## 新用户引导

案例证据：[Airtable、Facebook / Meta、Naomi Gleit 与 Lauryn Isford](../../case-library/themes/activation/README.zh.md#概念证据地图)。

Onboarding 应缩短用户意图与首次价值之间的距离。它可以使用模板、默认值、渐进设置、导入上下文、教育或人工协助，但完成引导不是结果。

只有意图、角色、前置条件或价值路径实质变化时才分群，并通过激活质量和留存行为验证每条路径。

## 转化率优化

案例证据：[Airtable 实验与 Sean Ellis 的价值路径](../../case-library/themes/activation/README.zh.md#概念证据地图)。

CRO 改善明确转化步骤，但不能损害下游价值。先从约束和机制开始，而不是从表面变化开始。需要同时读取转化、留存激活、错误、支持、信任与经济性。

`合格转化率 = 到达下游价值状态的用户 / 进入测试步骤的合格用户`

## 生命周期 Reach 与 Next Feature Fallacy

如果能接触新功能的人太少，它就无法实质改变增长。位于“参与墙”之后的功能要求用户先投入、配置、创作或协作，新用户和非活跃用户可能永远无法体验。

```text
预期绝对增量 = 合格 Reach x 预期 Impact x 置信度
```

需要判断约束位于落地页、新用户引导、首次价值、召回，还是留存用户体验内部。高 Reach 的早期环节即使相对提升较小，也可能比低使用功能的大幅提升带来更多下游用户。

## Setup、Aha 与 Habit

| 阶段 | 目的 | 失败信号 |
| --- | --- | --- |
| Setup | 收集价值所需的最少输入、配置、内容或连接 | 用户完成注册，却无法获得有效结果 |
| Aha | 交付一次可以识别的核心价值 | 用户完成设置，却不理解产品为何重要 |
| Habit | 按产品自然节奏建立可信的重复价值路径 | 出现初始价值，但 Cohort 不再回来 |

不能独立优化每个阶段。删除 Setup 可能提高完成率却削弱 Aha；过早强迫投入可能伤害信任；没有重复价值时，提醒不能创造 Habit。

## 入口状态与激活起点

激活不一定从注册开始。应从用户第一次真正有机会追求价值的时点定义起点。

| 入口状态 | 对激活的影响 |
| --- | --- |
| 自主进入的新用户 | 需要理解产品并创建足够上下文以获得首次价值 |
| 受邀协作者 | 已经带着成果、团队、权限与社交上下文进入 |
| 回访但未激活用户 | 可能保留账户状态，却仍未完成价值事件 |
| 导入或迁移账户 | Setup 可能部分完成，但数据质量和工作流准备度不确定 |
| 销售辅助客户 | 预期、配置与实施可能在进入产品前开始 |
| Marketplace 参与者 | 激活依赖另一侧、本地流动性、信任与交易成功 |

资格和前置条件不同时，不能把这些状态放进同一个分母。受邀协作者看似激活更快，可能是因为另一位用户已经完成大量 Setup。

## 空状态、模板与渐进 Setup

灵活产品经常在价值前让用户面对空白状态。激活设计应提供最小有用上下文，而不是强迫过早配置。

| 模式 | 适用情况 | 主要风险 |
| --- | --- | --- |
| 模板 | 常见任务具有可识别起始结构 | 限制特殊用途或创建无关内容 |
| 示例数据 | 客户数据就绪前可以演示价值 | Demo 价值无法迁移到真实工作 |
| 导入 | 已有数据或成果加快连续性 | 映射、权限、错误与等待时间 |
| 默认值 | 广泛安全选择减少决策负担 | 默认值对重要分群不合适 |
| 渐进 Setup | 需求可以推迟到真正需要时 | 后续缺少设置形成隐藏故障 |
| 引导清单 | 用户需要理解多个前置条件 | 完成 Checklist 取代价值 |
| 人工协助 | 复杂度、风险或实施需要专业能力 | 高成本、交付不一致与自助学习薄弱 |

衡量这些模式是否改善首次价值质量和留存价值，而不只是减少点击。

## 首次价值质量

二元激活会掩盖结果是否有用。按照客户任务定义质量维度。

| 维度 | 示例问题 |
| --- | --- |
| 成功 | 任务是否无错误完成？ |
| 相关性 | 结果是否匹配用户意图与上下文？ |
| 完整性 | 是否完成了足够任务，能够产生用途？ |
| 信任 | 用户能否理解、验证并安全依赖结果？ |
| 投入 | 价值是否匹配时间、数据与认知工作？ |
| 可重复 | 是否存在再次获得价值的可信理由与路径？ |
| 协作 | 更多参与者是否加深客户结果？ |

适当时使用分层定义：`技术成功 -> 可用结果 -> 有意义价值 -> 留存价值`，避免把技术上完成的行动当成客户成功。

## 不同产品结构的激活

### 消费与习惯产品

激活应交付可识别首次结果，并建立下一次自然机会。只有发生第一次有意义行动后，进度、提醒与身份才可能强化价值。

### B2B 与协作产品

区分用户、Workspace、账户与付款人激活。用户可以激活而账户仍未配置；账户可以签约而很少用户获得价值。有效信号包括首次工作流、成果成功、协作者价值、团队重复使用、管理员准备度与购买者证明。

### Marketplace 与网络

激活可能要求与另一位参与者成功互动。衡量供给或内容可用性、匹配质量、匹配时间、交易完成、信任与双方回访。没有接收者价值时，注册或发布供给只是 Setup。

### AI 产品

AI 激活应围绕成功任务或工作流，而不是 Prompt 数、Token 或生成产物。纳入结果质量、验证、编辑、重试、延迟、成本、信任，以及客户能否重复任务。更高使用可能代表价值，也可能代表反复失败。

### 低频与阶段性产品

下一次价值机会可能在数周或数月后。应使用任务完成、准备度、已保存状态或机会型追踪，而不是强迫日或周习惯模型。

## 辅助式激活

销售、实施、客户成功、社区或支持可以帮助客户到达首次价值。需要记录谁完成每个步骤，以及客户之后可以独立重复什么。

```text
人工辅助 Setup
-> 客户获得首次价值
-> 责任与知识转移
-> 客户或团队重复工作流
-> 留存账户价值
```

当价值、风险、数据集成或组织变化能够支持成本时，辅助式激活是合理的。定制劳动掩盖产品缺口、每个客户需要不同工作流，或首次价值无法在没有永久干预时持续，则说明存在问题。

跟踪实施时间、人工工时、阻塞依赖、客户参与、首次价值、独立重复价值、支持成本、Margin 与账户留存。

## 按周期读取激活实验

在三个周期读取实验结果：

| 周期 | 核心问题 | 示例指标 |
| --- | --- | --- |
| 即时 | 干预是否减少诊断出的障碍？ | 步骤转化、错误、时间、理解 |
| 首次价值 | 是否有更多合格用户获得有意义价值？ | 激活率、TTV、质量、支持 |
| 下游 | 首次价值是否改善重复价值？ | 留存激活、账户采用、收入或信任护栏 |

更短流程可以改善即时转化，却降低结果质量或准备度。下游读取应保留原始分配与分母。

## 成熟度模型

| 阶段 | 特征 | 下一项约束 |
| --- | --- | --- |
| 1. 界面完成 | 把注册和引导完成当成激活 | 定义客户任务、价值事件、窗口与质量 |
| 2. 价值事件衡量 | 已有产品特定激活与 TTV | 验证留存与分群差异 |
| 3. 路径优化 | 意图路径、模板、诊断与实验改善首次价值 | 增加账户、协作、信任与成本视角 |
| 4. 自适应激活 | 按意图、角色、状态或上下文路由，并受治理自动化 | 控制偏差、复杂度、隐私与因果有效性 |
| 5. 价值交付系统 | 获客承诺、引导、产品、辅助、留存与商业化共享价值模型 | 防止指标膨胀并保留客户判断 |

成熟度应跟随真实路径差异。只有一个简单任务的产品可能只需要一个清晰激活事件，而不需要自适应 Onboarding。

## 建设顺序

1. 定义客户任务、入口状态、合格人群与第一项可信价值。
2. 绘制前置条件、产品步骤、人工工作、错误、等待与依赖。
3. 定义激活事件、质量、窗口、分母与下游验证。
4. 按意图、来源、角色、设备、市场与入口状态比较成熟度相同的 Cohort。
5. 结合过程数据与客户证据识别最大约束。
6. 测试一种机制，例如模板、默认值、渐进 Setup、导入、连接或辅助。
7. 共同读取即时转化、首次价值质量、留存激活、支持、信任与经济性。
8. 只有需求和价值路径确实不同，才增加差异化路径。
9. 产品、受众、获客承诺或自然频率变化时重新验证定义。

## 常见错误

- 复制其他产品的 Aha Moment、阈值或时间窗口。
- 把注册、完成向导或发出邀请当成已经获得价值。
- 优化激活，却忽略留存激活、错误、信任、支持成本或经济性。
- 把激活和留存之间的相关关系当成因果证明。
- 增加无法改变决策或产品路径的分群。
- 把受邀、自主进入、回访和销售辅助用户混在一个分母中。
- 在客户任务成功前，把生成产物、发布供给、导入数据或邀请协作者当成价值。
- 删除结果质量或后续留存所需的 Setup。
- 用永久人工协助掩盖无法重复交付价值的产品。

# 激活

> 激活是新用户已经获得有意义产品价值的第一项可靠证据。它是需要通过后续价值验证的行为假设，而不是完成注册或新手引导。

**[English](README.md)｜[简体中文](README.zh.md)**

## 定义与边界

激活回答：**新用户是否已经获得足够的价值，从而更有可能继续使用产品？** 有效定义需要包含可观察的价值事件、明确的时间窗口，以及能够验证该事件的下游结果。

| 概念 | 衡量对象 | 为什么不能单独代表激活 |
| --- | --- | --- |
| 注册 | 创建账户或进入产品 | 获得访问权限不等于获得价值。 |
| 完成新手引导 | 走完产品设计的流程 | 用户可能完成界面操作，却没有得到有用结果。 |
| 参与度 | 活动频率或数量 | 活动可能很浅，或者与价值无关。 |
| 留存 | 一段时间内持续获得价值 | 留存能够验证激活，但发生得更晚。 |

不存在通用激活事件。事件和窗口取决于客户任务、产品频率、用户角色与价值交付方式。

## 激活模型

| 阶段 | 核心问题 | 证据 | 常见失败 |
| --- | --- | --- | --- |
| **1. 用户意图** | 用户为什么来到这里？ | 来源、用途、角色、导入的上下文 | 不同意图进入同一条通用路径。 |
| **2. 首次价值** | 用户是否完成有意义的任务？ | 成功发生的价值事件 | 完成设置取代了客户结果。 |
| **3. 价值强化** | 用户是否加深、重复或分享价值？ | 重复使用、更丰富成果、协作 | 第一次成功没有创造回访理由。 |
| **4. 留存验证** | 该事件能否预测持续价值？ | Cohort 留存、扩展、重复任务 | 方便统计的事件被误认为因果节点。 |

产品不需要设计四个独立界面，但衡量系统需要区分意图、首次成功、价值强化和下游验证。

## 指标树

```text
持续客户价值
└── 留存激活
    ├── 激活结果：激活率、首次价值时间
    ├── 过程诊断：步骤转化、放弃节点、步骤间耗时
    ├── 激活质量：深度、广度、重复价值、协作
    └── 护栏：错误、支持成本、信任、留存、经济性
```

- **结果指标**说明用户是否到达候选价值节点。
- **过程指标**定位摩擦，但不能证明价值。
- **质量指标**区分表面完成与有意义的成功。
- **护栏指标**防止局部转化提升损害持续结果。

## 指标字典

每个指标都必须记录合格用户、起算事件、价值事件、窗口、排除条件、分群与负责人。

| 指标 | 定义与公式 | 窗口与有效拆分 | 解读方式 |
| --- | --- | --- | --- |
| **激活率** | `W 时间内完成价值事件的用户数 / Cohort 中符合条件的新用户数` | 会话、24 小时、7 天；来源、意图、角色、设备 | 核心结果；不能悄悄排除遭遇摩擦的用户。 |
| **首次价值时间** | `首次价值事件时间 - 激活起算时间`；报告中位数与分位数 | 与激活一致；意图、来源、设备 | 只有事件代表真实价值时，更快才有意义。 |
| **步骤转化率** | `完成步骤 n 的用户数 / 符合步骤 n 条件的用户数` | 会话与 Cohort；路径、设备、实验版本 | 定位摩擦；流程完成率高不代表激活强。 |
| **步骤放弃率** | `未进入 n+1 的用户数 / 到达步骤 n 的用户数` | 会话超时加回访窗口 | 说明在哪里停止，不能单独说明原因。 |
| **留存激活率** | `R 周期重复价值的激活用户数 / 符合回访条件的激活用户数` | 符合自然日、周或月频率 | 检查激活质量与重复价值。 |
| **激活带来的留存差值** | `激活用户留存率 - 可比未激活用户留存率` | 固定留存周期；在 Cohort 内比较 | 是预测证据，不是因果证明；仍有选择效应。 |
| **价值深度 / 广度** | 首次成功后获得价值的程度或范围 | 早期窗口与下游 Cohort | 更多活动不一定代表更多价值。 |
| **错误率** | `相关任务失败次数 / 总尝试次数` | 步骤、平台、集成、地区、版本 | 必须说明如何计算重试。 |
| **支持成本** | `激活相关咨询次数 / 符合条件的新用户数` | Cohort 与发布周期；问题类型 | 依赖过多人工协助的提升可能无法规模化。 |

### 计算示例

如果 1,000 个符合条件的新工作区中，有 420 个在 7 天内完成价值事件：

```text
7 日激活率 = 420 / 1,000 = 42%
```

如果其中 210 个在第四周重复价值事件：

```text
第 4 周留存激活率 = 210 / 420 = 50%
```

这些数字只是计算示例，不是行业基准。只有定义稳定，并确认价值事件能够预测持续结果时，它们才有意义。

## 分群与 Cohort 分析

整体激活率可能因为用户构成变化而变化。只比较成熟度相同的 Cohort，并只按能够改变决策的维度拆分。

| 维度 | 诊断问题 |
| --- | --- |
| 来源或 Campaign | 获客承诺是否与产品体验一致？ |
| 意图或用途 | 不同任务是否被迫走同一条路径？ |
| 角色或账户类型 | 用户是否具备所需的数据、权限或协作者？ |
| 设备、平台或版本 | 摩擦来自体验还是技术问题？ |
| 市场、语言或地区 | 内容、集成和价值假设能否迁移？ |
| 新用户、受邀用户、回访用户 | 不同起始上下文是否被混在一起？ |

同时报告比率和绝对人数。不能把已经完整观察 7 天的 Cohort 与昨天刚进入的用户比较。

## 激活问题诊断

| 观察模式 | 下一步检查 | 可能的行动 |
| --- | --- | --- |
| 注册多，第一步启动率低 | 按来源与意图看 Cohort、入口行为、访谈 | 对齐承诺、预填上下文或按意图分流。 |
| 引导完成率高，价值事件完成率低 | 对比流程与留存用户的成功路径 | 删除设置，把产生价值的操作提前。 |
| 首次价值不错，但耗时很长 | 按步骤、角色、平台与错误查看分位数 | 使用模板、渐进设置、默认值或异步准备。 |
| 激活率高，留存激活率低 | 按激活深度看留存与重复任务 | 重定义价值阈值并强化重复价值。 |
| 整体强，关键分群弱 | 检查来源、意图、角色、市场与设备构成 | 只有需求确实不同时才建立独立路径。 |
| 转化提升，但错误或咨询增加 | 错误、咨询、取消与信任信号 | 回滚、限制覆盖或在护栏下重做。 |

观察模式只能缩小调查范围，不能直接证明根因。

## 实验与验证方法

1. 写明客户任务与候选价值事件。
2. 固定合格条件、分子、分母、起算事件、窗口、排除条件、分群和护栏。
3. 用成熟 Cohort 检查该事件是否先于更强留存或重复价值。
4. 结合过程数据与定性证据定位主要约束。
5. 测试一项与诊断直接相关的机制。
6. 同时读取激活、质量、留存激活、错误、支持、信任与经济性。
7. 随产品与获客结构变化持续重新验证。

实验可以估计干预效果，但不能挽救薄弱的价值定义。观察关系可以用于预测，却不一定具有因果性。

## 操作定义

```text
用户与意图：
客户任务：
合格用户：
激活起算事件：
首次价值事件：
观察窗口：
价值强化信号：
留存验证周期：
主要分群：
质量指标：
护栏指标：
决策负责人：
现有证据与置信度：
```

当客户任务或价值路径存在实质差异时，保留不同激活定义。产品、目标客户、获客结构或自然使用频率变化时，需要重新审查定义。

## 漏斗

漏斗是符合条件的用户在明确状态之间移动的诊断模型。每一步都需要资格规则、事件、分母、窗口与决策负责人。

```text
合格进入 -> 首次行动 -> 价值事件 -> 留存价值
```

步骤转化与端到端转化需要同时报告。中间步骤提升可能只是把摩擦转移到下游。

## 新用户引导

Onboarding 应缩短用户意图与首次价值之间的距离。它可以使用模板、默认值、渐进设置、导入上下文、教育或人工协助，但完成引导不是结果。

只有意图、角色、前置条件或价值路径实质变化时才分群，并通过激活质量和留存行为验证每条路径。

## 转化率优化

CRO 改善明确转化步骤，但不能损害下游价值。先从约束和机制开始，而不是从表面变化开始。需要同时读取转化、留存激活、错误、支持、信任与经济性。

`合格转化率 = 到达下游价值状态的用户 / 进入测试步骤的合格用户`

## 常见错误

- 复制其他产品的 Aha Moment、阈值或时间窗口。
- 把注册、完成向导或发出邀请当成已经获得价值。
- 优化激活，却忽略留存激活、错误、信任、支持成本或经济性。
- 把激活和留存之间的相关关系当成因果证明。
- 增加无法改变决策或产品路径的分群。

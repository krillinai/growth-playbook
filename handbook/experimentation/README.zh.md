# 实验系统

> 实验是降低重要不确定性的决策系统。可信实验项目需要连接因果设计、可靠数据、实际阈值、客户价值与组织行动。

**[English](README.md)｜[简体中文](README.zh.md)**

## 定义与边界

在线受控实验把合格单元随机分配到不同条件，并在明确假设下估计干预效应。A/B Testing 是最常见的简单形式。

实验不能替代 Product-Market Fit、客户研究、战略、工程判断或伦理。它最适合可表示为可逆干预、曝光能够控制、结果能在有用窗口成熟、干扰能够管理的决策。

## 决策模型

```text
决策
-> 重要不确定性
-> 因果机制
-> 实验或证据设计
-> 可信观察
-> 行动
-> 可复用学习
```

发布前定义正向、负向、有害和不确定结果分别会触发什么行动。没有决策负责人或行动规则的实验只能产生分析，不能产生学习。

## 什么时候应该实验

适合使用受控实验的条件：

- 存在多个可信选项；
- 干预合法、合乎伦理且足够可逆；
- 分配与曝光能够控制；
- 合格人群能够支持所需敏感度；
- 结果能在适合决策的周期内成熟；
- 溢出能够预防、建模或通过设计处理。

严重缺陷本应直接修复、变化不合法或不合伦理、流量长期不足、曝光无法隔离、决策不可逆或结果成熟过慢时，不应默认使用 A/B Test。应使用其他证据并保留不确定性。

## 实验定义

```text
决策：
假设与机制：
合格人群：
随机化单位：
分配比例：
曝光规则：
控制组与处理组：
Overall Evaluation Criterion：
主要结果：
诊断指标：
护栏指标：
最小有意义效应：
Power 与样本假设：
观察与成熟窗口：
排除条件与身份规则：
干扰风险：
分析计划：
决策规则：
负责人：
```

读取结果前冻结重要定义。曝光开始后的修改必须记录，除非分析计划明确允许，否则只能作为探索性分析。

## OEC 与指标体系

Overall Evaluation Criterion（OEC）表示组织真正希望改善的结果，需要连接短期敏感度与持续客户、业务价值。

| 指标角色 | 用途 | 常见风险 |
| --- | --- | --- |
| 主要结果 | 决定预期机制是否成立 | 看到结果后选择指标 |
| 诊断指标 | 解释变化在哪里或如何发生 | 把解释指标当成最终结果 |
| 护栏 | 发现质量、信任、可靠性、毛利或生态伤害 | 主要指标提高就忽略伤害 |
| 不变量 | 理论上不应随处理变化，用于发现缺陷 | 不变量失效仍解释结果 |
| 长期结果 | 验证持续客户或业务价值 | 使用未经验证的短期代理指标 |

避免不可解释的综合评分。OEC 组合多个指标时，需要记录方向、权重、权衡与失败条件。

## Power、效应与不确定性

计算样本和周期前定义最小有意义效应。考虑基线比率或方差、分配、资格、聚类、预期排除、重复查看、成熟度与多重比较。

解释结果时需要同时考虑：

- 效应大小与置信区间；
- 区间是否包含实质收益或伤害；
- 实际价值、成本与可逆性；
- 统计 Power 与设计敏感度；
- 分群与生态一致性；
- 成熟度与长期风险。

统计显著不代表业务价值。不显著也不代表零效应或等价。

## 信任与实验质量

```text
资格
-> 分配
-> 曝光
-> 身份与排除
-> 事件记录
-> 指标计算
-> 分析
-> 决策
```

验证每一项转化。常用控制包括 A/A Test、确定性分配检查、不变量指标、自动 Scorecard、可复现查询、数据新鲜度检查，以及高影响决策的独立复核。

### Sample-Ratio Mismatch

Sample-Ratio Mismatch（SRM）表示实际分配与预期比例存在实质差异。原因可能包括分配缺陷、选择性曝光、资格差异、数据缺失、Bot、身份错误、事件延迟，或受处理影响的排除规则。

重大 SRM 得到解决或明确边界前，不应解释处理效应。

### Twyman's Law

异常有趣的结果往往是错的。巨大的意外效应需要先检查分母变化、日志、实现、异常值、重复分析、Novelty 与样本比例问题。

## 长期效应与代理指标

留存、收入、Marketplace 均衡或生态结果成熟很慢，因此短期代理指标很常见。需要验证代理变化能否在历史实验、分群和产品状态中预测长期结果。

考虑：

- Novelty 与 Primacy；
- 学习与客户适应；
- 延迟流失或支持负担；
- 季节性与重复曝光；
- 内容或供给响应；
- 网络与 Marketplace 均衡；
- 成本、毛利、信任与战略依赖。

重要决策可以使用更长 Hold、后续窗口、Surrogate Validation、持续控制组或重复实验。

## 干扰与随机化单位

参与者相互影响时，个人随机化可能失效。例如家庭、团队、买卖双方、社交网络、城市、广告、共享供给和时间依赖系统。

选择匹配处理交付和互动的单元：用户、账户、家庭、Cluster、地区、市场、卖家或时间块。Cluster 和 Switchback 可以减少部分溢出，但会降低有效样本并增加假设。

应明确报告干扰风险，而不是用标准 A/B 术语隐藏它。

## 替代证据设计

无法随机化时，使用决策允许的最强设计：

| 设计 | 适用条件 | 主要局限 |
| --- | --- | --- |
| 定性研究 | 机制和客户解释不清楚 | 单独不能估计总体效应 |
| 离线评估或模拟 | 曝光成本或风险高 | 可能无法复制真实行为 |
| 有边界 Pilot | 运营可行性不确定 | 选择与 Novelty 限制迁移 |
| Switchback | 处理作用于市场或时间依赖系统 | Carryover 与时间趋势 |
| 中断时间序列 | 明确干预影响一个系统 | 同期变化与趋势假设 |
| Difference-in-Differences | 存在可比较处理与未处理群体 | 平行趋势假设 |
| Regression Discontinuity | 分配由阈值决定 | 只估计阈值附近局部效应 |
| 匹配比较 | 无法随机化 | 仍存在未观察混杂 |

不能把观察证据重新命名为实验。明确写出假设与因果置信度。

## 决策规则

| 结果 | 可能行动 |
| --- | --- |
| 有意义收益且护栏通过 | 发布到已测试人群并监控成熟结果 |
| 出现伤害或护栏破坏 | 停止、回滚或限制曝光 |
| 方向有用但低于阈值 | 只有机制证据支持时才继续迭代 |
| 区间排除重大伤害，且变化降低成本或复杂度 | 可以考虑发布 Flat Result |
| Power 不足、结果未成熟或质量检查失败 | 继续收集证据，不分配效应状态 |
| 分群效应冲突 | 保留差异并测试有边界决策 |

预先定义规则，但假设失效时允许有责任人的判断。记录所有 Override。

## 实验运行系统

1. 维护共享指标与资格注册表。
2. 曝光前审查实验定义。
3. 自动检查分配、SRM、不变量、日志与成熟度。
4. 区分平台健康和产品解释。
5. 使用书面决策复盘，而不是只看结果 Dashboard。
6. 在可搜索知识库中保留失败、负向、Flat 与矛盾结果。
7. 衡量证据是否改变发布决策与后续模型。
8. 产品变化时重新验证代理指标、护栏与平台假设。

实验数量只是活动指标。更有价值的项目指标包括决策周期、确定结论比例、护栏事故、学习复用、有可信证据的发布比例与实际下游价值。

## 诊断

| 模式 | 检查内容 |
| --- | --- |
| 实验很多，结果变化很小 | 战略薄弱、Reach 低、效应小或没有后续行动 |
| 大多不确定 | Power、流量、方差、资格、埋点与效应阈值 |
| 主要指标提高，护栏受损 | OEC 错位或局部优化 |
| 团队不相信结果 | SRM、Scorecard 可复现性、数据质量与领导行为 |
| 意外提升很少复现 | Twyman's Law、Novelty、Peeking、多重检验与实现缺陷 |
| 结果很少发布 | 责任、决策规则、可逆性与实施成本 |
| 短期提升随后衰减 | 代理指标有效性、Novelty、延迟流失与均衡效应 |

## 常见错误

- 最大化实验数量而不是决策价值。
- 看到结果后选择人群、指标或分群。
- 没有有效 Sequential Design 就 Peeking 并提前停止。
- 测试大量结果却不处理 False Discovery。
- 忽略 SRM、曝光、身份和日志缺陷。
- 把显著当成价值，或把不显著当成没有效应。
- 没有留存价值和护栏证据就发布短期提升。
- 干扰占主导时仍随机化个人。
- 把结果推广到未测试人群、产品状态或时间周期。

## 证据与局限

- 统计方法依赖一些假设，这些假设可能在干扰、数据缺失、人群变化与重复决策中失效。
- 公司实验叙述很少公开完整分配、分析与长期结果数据。
- 阈值、样本量与 Win Rate 在产品、指标、方差、流量与决策背景不匹配时不能迁移。
- 高风险决策需要适当的统计、法律、伦理、隐私与领域审查。

# 留存

> 留存衡量一个 Cohort 是否在产品的自然使用周期内持续获得重复价值。

**[English](README.md)｜[简体中文](README.zh.md)**

## 留存模型

先定义重复价值事件和自然间隔，再选择日、周或月窗口。持续留存、召回、扩展、收缩与流失必须分别报告。

```text
起始 Cohort -> 留存价值 -> 价值强化 -> 持续留存
            -> 中断 -> 召回或流失
```

## 指标体系

| 指标 | 公式或定义 | 用途 |
| --- | --- | --- |
| Cohort 留存率 | `t 周期活跃用户数 / 原始合格 Cohort 用户数` | 判断回访形态与持续性 |
| 客户留存率 | `留存账户数 / 起始账户数` | 判断账户连续性 |
| 收入留存率 | `(起始收入 - 流失 - 收缩 + 扩展) / 起始收入` | 判断商业持续性 |
| 召回率 | `回访流失用户数 / 符合条件的流失用户数` | 单独衡量恢复的价值 |
| 价值频率 | 每位留存用户的价值事件数与间隔 | 判断深度与习惯质量 |

无界、固定窗口和滚动留存需要保持一致。曲线变平在低频、季节性和持续使用产品中含义不同。

## 问题诊断

| 模式 | 下一步检查 |
| --- | --- |
| 第一周期快速流失 | 激活质量、预期错位、早期失败 |
| 早期健康，后期下降 | 重复价值、内容或工作流耗尽 |
| DAU 增长，Cohort 不变 | 获客增长掩盖弱留存 |
| 召回高，持续留存弱 | 依赖提醒，或核心价值未解决 |
| 使用强，收入留存弱 | 包装、降级、付款人与用户错位 |

按起始 Cohort、激活路径、来源、意图、用途、账户规模、地区和产品状态拆分，不能混合成熟度不同的 Cohort。

## 操作方法

1. 定义重复价值事件和自然间隔。
2. 先建立 Cohort 曲线和增长核算，再增加策略。
3. 找到曲线第一次实质变化及受影响分群。
4. 结合行为路径、错误、取消和定性证据。
5. 测试核心价值、价值强化、恢复或生命周期触达。
6. 把退订、疲劳、信任、支持和经济性作为护栏。

## 用户生命周期

案例证据：[Facebook 增长核算与 Duolingo 召回](../../case-library/themes/retention/README.zh.md#概念证据地图)。

生命周期状态应该反映价值与行动变化，而不是任意 Campaign 标签：

```text
新增 -> 激活 -> 留存 -> 扩展
    -> 风险 -> 中断 -> 召回或流失
```

每个状态都需要进入与退出事件。用户可能非线性移动，账户生命周期也可能不同于个人用户生命周期。

## 生命周期营销

案例证据：[Duolingo 通知与字节跳动激励](../../case-library/themes/retention/README.zh.md#概念证据地图)。

生命周期营销使用消息、教育、产品界面、支持和 Offer 帮助用户到达或恢复价值。触达应由用户状态和可能需求触发，而不是固定群发日历。

衡量增量价值事件、留存行为，以及退订或投诉护栏。打开和点击是交付诊断，不是最终结果。

## 流失预防

案例证据：[Duolingo、Airtable 与 Brian Balfour](../../case-library/themes/retention/README.zh.md#概念证据地图)。

区分主动流失、被动流失、不活跃、收缩与切换。诊断原因是价值不足、产品失败、价格、需求变化、支付失败还是组织变化。

只有仍存在可信价值路径时才在风险前后干预。折扣和提醒可能延迟流失，却隐藏根本问题。

## 早期留存与使用频率

最陡峭的流失通常发生在最初几次使用或最初几天，但正确周期取决于产品节奏。需要分别诊断用户是否完成激活、使用场景自然重复频率、长期 Cohort 是否稳定，以及留存用户是否执行承载价值的行为。

增加通知通常无法修复薄弱的首次价值路径，也不能为没有重复任务的产品创造留存。选择干预前，应按意图、渠道、设备、市场和激活路径拆分早期曲线。

## 留存调整后的病毒传播

```text
长期病毒传播贡献
= X x Y x Z + X x Y x Z^2 + X x Y x Z^3 + ...
```

其中 `X` 是参与分享行为的比例，`Y` 是每次行为触达的人数，`Z` 是两次机会之间的留存。这一模型更支持有用的协作和内容分享，而不是强制邀请，并要求接收者完成激活和留存后才认为循环有效。

## 常见错误

- 用整体 DAU 或回访人数代替留存。
- 把召回算作连续留存。
- 修复重复价值之前先增加 Streak 或通知。
- 使用与自然频率冲突的任意窗口。

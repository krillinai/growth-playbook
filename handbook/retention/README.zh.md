# 留存

> 留存衡量一个明确 Cohort 是否在产品自然使用周期内持续获得重复价值。它是价值持续存在的行为证据，不只是用户回来或仍在付费。

**[English](README.md)｜[简体中文](README.zh.md)**

## 定义与边界

留存需要明确四项选择：

1. **实体：**用户、账户、Workspace、家庭、卖家、买家、订阅或收入。
2. **Cohort：**共同起始事件与资格规则。
3. **价值事件：**代表重复客户价值的行为或结果。
4. **周期：**产品自然产生下一次价值机会的时间。

```text
合格 Cohort
-> 到达重复价值
-> 在自然机会再次行动
-> 价值被强化或加深
-> 保持留存、扩张、中断、召回、收缩或流失
```

登录、打开 App、消息送达、账单状态和页面访问可以用于诊断，但只有可靠代表已交付价值时才属于留存。

留存不代表所有客户都应每日使用或永不离开。低频、季节性、项目型、Marketplace 和企业产品需要不同价值事件与观察窗口。

## 自然频率与机会

根据客户任务选择周期，而不是使用 Dashboard 默认值。

| 产品模式 | 可用周期 | 留存解释 |
| --- | --- | --- |
| 每日学习或沟通 | 日或周 | 用户是否完成有意义重复行动？ |
| 团队工作流或财务运营 | 周或月 | 账户是否继续运行工作流？ |
| 旅行或重大购买 | 季、年或机会 | 需求再次出现时客户是否回来？ |
| Marketplace 供给 | 有效可用窗口 | 相关需求出现时参与者是否回来？ |
| 季节性产品 | 可比较季节或事件 | 下一次有效机会时 Cohort 是否回来？ |

当客户行动机会不相等时，机会留存可能比日历留存更准确。

## Cohort 设计

### 起始事件

注册 Cohort 回答获客质量问题；激活 Cohort 回答已体验价值是否持续。对于重复价值开始较晚的产品，首次购买、项目、交易或协作 Cohort 可能更合适。

### Cohort 成熟度

不能在相同日历日期比较成熟度不同的 Cohort。使用 Cohort Age，并保留尚未完整成熟的右删失周期。

### 分群

按照以下维度拆分：

- 获客来源与 Campaign；
- 激活路径与 Time to Value；
- 用途、意图与到达价值；
- 产品、套餐、设备、市场与地域；
- 账户规模、角色与协作状态；
- 价格、折扣、激励与实验曝光；
- 支持、错误、可靠性与取消经历。

分群应帮助提出机制和决策，而不是制造数百个不稳定切片。

## 留存定义

| 定义 | 回答的问题 | 主要风险 |
| --- | --- | --- |
| 固定或有界留存 | 实体是否在指定周期活跃？ | 忽略窗口外的有效回访 |
| 滚动或无界留存 | 实体是否在当前或后续任一周期回来？ | 可能掩盖频率下降 |
| 经典 Cohort 留存 | 原始 Cohort 在年龄 `t` 仍活跃多少？ | 强依赖起点与价值事件 |
| 区间留存 | 实体是否在一段周期范围行动？ | 区间可能隐藏关键时机 |
| 机会留存 | 相关机会出现时实体是否行动？ | 机会识别可能复杂 |
| Survival | 距离中断或流失有多长？ | 需要清晰事件与删失规则 |

决策使用一个主要定义，并记录其他定义。不能在缺少明确标签时把滚动与有界留存放进同一趋势。

## 留存曲线

```text
Retention(t)
= Cohort Age t 完成价值事件的成员
/ 原始 Cohort 合格成员
```

解读形态：

| 形态 | 可能解释 | 下一步检查 |
| --- | --- | --- |
| 早期快速下降 | 激活弱、预期错位、Setup 摩擦或获客质量差 | 激活路径与首次价值失败 |
| 早期趋稳 | 某个留存分群可能存在重复价值 | 分群规模、质量与长期持续性 |
| 持续缓慢下降 | 价值耗尽、竞争、需求变化或节奏错位 | 用途、频率与定性流失原因 |
| 新 Cohort 改善 | 产品或受众质量可能提高 | 比较 Mix、季节性与实验曝光 |
| 总量增长但 Cohort 不变 | 获客掩盖留存没有改善 | 增长核算与渠道 Mix |
| 召回高但连续留存弱 | 产品依赖提醒、Campaign 或低频需求 | 区分自然节奏与未解决价值损失 |

曲线变平是持续 PMF 的证据，但不是证明。仍需考虑 Cohort 选择、市场条件、产品频率、幸存者偏差与观察长度。

## 增长核算

增长核算解释两个周期之间活跃基础如何变化。

```text
期末活跃
= 留存活跃
+ 新增活跃
+ 召回活跃
- 流失活跃
```

用互斥转化规则定义状态。

| 状态 | 定义 |
| --- | --- |
| 新增 | 本周期首次发生承载价值的活动 |
| 留存 | 上一有效周期与当前周期都活跃 |
| 召回 | 一个或多个周期不活跃后再次活跃 |
| 流失或中断 | 曾经活跃，但当前周期不活跃 |

可以计算新增、留存、召回和流失占上一周期或期末活跃用户的比例。绝对数量与比例应共同报告，避免小基数产生误导。

增长核算不能替代 Cohort 曲线。前者解释当前变动，后者解释不同起始群体的持续性。

## 用户、账户与收入留存

多用户、多付款人或多 Workspace 产品需要分别衡量。

| 层级 | 核心问题 | 示例 |
| --- | --- | --- |
| 用户留存 | 个人是否持续获得价值？ | 学习者练习或员工完成工作流 |
| 账户或 Logo 留存 | 客户组织是否保持活跃或合同？ | Workspace 继续使用产品 |
| 产品留存 | 账户是否保留某个产品或模块？ | 客户保留分析但停止消息模块 |
| 收入留存 | 经流失、收缩与扩张后收入是否持续？ | 订阅经变化后的收入 |

```text
GRR = (期初经常性收入 - 流失 - 收缩) / 期初经常性收入
NRR = (期初经常性收入 - 流失 - 收缩 + 扩张) / 期初经常性收入
```

合同、涨价或延迟取消可能让收入留存健康，同时用户价值下降。用户活动也可能很强，但因付款人价值、套餐或预算变化造成收入收缩。两者必须共同阅读。

## 重复价值模型

留存机制通常分为四类：

| 模式 | 重复价值 | 强化方式示例 |
| --- | --- | --- |
| 习惯 | 重复个人行动产生价值 | 进度、提醒、降低摩擦、身份 |
| 网络 | 其他参与者使回访更有价值 | 相关连接、流动性、内容、信誉 |
| 已存价值 | 数据、资产、历史、配置或学习累积 | 可搜索性、连续性、个性化、切换成本 |
| 工作流 | 产品进入重复团队或业务运营 | 协作、集成、审批、自动化 |

缺少持续价值的切换成本只能延迟流失，不能创造健康留存。强化机制应加深客户结果，而不是只增加退出摩擦。

## 激活到留存

激活是价值的早期表达，留存检验价值是否重复出现。

```text
客户意图
-> 完成 Setup
-> 首次价值事件
-> 重复价值事件
-> 稳定留存或扩张
```

按分群验证候选激活行为与后续留存，再测试改善该行为是否因果改变留存。与留存相关的行为可能只是在识别高意图用户。

共同跟踪激活完成、Time to Value、价值质量、下一次价值间隔与留存 Cohort 结果。

## 客户生命周期状态

生命周期状态应反映价值和行动变化，而不是任意 Campaign 标签。

```text
新增 -> 激活 -> 留存 -> 扩张
    -> 风险 -> 中断 -> 召回或流失
```

为每个状态定义进入、退出、最短持续时间与优先级。用户可能非线性移动，账户生命周期也可能不同于个人生命周期。

“风险”状态应对应可行动机制，例如价值频率下降、工作流未完成、协作减少、可靠性故障、支付问题或需求变化。没有干预路径的模型分数不是生命周期策略。

## 生命周期触达

生命周期触达通过产品界面、Email、Push、消息、教育、支持、销售和 Offer，帮助客户到达、重复或恢复价值。

```text
观察到的客户状态
-> 可能未满足需求或下一项价值
-> 合适渠道与时机
-> 客户行动与价值结果
-> 频控、疲劳与信任反馈
```

触达应由客户状态与可能需求触发，而不是固定群发日历。行动完成、不再相关、不安全或可能造成疲劳时应抑制消息。

衡量增量价值事件、留存行为，以及退订、投诉、支持与信任护栏。打开和点击只是交付诊断，不是最终结果。

## 流失分类

| 流失类型 | 示例 | 第一响应 |
| --- | --- | --- |
| 价值流失 | 产品不再解决任务 | 客户研究与产品修正 |
| 激活失败 | 客户从未到达重复价值 | 修复 Setup、预期与首次价值路径 |
| 可靠性流失 | 错误、延迟、数据丢失或交付失败 | 恢复服务并修复信任 |
| 需求流失 | 项目结束、季节变化或需求消失 | 区分自然中断与可预防流失 |
| 竞争流失 | 替代方案提供更好价值或匹配 | 理解决策标准与切换路径 |
| 价格流失 | 价值价格关系变弱 | 分群支付意愿、套餐与实现价值 |
| 被动流失 | 支付失败、卡过期、采购问题 | 恢复流程与账单运营 |
| 组织流失 | Champion 离开、团队变化、预算或政策改变 | 多线程采用与账户级价值 |
| 收缩 | Seat、用量、产品范围或收入下降 | 识别丢失用途与付款人用户错位 |

取消原因表单有用但存在偏差，需要结合行为、支持、账单、可靠性、销售与定性访谈。

## 流失风险与干预

风险信号可能包括价值频率下降、Setup 未完成、协作减少、内容或供给质量下降、重复错误、支持升级、容量闲置、支付失败或 Champion 离开。

预测不能证明为什么会流失，也不能证明干预有效。需要验证：

- 按分群与周期的校准；
- 距离流失的提前量；
- 误判与漏判成本；
- 信号是否可行动；
- 干预的增量效应；
- 公平、隐私与客户信任边界。

不能对所有高风险客户统一使用折扣、消息或销售触达。让干预匹配可能机制，并保留 Holdout。

## 召回与 Win-Back

召回是在有效中断后重新获得价值，不是连续留存。

```text
曾经留存
-> 在定义周期内不活跃
-> 可信价值机会再次出现
-> 客户重新进入并获得价值
-> 再次留存或中断
```

区分自然低频回访、Campaign 驱动回访、支付恢复、产品改进与需求真正重新出现。不能只看回访事件，还要评估召回后的第二条留存曲线。

Win-back 激励和提醒可能产生临时再激活。衡量增量、成本、回访后留存、投诉，以及原始流失原因是否解决。

## 协作与扩张

当更多参与者改善核心工作流或已存价值时，协作会强化留存；协作也可能只产生噪声活动。

衡量协作者激活、共享成果成功、多用户工作流重复完成、账户级价值、Seat 质量与单一 Champion 依赖。扩张应跟随更广泛实现价值，而不是只看邀请或已配置 Seat。

价格与收入扩张参见[商业化](../monetization/README.zh.md)。

## 留存实验

留存结果成熟较慢且经常重复曝光。上线前明确：

```text
目标 Cohort 与价值事件：
自然周期与成熟窗口：
干预机制：
随机化单位与曝光：
主要留存结果：
早期诊断与激活 Proxy：
质量、信任、疲劳与收入护栏：
干扰与季节性：
最小实际效应：
决策与后续窗口：
```

短期 Proxy 可能反转时，使用更长追踪、持续 Holdout、重复 Cohort 分析或替代指标验证。如果长期价值、学习质量、协作、Margin 或信任可能下降，不能停在 Engagement 提升。

不能在 Treatment 分配后排除流失用户，也不能在看到结果后重新定义活跃。

## 留存调整后的增长循环

```text
长期传播贡献
= X x Y x Z + X x Y x Z^2 + X x Y x Z^3 + ...
```

`X` 是参与分享行为的比例，`Y` 是每次行为的接收者数量，`Z` 是两次机会之间的留存。该模型更支持有用协作和内容分享，而不是强制邀请，并要求接收者完成激活与留存后才认为循环成功。

## 指标体系

### 核心留存

- 有界、滚动或机会留存；
- 按 Age 与价值事件的 Cohort 曲线；
- 用户、账户、产品与 Logo 留存；
- 留存、新增、召回、流失与中断状态；
- Survival、中断时间与频率分布。

### 价值与质量

- 重复价值事件完成与质量；
- 价值事件间隔；
- 深度、协作、已存价值与工作流完成；
- 可靠性、支持、信任与客户结果；
- 激活到留存转化。

### 商业留存

- GRR 与 NRR；
- Logo 流失、收入流失、收缩与扩张；
- 续费、降级、支付恢复与回本；
- 付款人用户对齐与账户健康。

### 生命周期与干预

- 触达或 Offer 产生的增量留存价值；
- 召回与第二条留存曲线；
- 干预覆盖、资格、抑制与疲劳；
- 退订、投诉、支持与折扣依赖。

## 诊断地图

| 症状 | 优先检查的问题 | 第一份证据 |
| --- | --- | --- |
| 第一周期快速下降 | 用户是否到达并理解首次价值？ | 激活路径与早期 Cohort 对比 |
| 新 Cohort 变差 | 渠道、分群、产品、价格或预期 Mix 是否改变？ | Cohort 构成与发布时间线 |
| DAU 增长但 Cohort 曲线不变 | 获客是否掩盖重复价值薄弱？ | 按来源增长核算 |
| 通知提高回访但不提高价值 | 触达是否只产生打开，而非有意义行动？ | 增量价值事件与疲劳分析 |
| 召回高但反复中断 | 原始流失机制是否未解决？ | 召回后留存曲线 |
| 用户留存健康但 NRR 弱 | 账户是否收缩、降级或失去付款人价值？ | 账户与收入转化分析 |
| NRR 健康但使用下降 | 合同或涨价是否掩盖价值损失？ | 用户、账户、支持与续费领先指标 |
| 协作与留存相关 | 协作创造价值，还是只识别高意图账户？ | 实验或匹配路径分析 |
| 风险模型标记大量客户 | 分数是否校准，干预是否有增量？ | 按分数区间 Holdout 与可行动性 |

## 成熟度模型

| 阶段 | 特征 | 下一项约束 |
| --- | --- | --- |
| 1. 总量活动 | 只有 DAU/MAU 和回访用户，没有稳定 Cohort | 定义实体、Cohort、价值事件与周期 |
| 2. Cohort 可见 | 标准曲线与基础分群 | 增加增长核算、流失分类与质量 |
| 3. 生命周期运营 | 状态、触发干预、账户与收入视图 | 衡量增量、疲劳与跨层价值 |
| 4. 预测留存 | 风险模型、干预选择与长期实验 | 控制偏差、校准、因果与隐私 |
| 5. 价值系统 | 产品、生命周期、协作、收入与组织共享重复价值模型 | 防止指标复杂化并保留客户判断 |

成熟度应跟随决策需要。只有一个重复行动的简单产品可能不需要预测评分或复杂生命周期编排。

## 建设顺序

1. 定义实体、合格 Cohort、重复价值事件与自然周期。
2. 建设清晰 Cohort 曲线并验证身份、事件与成熟度规则。
3. 增加增长核算，区分连续留存与召回。
4. 按激活、意图、来源、用途、产品与账户语境分群。
5. 找到最大的曲线损失，并结合行为与定性证据。
6. 定义流失机制和最小产品或生命周期干预。
7. 测试增量留存价值，并使用质量、信任、疲劳与经济性护栏。
8. 只有基础证据可信后，才增加账户、收入、风险与自动化。
9. 随产品、客户或市场变化重新检查价值事件与周期。

## 常见错误

- 用整体 DAU、MAU 或回访用户代替留存。
- 不考虑产品节奏就选择日、周或月窗口。
- 混合注册 Cohort 与激活 Cohort。
- 比较成熟度不一致的 Cohort。
- 把召回算成连续留存。
- 把早期行为与留存相关性当成因果。
- 修复重复价值前先增加通知、Streak、折扣或销售触达。
- 优化用户留存却忽略账户、收入、质量或付款人结果。
- 建设无法对应机制或缺少干预 Holdout 的流失评分。
- 未经验证就用短期 Engagement Proxy 代表长期留存。

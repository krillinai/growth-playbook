# 漏斗与约束模型

> 漏斗表示合格客户如何在明确定义的状态之间移动。约束分析识别当前最限制目标客户与业务结果的因素。

**[English](README.md)｜[简体中文](README.zh.md)**

## 定义与边界

| 模型 | 主要用途 | 边界 |
| --- | --- | --- |
| 客户旅程 | 描述客户随时间变化的情境、任务、问题与互动 | 其中可能包含无法作为严格顺序衡量的行为 |
| 漏斗 | 衡量明确合格状态之间的转化与损失 | 它是诊断视角，不是增长的完整解释 |
| 状态模型 | 表示状态之间的移动、返回、跳过与重复 | 适用于非线性客户过程 |
| 增长模型 | 连接价值、获客、留存、商业化、扩张与再投资 | 其中可以包含多个漏斗、状态模型和循环 |
| 约束模型 | 选择当前最限制系统输出的因素 | 转化率最低的步骤不一定是主要约束 |

有效漏斗让定义和损失变得可见。有效约束诊断则解释为什么改变某个部分比改变其他部分更可能改善最终结果。

## 状态设计

每个状态都需要：

- 合格人群；
- 可观察的进入事件；
- 清晰的成功事件；
- 成熟或观察窗口；
- 排除条件与身份规则；
- 对客户价值的解释；
- 对应的决策负责人。

```text
合格需求
-> 合格进入
-> 首次价值
-> 重复价值
-> 留存客户
-> 商业化或扩张价值
```

不能因为界面中存在页面或按钮就增加步骤。只有转化代表有意义的客户或运营决策时才应该加入。

## 指标体系

| 指标 | 公式或定义 | 用途 |
| --- | --- | --- |
| 步骤转化 | `进入下一状态的用户 / 进入当前状态的合格用户` | 局部转化健康度 |
| 端到端转化 | `到达目标状态的用户 / 合格起始人群` | 系统总产出率 |
| 绝对损失 | `合格用户 - 到达下一状态的用户` | 转化对总量的贡献 |
| 状态停留时间 | 从进入到离开状态的时间 | 延迟、摩擦与容量诊断 |
| 成熟转化 | 所有合格单元完成观察窗口后的转化 | 避免不完整 Cohort 偏差 |
| 留存转化 | 到达下游重复价值的用户 / 合格起始人群 | 保护客户质量 |
| 经济产出 | 贡献或持续价值 / 合格起始人群 | 连接转化与业务价值 |

同时报告比率和绝对数量。小步骤提升 20% 的相对转化，可能不如影响大多数合格客户的 2% 提升产生更多总结果。

## 约束类型

| 约束 | 常见信号 | 需要检查的证据 |
| --- | --- | --- |
| 需求 | 进入系统的合格客户太少 | 可服务需求、触达、意图、渠道增量 |
| 承诺与资格 | 进入量高，但下游价值弱 | 来源、信息、用途、ICP、留存质量 |
| 产品价值 | 客户完成步骤，却没有获得或重复价值 | 访谈、价值事件、留存、替代方案 |
| 摩擦 | 合格客户在可避免步骤失败 | 错误、时间、放弃、可用性、支持联系 |
| 容量 | 需求超过人员、供给、系统或运营吞吐量 | 排队时间、可用性、利用率、服务水平 |
| 信任与风险 | 客户因后果或不确定性而犹豫 | 异议、权限、安全、投诉、撤销行为 |
| 留存 | 新客户进入速度高于价值留存速度 | 成熟 Cohort、流失、召回、自然频率 |
| 经济性 | 出现增长，但无法持续盈利 | 边际 CAC、回本、服务成本、毛利、激励依赖 |
| 学习 | 团队无法区分原因或做出决策 | 埋点、分配、定义、责任、决策周期 |

多个约束可以同时存在。在当前决策周期中选择一个主要约束，同时保留可信备选解释与护栏。

## 绝对贡献

按预期绝对变化确定机会优先级，而不是只看转化率：

```text
预期下游变化
= 合格数量
x 预期转化提升
x 下游保留比例
x 置信度
```

这个公式用于优先级判断，不是预测。选择工作前还要考虑实施成本、延迟、可逆性、运营容量、经济价值与风险。

## 分群与 Cohort

用户组合变化会改变总漏斗。比较成熟度相同的 Cohort，只在差异能够改变决策时拆分。

有用维度包括：

- 客户任务、ICP、角色或账户类型；
- 来源、承诺、Campaign 或合作伙伴；
- 产品版本、设备、平台或入口界面；
- 地区、语言、监管或市场成熟度；
- 新增、受邀、回访、召回或扩张客户；
- 价格、套餐、销售模式或实施路径。

在分群之外保留一个标准总视图。过度分群会隐藏真实系统贡献，并产生不稳定结论。

## 约束诊断

1. 定义最终客户与业务结果。
2. 只画出解释该结果所需的状态。
3. 固定合格条件、事件、窗口、排除规则与责任。
4. 对账步骤数量、端到端总量与成熟 Cohort。
5. 计算转化、绝对损失、时间、留存质量与经济性。
6. 在最可能产生重大损失的位置检查过程证据与客户解释。
7. 按绝对影响、证据、控制力、学习速度、成本与风险比较约束。
8. 选择一个主要约束并记录备选解释。
9. 使用决策规则与下游护栏测试机制。
10. 根据结果更新漏斗与增长模型。

模式只能缩小调查范围，不能证明根因。低转化可能是有意资格过滤；高转化也可能把低价值客户传递到下游；明显的界面问题可能次于弱需求或产品价值。

## 实验与决策设计

| 字段 | 要求 |
| --- | --- |
| 人群 | 合格状态入口与排除条件 |
| 机制 | 干预为什么会改变约束 |
| 主要结果 | 需要决策的转化或下游价值 |
| 成熟度 | 完成观察窗口后再做最终解释 |
| 护栏 | 留存、信任、错误、支持、毛利、风险 |
| 决策规则 | 发布、迭代、停止或继续收集证据 |

当分配与干扰条件允许时使用随机实验；否则可以使用有边界 Pilot、中断时间序列、匹配比较、Switchback、定性证据或运营衡量，并明确写出局限。

## 常见错误

- 把页面访问或 UI 步骤当成客户状态。
- 只看转化率，不看绝对贡献。
- 比较成熟窗口不同的 Cohort。
- 看到结果后修改分母。
- 删除保护资格、信任或安全的摩擦。
- 把步骤与留存的相关性当成因果证明。
- 为每个分群建立漏斗，却没有标准总视图。
- 在检查需求、价值、容量和经济性前把症状称为约束。
- 局部指标提高，但下游留存价值下降。

## 证据与局限

- 漏斗定义取决于产品与决策，不匹配合格条件和窗口时，Benchmark 很少能够直接迁移。
- 观察转化用于识别调查位置，不一定能说明损失原因。
- 小分群与稀有结果需要体现不确定性并保护隐私。
- 客户组合、产品、容量、渠道、价格或市场变化时，主要约束也会移动。

# 商业化

> 商业化把已经交付的客户价值连接到定价模型、付费决策和可持续单位经济性。

**[English](README.md)｜[简体中文](README.zh.md)**

## 商业化模型

```text
用户与付款人 -> 价值事件 -> 价值度量 -> 套餐 -> 价格
             -> 转化 -> 留存收入 -> 毛利与回本
```

用户、购买决策者和付款人可能不同。计价单位既要随价值变化，也要避免不可预测账单或损害使用的激励。

## 指标体系

| 指标 | 公式或定义 | 决策 |
| --- | --- | --- |
| 付费转化率 | `新增付费客户 / 符合条件的免费或试用客户` | 套餐能否转化合格价值？ |
| ARPA / ARPU | `收入 / 平均付费账户数或用户数` | 收入水平与结构 |
| 毛利率 | `(收入 - 直接服务成本) / 收入` | 交付经济性 |
| 贡献毛利 | 收入减可变服务、支持和获客成本 | 可扩张贡献 |
| 回本周期 | `获客成本 / 周期贡献毛利` | 收回获客投入所需时间 |
| 净收入留存率 | `(起始收入 - 流失 - 收缩 + 扩展) / 起始收入` | 收入留存与扩展 |

按 Cohort 和分群分析转化、折扣、降级、扩展、流失、服务成本与支付意愿。只有留存收入和毛利成熟后，定价提升才算完整。

## 问题诊断

| 模式 | 下一步调查 |
| --- | --- |
| 使用强，付费转化弱 | 付款人错位、付费价值不清、套餐设计 |
| 转化强，快速流失 | 过度承诺、重复价值弱、年付扭曲 |
| 收入增长，毛利下降 | 服务成本、折扣、支持成本 |
| 扩展高，收缩也高 | 计价单位波动或账户调整 |
| LTV 预测好，实际 Cohort 弱 | 模型假设、成熟度、生存偏差 |

## 操作方法

1. 绘制用户、购买者、付款人、价值事件、成本驱动和替代方案。
2. 选择客户能理解的价值度量和套餐边界。
3. 联合建模价格、转化、留存、扩展、成本与回本。
4. 先测试支付意愿和套餐，再优化结账界面。
5. 用成熟 Cohort 经济性和客户信任作为护栏。

## 定价

定价定义价值度量、套餐边界、价格水平、折扣规则与套餐迁移路径。支付意愿需要与留存使用、服务成本、竞争替代和购买风险一起评估。

## 收入模型

| 模型 | 适用条件 | 主要风险 |
| --- | --- | --- |
| 订阅 | 价值稳定重复 | 付费但没有重复价值会增加流失 |
| 用量计费 | 使用量随客户价值变化 | 账单和毛利可能不可预测 |
| 交易 / Take Rate | 平台促成可衡量交换 | 费率过高会削弱流动性 |
| 广告 | 注意力创造广告主价值 | 广告可能损害用户价值与信任 |
| 混合 | 多种价值流共存 | 复杂度掩盖经济性与选择 |

根据价值交付和成本结构选择，而不是追随品类流行。

## 盈利性增长

盈利性增长意味着增量 Cohort 在获客、服务、支持、激励、退款与风险成本后仍产生持续贡献。需要报告边际回本与成熟 Cohort 贡献，而不是只看收入增长。

`Cohort 贡献 = 留存收入 - 可变获客 - 服务 - 支持 - 激励 - 风险成本`

## 常见错误

- 把免费增值、试用、订阅或用量计费当成信仰。
- 用收入或签约额证明商业化健康。
- 忽略折扣、退款、服务成本和支持。
- 使用预测 LTV，却不与实际 Cohort 对账。

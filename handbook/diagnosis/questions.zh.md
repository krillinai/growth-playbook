# 增长诊断问题清单

**[English](questions.md)｜[简体中文](questions.zh.md)**

每次诊断都先完成核心信息收集，再填写与目标结果最匹配的专项路径。没有证据时回答**未知**，不能根据假设推断精确答案。如果整项列表都未知，只填写 `未知`。

## 产品背景

| 问题 | 预期答案格式 |
| --- | --- |
| 本次诊断包含哪个产品或产品界面？ | `产品类型；具体界面或工作流；交付的核心价值`，或 `未知` |
| 谁是主要用户、付款人和受益人？ | `用户：...；付款人：...；受益人：...；优先分群：...`，或 `未知` |
| 目标客户当前使用什么替代方案？ | `替代方案；选择原因；切换触发条件`，或 `未知` |
| 商业模式和价值指标是什么？ | `收入模式；价格或抽成率；价值指标；主要可变成本`，或 `未知` |
| 本次诊断包含哪个市场和产品阶段？ | `市场/地区；阶段：PMF 前 / 早期可重复 / 规模化 / 扩张 / 成熟优化`，或 `未知` |
| 产品的自然使用频率是什么？ | `预期频率；产生下一次需求的事件；证据来源`，或 `未知` |

## 目标结果

| 问题 | 预期答案格式 |
| --- | --- |
| 应改变哪一项客户或业务结果？ | `指标名称；精确定义；为何代表价值`，或 `未知` |
| 基线和目标是什么？ | `基线：数值/单位；目标：数值/单位；分群；衡量周期；来源`，或 `未知` |
| 决策周期是什么？ | `开始日期；决策日期；选择该周期的原因`，或 `未知` |
| 哪些护栏指标不能恶化？ | 每项填写 `护栏指标；当前值；不可接受阈值`，或 `未知` |

## 当前症状

| 问题 | 预期答案格式 |
| --- | --- |
| 哪项结果发生了变化，或没有按预期变化？ | `观察到的指标变化；预期变化；绝对与相对差距`，或 `未知` |
| 症状从何时开始，是否持续存在？ | `首次观察日期；持续时间；趋势或季节性模式`，或 `未知` |
| 哪些用户或环节受到影响？ | `Cohort/分群；产品界面；市场/渠道；占合格总体比例`，或 `未知` |
| 哪项比较能够证明这是一个问题？ | `对照 Cohort、历史周期、计划、Holdout 或基准；差距大小；可比性限制`，或 `未知` |

| 当前可行的执行路径是什么？ | `自助执行 / Growth & Marketing Skills / clawee.ai 企业实施；负责人；所需访问权限；依赖`，或 `未知` |

## 获客

从[获客知识模块](../acquisition/README.zh.md)和[获客案例](../../case-library/themes/acquisition/README.zh.md)开始。

### 症状与机制

获客症状是合格需求、触达、渠道规模或获客经济性发生变化。机制解释变化原因，例如受众饱和、承诺与渠道不匹配、素材供给下降、竞价压力、归因重叠或下游价值偏低。

### 诊断问题

| 问题 | 预期答案格式 |
| --- | --- |
| 缺口来自总量、合格流量还是留存需求？ | `每一层指标；基线；当前值；目标；分群/周期`，或 `未知` |
| 哪个渠道、受众、承诺、入口页面、市场或 Cohort（同期群）造成了大部分缺口？ | `维度；分群；对缺口的绝对贡献；比较对象；来源`，或 `未知` |
| 不同来源的边际 CAC（客户获取成本）、回本、激活和留存有何差异？ | 表格/列表：`来源；边际 CAC；回本；激活；留存；周期`，或 `未知` |
| 获客中有多少是增量，而不只是归因结果？ | `方法：Holdout/Lift/地域实验/模型；增量规模；不确定性；周期`，或 `未知` |
| 哪个机制能够产生观察到的分群模式？ | `候选机制；可观察预测；支持证据；矛盾证据`，或 `未知` |
| 哪项证据能够区分主要机制与替代解释？ | `证据/测试；可区分的竞争机制；决策规则；负责人/日期`，或 `未知` |

### 证据充分性检查

只有在合格需求缺口已经分群、下游价值和经济性已知，并且至少一个机制具有可证伪预测时，才能继续。否则回答 `证据不足：缺失证据；负责人；收集日期`，或 `未知`。

## 激活

从[激活知识模块](../activation/README.zh.md)和[激活案例](../../case-library/themes/activation/README.zh.md)开始。

### 症状与机制

激活症状是用户没有获得首次价值或完成合格的下一步行为。机制解释失败原因，例如意图不匹配、设置负担、价值不清晰、产品错误、信任摩擦、供给不足或反馈延迟。

### 诊断问题

| 问题 | 预期答案格式 |
| --- | --- |
| 哪种用户意图和资格规则定义激活 Cohort？ | `意图；资格规则；排除规则；Cohort 规模；来源`，或 `未知` |
| Setup、Aha 和 Habit 事件分别是什么？ | `阶段；事件定义；代表的价值；与留存关系的验证`，或 `未知` |
| 哪个转化环节损失的合格用户最多，或延迟最大？ | `起点/终点事件；完成率；用时分布；分群；周期`，或 `未知` |
| 症状是无法完成，还是完成后无法留存？ | `完成率；留存激活率；比较对象；分群/周期`，或 `未知` |
| 哪个机制与观察到的错误、行为和定性证据一致？ | `候选机制；行为信号；定性信号；受影响分群；矛盾证据`，或 `未知` |
| 哪项证据能够区分摩擦、弱意图和弱价值？ | `证据/测试；可区分的竞争机制；决策规则；负责人/日期`，或 `未知` |

### 证据充分性检查

只有在资格规则、首次价值、损失最大环节与留存激活均已定义，并有证据区分摩擦、意图和价值时，才能继续。否则回答 `证据不足：缺失证据；负责人；收集日期`，或 `未知`。

## 留存

从[留存知识模块](../retention/README.zh.md)和[留存案例](../../case-library/themes/retention/README.zh.md)开始。

### 症状与机制

留存症状是用户回访、重复价值、使用深度、召回或扩展偏弱。机制解释失败原因，例如重复价值消失、习惯形成失败、工作流中断、质量或可靠性下降、网络减弱或服务失败。

### 诊断问题

| 问题 | 预期答案格式 |
| --- | --- |
| 哪个时间间隔符合产品的自然使用周期？ | `时间间隔；重置周期的用户需求；证据；敏感性检查`，或 `未知` |
| 缺口发生在早期留存、长期留存、频率、深度、召回还是扩展？ | `留存组成；基线/当前值/目标；Cohort；周期`，或 `未知` |
| 哪种意图、激活路径、来源、市场或产品状态 Cohort 造成了缺口？ | `维度；Cohort；对缺口的绝对贡献；比较对象；来源`，或 `未知` |
| 哪项重复价值消失了，或变得更难获得？ | `价值事件；此前/当前频率或质量；受影响 Cohort；证据`，或 `未知` |
| 哪个机制能够预测留存变化的形状和时间？ | `候选机制；预测曲线/行为；支持证据；矛盾证据`，或 `未知` |
| 哪项证据能够区分价值损失与提醒、定价或获客质量影响？ | `证据/测试；可区分的竞争机制；决策规则；负责人/日期`，或 `未知` |

### 证据充分性检查

只有在自然周期、受影响的留存组成、Cohort 模式与重复价值机制均已确定时，才能继续。否则回答 `证据不足：缺失证据；负责人；收集日期`，或 `未知`。

## 商业化

从[商业化知识模块](../monetization/README.zh.md)和[商业化案例](../../case-library/themes/monetization/README.zh.md)开始。

### 症状与机制

商业化症状是转化、收入、扩展、毛利或回本偏弱。机制解释失败原因，例如价值实现不足、付款人与用户不一致、套餐薄弱、价格与价值错配、支付摩擦、服务成本过高或渠道经济性不健康。

### 诊断问题

| 问题 | 预期答案格式 |
| --- | --- |
| 哪项收入结果低于目标，影响哪个付款人分群？ | `指标；定义；基线/当前值/目标；付款人分群；周期`，或 `未知` |
| 用户、付款人、受益人与采购决策者是否为同一角色？ | `角色；主体；获得的价值；决策权；冲突`，或 `未知` |
| 不同分群的转化、ARPU（每用户平均收入）/ACV（年度合同价值）、收入留存、扩展、毛利与回本有何差异？ | 表格/列表：`分群；各项指标；周期；来源`，或 `未知` |
| 受影响 Cohort 是否在商业化节点前获得并留住产品价值？ | `价值事件；付费前激活；付费后留存；比较对象；来源`，或 `未知` |
| 哪个机制最能解释收入与客户价值模式？ | `候选机制；预测商业行为；支持证据；矛盾证据`，或 `未知` |
| 哪项证据能够区分价格、套餐、摩擦和底层价值？ | `证据/测试；可区分的竞争机制；决策规则；护栏；负责人/日期`，或 `未知` |

### 证据充分性检查

只有在付款人分群、留存产品价值、收入组成与贡献经济性均已知，并有测试区分商业设计和弱价值时，才能继续。否则回答 `证据不足：缺失证据；负责人；收集日期`，或 `未知`。

## 增长循环

从[推荐与扩张知识模块](../referral-and-loops/README.zh.md)和[增长循环案例](../../case-library/themes/referral-and-loops/README.zh.md)开始。

### 症状与机制

增长循环症状是参与、扩散、接收者转化、循环速度或重复价值偏弱。机制解释失败原因，例如参与动机不足、Payload 薄弱、接收者不信任、场景不合适、周期过长、接收者价值不足、滥用控制或平台依赖。

### 诊断问题

| 问题 | 预期答案格式 |
| --- | --- |
| 哪个循环应当把产品行为转化为更多分发或价值？ | `循环类型；输入事件；参与者；接收者；回流价值；闭环事件`，或 `未知` |
| 循环在哪个环节损失的产出最多？ | `参与率；单个参与者 Payload；接收者转化；周期时间；接收者留存`及对应数值和周期，或 `未知` |
| 衡量到的扩散是否为增量？ | `Holdout/Lift 方法；总扩散；增量扩散；不确定性`，或 `未知` |
| 接收者是否获得价值，并具备继续循环的能力或动机？ | `接收者价值事件；价值实现时间；留存；下一轮参与`，或 `未知` |
| 哪个机制能够预测薄弱环节和受影响参与者？ | `候选机制；预测模式；支持证据；矛盾证据`，或 `未知` |
| 哪项证据能够区分动机、Payload、转化和接收者价值问题？ | `证据/测试；可区分的竞争机制；决策规则；负责人/日期`，或 `未知` |
| 哪些成本或伤害限制循环？ | `奖励/平台成本；滥用；蚕食；信任影响；阈值/负责人`，或 `未知` |

### 证据充分性检查

只有在循环端到端定义清楚、最弱转化与周期时间已衡量、接收者留存已知，并且增量与伤害有明确边界时，才能继续。否则回答 `证据不足：缺失证据；负责人；收集日期`，或 `未知`。

## 增长系统

从[增长指标体系](../metrics/README.zh.md)、[实验系统](../experimentation/README.zh.md)、[增长基础设施](../growth-infrastructure/README.zh.md)和[增长组织](../growth-organization/README.zh.md)开始。

### 症状与机制

增长系统症状是衡量、决策、执行或学习缓慢或不一致。机制解释失败原因，例如所有权不清、指标定义冲突、数据质量低、依赖排队、交付能力缺失、实验设计薄弱或知识无法复用。

### 诊断问题

| 问题 | 预期答案格式 |
| --- | --- |
| 系统没有改善哪项客户价值结果？ | `结果指标；基线/当前值/目标；分群；负责人；周期`，或 `未知` |
| 工作在哪个环节停滞：衡量、优先级、审批、交付、分析还是应用？ | `环节；中位数/分位数延迟；队列规模；失败率；来源`，或 `未知` |
| 指标定义、数据质量和决策权是否明确？ | `项目；负责人；定义/位置；质量状态；未解决冲突`，或 `未知` |
| 实验是否针对当前约束并最终形成决策？ | `实验；关联约束；决策规则；结果；决策；复用情况`，或 `未知` |
| 问题属于能力缺失、流程薄弱、所有权不清还是需求不稳定？ | `候选机制；预测运营模式；支持证据；矛盾证据`，或 `未知` |
| 哪项证据能够区分这些系统机制？ | `证据/测试；可区分的竞争机制；决策规则；负责人/日期`，或 `未知` |
| 哪项重复、稳定的工作拥有明确内部使用者，值得建设共享基础设施？ | `工作流；频率；变异程度；使用者；当前成本；预期收益`，或 `未知` |

### 证据充分性检查

只有在客户价值结果、负责人、可衡量瓶颈与机制均已明确，并且任何共享能力都具备重复需求和明确使用者时，才能继续。否则回答 `证据不足：缺失证据；负责人；收集日期`，或 `未知`。

# 产品市场匹配

> 产品市场匹配（PMF）是一项可辩护的判断：一个明确市场能够反复从产品获得有意义价值。它不能单独证明分发、经济可行性或规模，也不是单一分数或永久徽章。

**[English](README.md)｜[简体中文](README.zh.md)**

## 定义与边界

PMF 连接具体市场、问题、产品与重复结果。它比兴趣、注册增长、满意度、收入或单一调查分数更严格。

| 术语 | 含义 | 边界 |
| --- | --- | --- |
| 问题证据 | 一个明确群体存在重要任务、痛点或期望结果 | 频率与严重度不能证明产品解决了问题。 |
| 产品价值 | 产品帮助该群体取得有意义进展 | 成功的首次体验不能证明重复价值。 |
| 产品市场匹配 | 一个明确分群反复获得足够价值，因而继续使用、采用或付费 | 匹配仍可能狭窄、经济性薄弱或难以分发。 |
| 增长 | 留存客户价值与商业价值随时间增加 | 获客可以放大匹配，但不能制造匹配。 |
| 规模化 | 机制能够跨更多流量、分群、市场或产品运作 | 规模化可能削弱最初小众市场中的匹配。 |

PMF 是针对一个明确单位的诊断。判断前需要写清客户或参与者、使用场景、地区、产品、付款人、自然频率与时间范围。

## 匹配系统

    明确市场与问题
    -> 产品承诺
    -> 首次价值
    -> 重复价值与留存
    -> 采用或支付意愿
    -> 可重复分发与可行经济性

PMF 描述这套更广系统中的重复市场与产品价值关系；分发和经济性决定该匹配能否支持目标增长与商业模式。

整条链的强度取决于最重要的薄弱约束。产品可能拥有留存用户却没有可行价格，也可能需求强但没有重复使用，或者经济性健康但分群太小，无法支持目标业务。

## 匹配单位

PMF 应在能够保留一致问题与价值机制的最小单位上判断。

| 产品结构 | 有用的匹配单位 | 聚合数据可能隐藏的证据 |
| --- | --- | --- |
| 消费产品 | 用户 x 使用场景 x 自然频率 | 广泛低意图获客掩盖高留存核心用户 |
| B2B 产品 | 角色 x 工作流 x 账户类型 x 付款人 | 终端用户使用强，但买方价值或账户留存弱 |
| 协作产品 | 个人价值 x Workspace 或账户采用 | 有邀请或席位，却没有重复协作工作 |
| Marketplace | 参与方 x Atomic Market x 品类 | 全球增长掩盖本地流动性或单边健康问题 |
| 多产品公司 | 产品 x 分群 x 产品组合关系 | 互导或收入蚕食其他产品的留存价值 |
| AI 产品 | 任务 x 结果质量 x 用户或账户环境 | 试用量或 Prompt 活跃不能产生可靠重复结果 |

一个单位中的匹配不能在缺少新证据时推广到相邻分群。

## 证据栈

没有单一信号能够证明 PMF。更强判断需要对齐多层证据，并让冲突保持可见。

| 层级 | 证据 | 诊断问题 | 常见假阳性 |
| --- | --- | --- | --- |
| 问题 | 频率、严重度、替代方案、Workaround、切换行为 | 问题是否重要到足以改变行为？ | 访谈热情，但当前行为没有成本 |
| 首次价值 | 成功价值事件、Time to Value、定性解释 | 合格用户能否取得有意义结果？ | 完成引导或界面操作 |
| 重复价值 | 按价值行为和自然周期观察成熟留存 | 价值能否在不持续重新获客时重复？ | 新 Cohort 推动的聚合活跃 |
| 声明依赖 | 合格用户的 Must-Have 调查及原因 | 成功用户失去产品后是否会受到实质影响？ | 粉丝、员工或过早受访者带来的高分 |
| 分群 | 按场景或客户条件划分的留存、价值与依赖 | 匹配在哪里最强、最弱或不同？ | 把不兼容用户混成一个分数 |
| 自然拉力 | Direct Demand、回访、推荐、客户语言、销售拉力 | 价值是否创造自发需求或倡导？ | 没有激活或留存的公开关注 |
| 经济性 | 支付意愿、贡献、回本、服务成本 | 已交付价值能否支持可行模式？ | 忽略变动成本、流失或渠道成本的收入 |
| 分发 | 增量合格获客与 Channel-Model Fit | 能否反复触达市场而不破坏经济性？ | 临时渠道高峰或补贴增长 |

独立信号相互一致时，置信度应提高；证据不成熟、经过选择、由模型产生或相互矛盾时，置信度应下降。

## Sean Ellis Test

Sean Ellis Test 询问用户：如果不能继续使用该产品，他们会有什么感受。常被引用的 40% “非常失望”是一项经验启发，不是通用认证。

合格样本应满足：

- 用户已经体验核心价值；
- 已经过足够时间判断重复有用性；
- 受访者属于命名清晰的分群或使用场景；
- 记录获客来源、使用时长、套餐、角色与活跃程度；
- 分数需要与开放文本原因一起分析；
- 保留未回复偏差与幸存者偏差。

    非常失望比例
    = 选择“非常失望”的合格受访者
      / 所有完成回答的合格受访者

报告样本量、资格条件、回复率、置信区间、调查日期与分群构成。把回答与已观察激活、留存、扩张和流失对照，而不是把分数变成通过或失败门槛。

## 留存证据

留存是最广泛适用的重复价值行为证据，但必须使用正确行为、Cohort 与周期。

    合格 Cohort
    -> 到达首次价值
    -> 重复承载价值的行为
    -> 在产品自然周期内留存

有用分析包括：

- 获客、激活与日历 Cohort；
- 适用情况下分别读取用户、账户、收入、供给与需求留存；
- 使用承载价值的行为，而不只是登录；
- 为使用频率不同的产品观察频率分布；
- 把召回和扩张与原始留存分开；
- 在相同年龄比较成熟 Cohort；
- 同时报告分群留存曲线与绝对 Cohort 规模。

趋于平稳的曲线支持存在一个留存群体，但不能建立市场规模、因果机制、毛利或继续获得类似用户的能力。

## 定性证据

访谈用于解释行为背后的机制。优先覆盖反复获得价值的用户、尝试后离开的用户、没有采用的买方，以及行为发生变化的客户。

| 证据 | 问题 |
| --- | --- |
| 触发 | 寻找解决方案前发生了什么变化？ |
| 替代方案 | 客户此前使用、忍受或付费购买什么？ |
| 进展 | 哪项结果变得更简单、更快、更安全或更有价值？ |
| 取舍 | 采用需要承担哪些成本、风险、设置、行为或依赖？ |
| 重复 | 为什么问题或期望结果会再次出现？ |
| 失去 | 如果产品消失，会发生什么？ |
| 语言 | 客户使用哪些词描述问题与价值？ |

不能把赞美、功能请求或加入 Waitlist 的意愿等同于行为变化。

## Four Fits

| Fit | 核心问题 | 证据 |
| --- | --- | --- |
| Market-Product Fit | 产品是否为明确市场解决重要问题？ | 价值事件、留存、客户解释、替代方案 |
| Product-Channel Fit | 产品是否适合渠道分发与转化需求的方式？ | 渠道机制、激活质量、边际触达 |
| Channel-Model Fit | 渠道经济性是否支持商业模式？ | CAC、回本、贡献、销售或转化成本 |
| Model-Market Fit | 市场规模、价格与成本是否支持目标业务？ | 可触达客户、ARPU、毛利、可获得份额 |

四种匹配相互约束。市场、产品、渠道、价格、套餐或销售动作发生变化时，其他假设也需要重新审视。

## Model-Market Fit

Model-Market Fit 用来检验目标市场与收入模式能否支持预期规模。

    单客年度收入
    x 明确市场中的可触达客户
    x 可现实获得的份额
    = 潜在年度收入

这个公式是一项假设，不是预测。市场定义、支付意愿、可触达客户数、竞争反应、流失、毛利与获得份额所需时间，都可能使其失效。

狭窄初始分群可能非常适合寻找留存价值，却不足以支持长期收入目标。需要记录可能扩展的市场，以及每次转换时必须重新证明哪些 Fits。

## 经济性与支付意愿

免费使用、收入与 PMF 是不同信号。经济证据用于判断价值能否支持客户关系与交付系统。

检查：

- 付款人、受益人和决策者；
- Price Metric 与支付意愿；
- 毛利与贡献毛利；
- 实施、支持、推理、履约或供给成本；
- 获客成本与回本；
- 按套餐或分群划分的留存与扩张；
- 折扣与激励依赖；
- 现金、容量、信任与监管约束。

强留存与结构性负贡献可能证明用户价值，却不能证明可行商业模式。强劲早期收入也可能来自合同、新鲜感或切换成本，而不是重复客户价值。

## 分群与扩张

PMF 通常先在狭窄条件中找到，再逐步扩大。

    找到留存分群
    -> 描述问题与价值机制
    -> 识别匹配边界
    -> 测试一个相邻分群
    -> 对比价值、留存、经济性与分发
    -> 扩张、调整或停止

相邻分群可能需要不同定位、新用户引导、工作流、产品深度、渠道、销售动作、定价或服务模式。应报告分群特定证据，而不是把转换隐藏在公司级平均值里。

## PMF Treadmill

产品进入新分群、改变承诺、采用新渠道或价格，以及技术、竞争、监管与用户预期变化时，匹配都可能减弱。

    已建立的匹配
    -> 环境或范围变化
    -> 旧信号减弱或产生误导
    -> 产品、渠道或模式调整
    -> 重新验证匹配

AI-native 品类尤其明显，因为模型能力、成本、界面、竞争者与用户预期都可能快速变化。只有发布速度持续连接客户证据与留存结果时，它才有价值。

## 指标体系

使用精简指标星座，而不是一个 PMF 分数。

| 层级 | 示例指标 |
| --- | --- |
| 结果 | 留存且获得价值的用户或账户、留存贡献 |
| 首次价值 | 合格激活、Time to Value、成功任务或交易 |
| 重复 | Cohort 留存、频率、重复工作流、重复交易 |
| 依赖 | 合格用户的 Must-Have 回答与原因 |
| 拉力 | Direct Demand、推荐、自然获客、销售拉力 |
| 经济性 | 支付意愿、贡献毛利、回本、服务成本 |
| 护栏 | 信任、质量、退款、支持负担、集中度、安全 |

正确的定义取决于产品结构与自然频率。把已观察行为同预测、调查声明与管理目标分开。

## 诊断地图

| 观察到的模式 | 下一步检查 | 可能的解释 |
| --- | --- | --- |
| 注册增长强，留存弱 | 来源、承诺、激活、使用场景、Cohort 年龄 | 分发正在放大薄弱或错位价值。 |
| 调查分数强，行为弱 | 资格、样本、回复偏差、产品使用时长 | 声明依赖经过选择，或用户没有反复到达价值。 |
| 留存强，自然拉力弱 | 切换成本、狭窄需求、协作、认知 | 价值可能真实，但不会自然形成分发。 |
| 小众留存强，扩张弱 | 相邻问题、角色、工作流、品类、新用户引导 | PMF 只存在于特定分群，扩张假设错误。 |
| 使用强，经济性弱 | 付款人、Price Metric、服务成本、Channel Model | 产品价值存在，但商业模式不可行。 |
| 收入增长，活跃价值下降 | 合同、折扣、结构、扩张质量 | 商业结果暂时与客户价值脱节。 |
| Marketplace 增长但一方流失 | Atomic Market、流动性、质量、激励、参与者经济性 | 聚合增长掩盖本地或双边匹配破损。 |
| 原本强的 Cohort 变弱 | 产品变化、竞争、技术、预期、可靠性 | PMF 已经衰退，需要重新获得。 |

这些模式用于指导调查，不能单独建立因果关系。

## 成熟度模型

| 阶段 | 特征 | 下一项约束 |
| --- | --- | --- |
| 1. 兴趣 | 关注、访谈、注册、Waitlist 或早期收入 | 定义分群、问题、价值事件与自然频率。 |
| 2. 初始价值 | 合格用户到达有意义的首次结果 | 建立重复行为与成熟 Cohort 证据。 |
| 3. 留存分群 | 一个命名清晰的分群反复获得价值 | 对齐调查、定性、经济与分发证据。 |
| 4. 可行匹配系统 | 产品、市场、渠道与模式协同运作 | 证明边际表现与一条相邻扩张路径。 |
| 5. 自适应匹配 | 公司能够发现衰退并在变化中重新验证匹配 | 在范围与组织扩张时保留客户理解。 |

成熟度取决于证据质量与决策准备度，而不是公司年龄、收入、人数、融资或一个阈值。

## 建设顺序

1. 定义客户或参与者、使用场景、问题、替代方案、付款人、地区与自然频率。
2. 使用稳定实体、窗口与 Cohort 定义首次价值和重复价值事件。
3. 访谈成功、失败、流失与未采用客户，解释机制与边界。
4. 按分群、来源、角色和使用场景衡量合格激活与成熟留存。
5. 只对合格用户运行 Sean Ellis Test，并保留样本、回复与开放文本背景。
6. 对齐已观察行为、声明依赖、客户语言、自然拉力与相反证据。
7. 测试支付意愿、贡献、服务成本、回本与 Channel-Model Fit。
8. 写清当前 PMF 主张、置信度、边界、最强反证与未解决风险。
9. 每次只规模化一个机制或相邻分群，并与原留存分群比较边际质量。
10. 市场、产品、渠道、价格、模式、技术或客户预期变化时重新验证。

## PMF 复盘

周期性 PMF 复盘应该回答：

- 当前主张针对哪个分群与使用场景？
- 哪个价值事件会重复，自然频率是什么？
- 成熟 Cohort 正在留存、扩张还是变弱？
- 合格用户说自己会失去什么，为什么？
- 哪些证据与 PMF 主张冲突？
- 价格、贡献、回本与服务成本是否仍然可行？
- 产品、市场、渠道或模式变化是否使旧证据失效？
- 下一项决策是什么：深化、规模化、调整、收窄还是停止？

## 常见错误

- 把发布高峰、Waitlist、收入、投资人兴趣或社交关注当成 PMF。
- 对不合格或混合受访者使用 40% Rule。
- 把趋于平稳的留存曲线当成大市场或盈利能力证明。
- 把强弱分群平均成一个分数。
- 把激活完成误认为重复客户价值。
- 招聘增长团队制造产品尚未交付的价值。
- 在理解成熟留存与经济性前规模化获客。
- 假设 PMF 一旦获得就永久存在。

## 证据与局限

- PMF 是由多种证据支持的诊断判断，不是可直接观察的二元状态。
- 调查、留存、市场、经济与分发证据可能冲突，冲突不应被隐藏。
- 阈值与周期取决于产品频率、市场、分群、商业模式与证据质量。
- 公开公司材料通常缺少完整 Cohort、调查样本、经济性、反事实与失败扩张证据。
- 公司结果不能归因于单一指标、框架、人物或团队。

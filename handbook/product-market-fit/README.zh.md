# 产品市场匹配

> 产品市场匹配（PMF）是证据表明：一个明确市场能够反复从产品获得有意义价值，并且这种关系可以支持持续使用与可行商业模式。它不是发布里程碑，也不是永久徽章。

**[English](README.md)｜[简体中文](README.zh.md)**

## 定义与边界

PMF 连接具体市场、问题、产品和重复结果。它比兴趣、注册增长、满意度或单一调查分数更严格。

| 信号 | 能说明什么 | 单独不能证明什么 |
| --- | --- | --- |
| 客户访谈 | 问题、语言、替代方案和感知价值 | 实际重复行为或市场规模 |
| Sean Ellis Test | 选定用户群声明的 Must-have 依赖 | 通用匹配或因果留存 |
| 激活 | 用户能够到达早期价值事件 | 价值能否持续一个自然周期 |
| 留存 Cohort | 一批用户的重复行为是否持续 | 健康经济性或广泛市场需求 |
| 自然需求 | 用户推荐、搜索或回到产品 | 增长是否增量且持续 |
| 支付意愿 | 部分用户在经济上重视结果 | 可扩张毛利、回本或市场总量 |

## 匹配系统

```text
明确市场与问题
-> 产品承诺
-> 首次价值
-> 重复价值与留存
-> 采用或支付意愿
-> 可重复分发与可行经济性
```

PMF 必须按分群判断。一个用途中的强匹配，可能被更大但弱匹配的人群平均掉。

## Four Fits

| Fit | 核心问题 |
| --- | --- |
| Market-Product Fit | 产品是否为明确市场解决重要问题？ |
| Product-Channel Fit | 产品是否适合渠道分发与转化需求的方式？ |
| Channel-Model Fit | 渠道经济性是否支持商业模式？ |
| Model-Market Fit | 市场规模、价格和成本是否支持目标业务？ |

四种匹配相互约束。产品价值可以真实存在，但分发或经济性仍可能无法成立。

## Model-Market Fit

Model-Market Fit 用来检验目标市场与收入模式能否支持预期规模。

```text
单客年度收入
x 目标市场客户数量
x 可现实获得的份额
= 潜在年度收入
```

这个公式是一项假设，不是预测。市场定义、支付意愿、可触达客户数量、竞争反应、流失、毛利与获得份额所需时间，都可能使计算失效。

狭窄初始分群可能非常适合寻找留存价值，却不足以支持长期收入目标。此时需要记录下一步扩展市场，以及每次转换时必须重新验证哪些 Four Fits。市场、产品、渠道、价格或套餐发生重要变化时，其他假设也需要重新评估。

## 证据框架

| 层级 | 证据 | 诊断问题 |
| --- | --- | --- |
| 问题 | 频率、严重度、替代方案、切换行为 | 问题是否重要到足以改变行为？ |
| 价值 | 成功价值事件与定性解释 | 强用户获得了什么结果？ |
| 留存 | 自然周期内成熟 Cohort 曲线 | 价值能否在不持续重新获客时重复？ |
| 分群 | 按用途划分的留存与 Must-have 回答 | 匹配在哪里最强、哪里最弱？ |
| 分发 | 增量获客质量与口碑 | 能否反复触达市场？ |
| 经济性 | 贡献毛利、回本、支付意愿 | 价值能否支持可行模式？ |

不存在证明 PMF 的通用阈值。Sean Ellis Test 中常被引用的 40% “非常失望”是经验启发，其有效性取决于受访者选择、发送时机、样本量、分群与产品频率。

## PMF Treadmill

产品进入新分群、改变承诺、采用新渠道或价格，以及技术、竞争和用户预期变化时，匹配都可能减弱。

```text
在狭窄分群找到匹配
-> 扩展到相邻需求
-> 发现留存或价值变化
-> 调整产品、渠道或模式
-> 重新验证匹配
```

AI-native 品类尤其明显，因为模型能力、成本、界面、竞争者和用户预期都可能快速变化。

## 问题诊断

| 模式 | 可能的问题 | 下一步证据 |
| --- | --- | --- |
| 获客强，留存弱 | 承诺错位或重复价值不足 | 按来源、用途和激活路径看留存 |
| 调查分数强，行为弱 | 选择偏差或声明价值未实现 | 审核合格样本与 Cohort 行为 |
| 小众留存强，扩张弱 | 相邻分群具有不同问题 | 分群访谈与独立 Cohort |
| 使用强，经济性弱 | 价格、付款人、成本或渠道模式错位 | 贡献毛利与支付意愿 |
| 原本强的 Cohort 变弱 | 产品、市场、竞争或预期变化 | Cohort 对比与流失用户研究 |
| 增长依赖折扣或付费 | 自然需求或渠道匹配弱 | Holdout、回本与激励后留存 |

## 操作方法

1. 定义市场、用户、问题、当前替代方案和自然使用频率。
2. 确定价值事件，并访谈反复获得该价值的用户。
3. 按用途、来源与分群衡量激活和成熟留存。
4. 把 PMF 调查当作一项输入，而不是通过或失败门槛。
5. 测试采用或支付意愿，并与服务成本和分发对账。
6. 明确写出置信度与相反证据。
7. 产品、分群、渠道、价格或市场变化时重新验证。

## 常见错误

- 把发布高峰、Waitlist、收入或投资人兴趣当成 PMF。
- 对不合格或混合受访者使用 40% Rule。
- 把强弱分群平均成一个分数。
- 招聘增长团队来制造产品尚未交付的价值。
- 在理解成熟留存与经济性前规模化获客。
- 假设 PMF 一旦获得就永久存在。

## 证据与局限

- PMF 是由多种证据支持的诊断判断，不是可直接观察的二元状态。
- 调查、留存、市场与经济性证据可能冲突，冲突不应被隐藏。
- 阈值和时间周期取决于产品频率、市场、分群与商业模式。

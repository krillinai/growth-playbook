# 推荐与增长循环

> 增长循环把产品价值或用户行为转化为新的输入，持续创造价值与留存分发。

**[English](README.md)｜[简体中文](README.zh.md)**

## 循环模型

```text
价值事件 -> 可分享成果或邀请 -> 接收者行动
         -> 接收者价值 -> 新成果 -> 下一轮循环
```

口碑是一种动机与传播过程；推荐是被产品推动的机制；病毒传播是可衡量的扩散形态；增长循环是完整、重复发生的系统。

## 指标体系

| 指标 | 公式或定义 | 用途 |
| --- | --- | --- |
| 参与率 | `分享或邀请用户数 / 符合条件用户数` | 循环入口 |
| 分支系数 | 每位参与者带来的平均合格接收者 | 潜在触达 |
| 接收者转化率 | `获得价值的接收者 / 合格接收者` | 接收者质量 |
| 留存调整 K 值 | 参与者 × 接收者 × 价值转化 × 留存质量 | 可持续扩散 |
| 循环周期 | 从一个价值事件到下一个生成价值事件的时间 | 循环速度 |
| 增量成本 | 每位增量留存用户对应的奖励与运营成本 | 经济性 |

即使 `K > 1`，如果接收者质量低、周期很慢、行为依赖补贴或产品无法留存需求，循环仍不可持续。

## 循环诊断

| 中断位置 | 下一步调查 |
| --- | --- |
| 很少用户分享 | 动机弱、时机错误、没有有用成果 |
| 发送多，打开少 | 信任低、接收者不相关、渠道疲劳 |
| 打开后激活弱 | 接收路径或价值错位 |
| 转化强，留存弱 | 激励用户或产品价值不足 |
| 推荐归因强，整体提升小 | 蚕食或归因重叠 |

## 操作方法

1. 画出从一个价值事件到下一个价值事件的每一步。
2. 分别定义参与者动机和接收者价值。
3. 衡量每条边的合格条件、转化、质量、周期和流失。
4. 使用 Holdout 或控制曝光估计增量。
5. 加入欺诈、疲劳、权限、信任、平台与毛利护栏。
6. 改善最弱环节，而不是增加更多提示。

## 常见错误

- 接收者尚未获得价值时就把邀请称为循环。
- 强迫分享或申请不必要权限。
- 只看总 K 值，不看留存与增量。
- 没有反作弊和蚕食控制就上线奖励。

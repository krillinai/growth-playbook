# 推荐与增长循环

> 增长循环把产品价值或用户行为转化为新的输入，持续创造价值与留存分发。

**[English](README.md)｜[简体中文](README.zh.md)**

## 循环模型

```text
价值事件 -> 可分享成果或邀请 -> 接收者行动
         -> 接收者价值 -> 新成果 -> 下一轮循环
```

口碑是一种动机与传播过程；推荐是被产品推动的机制；病毒传播是可衡量的扩散形态；增长循环是完整、重复发生的系统。

## 指标体系

| 指标 | 公式或定义 | 用途 |
| --- | --- | --- |
| 参与率 | `分享或邀请用户数 / 符合条件用户数` | 循环入口 |
| 分支系数 | 每位参与者带来的平均合格接收者 | 潜在触达 |
| 接收者转化率 | `获得价值的接收者 / 合格接收者` | 接收者质量 |
| 留存调整 K 值 | 参与者 × 接收者 × 价值转化 × 留存质量 | 可持续扩散 |
| 循环周期 | 从一个价值事件到下一个生成价值事件的时间 | 循环速度 |
| 增量成本 | 每位增量留存用户对应的奖励与运营成本 | 经济性 |

即使 `K > 1`，如果接收者质量低、周期很慢、行为依赖补贴或产品无法留存需求，循环仍不可持续。

## 循环诊断

| 中断位置 | 下一步调查 |
| --- | --- |
| 很少用户分享 | 动机弱、时机错误、没有有用成果 |
| 发送多，打开少 | 信任低、接收者不相关、渠道疲劳 |
| 打开后激活弱 | 接收路径或价值错位 |
| 转化强，留存弱 | 激励用户或产品价值不足 |
| 推荐归因强，整体提升小 | 蚕食或归因重叠 |

## 操作方法

1. 画出从一个价值事件到下一个价值事件的每一步。
2. 分别定义参与者动机和接收者价值。
3. 衡量每条边的合格条件、转化、质量、周期和流失。
4. 使用 Holdout 或控制曝光估计增量。
5. 加入欺诈、疲劳、权限、信任、平台与毛利护栏。
6. 改善最弱环节，而不是增加更多提示。

## 病毒传播

案例证据：[Facebook、Brian Balfour 与 Sean Ellis](../../case-library/themes/referral-and-loops/README.zh.md#概念证据地图)。

病毒传播是用户或产品输出触达并转化更多用户的扩散模式。衡量参与率、合格接收者、接收者价值转化、留存调整 K 值与循环周期。

没有接收者价值的病毒触达只是分发噪音。网络效应与病毒传播不同：产品可以快速扩散，却不随使用增长而变得更有价值。

## 网络效应

案例证据：[James Currier、Andrew Chen、Casey Winters、Facebook、Uber 与 Grubhub](../../case-library/themes/referral-and-loops/README.zh.md#概念证据地图)。

当相关参与者、互动、数据、兼容性、声誉或生态活动增长时，产品对参与者变得更有价值，这才构成网络效应。病毒传播描述产品如何扩散；网络效应描述参与增长如何改变价值。两者可以分别独立存在。

### 网络单元与核心互动

定义能够持续产生参与者价值的最小边界。网络单元可以按地区、品类、职业、社会关系、组织或时间划分，再明确核心互动及其成功条件。

总用户数或总供给可能掩盖破碎的局部网络。应在每个网络单元内衡量密度、可用性、匹配或互动成功、获得价值所需时间、重复行为与留存。

### White-Hot Center 与 Hard Side

从需求强烈、互动可以形成密度、产品能够稳定完成核心任务的窄分群开始。识别限制价值的参与者、供给、专业能力、信任或行为。Hard Side 必须根据稀缺性、可用性、质量、动机与留存诊断，不能根据 Marketplace 标签直接假设。

### 强化机制与防御性

明确新增参与如何改善价值的因果机制，例如更多选择、更快匹配、更丰富数据、更强声誉、兼容性、流动性、信任或工作流协同，然后检验竞争者能否复现。

Multi-homing、声誉可迁移、切换成本低、互动频率低、质量下降、密度破碎与供给商品化，都可能让大规模网络缺乏防御。相关性、信任、经济性、治理或参与者控制下降时，网络效应也会衰退。

### AI 时代网络审计

AI Agent 可以创造供给、完成工作、搜索、谈判或交易，从而改变稀缺参与者以及价值积累的位置。需要重新评估网络单元、Hard Side、专有互动数据、工作流位置、声誉、治理，以及人或 Agent 的新增参与是否仍能改善结果。

不能因为产品拥有用户、数据、集成或 AI 活动就判断其具备防御性。必须找到可观察的价值提升，以及竞争者仍难以复现的机制。

## 产品增长循环

案例证据：[Brian Balfour、Andrew Chen、Facebook、Uber 与 Lovable](../../case-library/themes/referral-and-loops/README.zh.md#概念证据地图)。

产品循环把增长写入正常产品价值。协作、共享成果、生成内容、交易和数据只有在接收者也获得价值时，才能创造下一项输入。

绘制每条边、资格规则、转化率、延迟与质量损失，并优先改善最弱环节，而不是继续增加顶部提示。

## AI 辅助 UGC 内容循环

AI 可以生成问题、结构或初始内容，降低用户参与创作所需投入。人工贡献随后增加具体性与新鲜度，最终结果重新进入搜索、Feed、分享与召回。

```text
需求或主题信号
-> AI 辅助初始内容
-> 合格人工贡献
-> 可索引或可分享输出
-> 接收者发现并获得价值
-> 新需求、贡献或回访
```

需要衡量贡献者参与率、被接受贡献的质量、页面价值、合格搜索或 Feed 需求、接收者激活、留存价值、更新成本与循环周期。还需要加入原创性、专家归属、Spam、错误信息、贡献者同意、搜索依赖与内容衰减护栏。

AI 输出数量不等于循环健康。只有内容能够帮助接收者，并且平台能够让质量和信任高于生产与审核成本时，机制才会产生复利。

## 用户倡导

案例证据：[Lovable、Duolingo 与 Ramp](../../case-library/themes/referral-and-loops/README.zh.md#概念证据地图)。

用户倡导是建立在真实价值和信任上的自愿推荐。客户故事、评论、推荐、社区和员工参与可以传播证明，但激励或脚本也可能削弱可信度。

衡量合格下游行为和信任，而不是只看 Testimonials 数量。

## 常见错误

- 接收者尚未获得价值时就把邀请称为循环。
- 在内容价值、原创性与接收者价值成立前，规模化 AI 生成或 AI 辅助页面。
- 强迫分享或申请不必要权限。
- 只看总 K 值，不看留存与增量。
- 没有反作弊和蚕食控制就上线奖励。

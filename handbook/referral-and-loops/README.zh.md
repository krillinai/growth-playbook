# 推荐与扩张

> 推荐与扩张把已经交付的产品价值转化为更多分发或更深客户价值。只有在考虑增量、成本、信任与滥用后，发送者、接收者、现有客户与业务仍能保留价值，机制才是健康的。

**[English](README.md)｜[简体中文](README.zh.md)**

## 定义与边界

这个生命周期环节包含两个相关结果：

- **推荐**通过现有用户、客户、合作伙伴或产品产出创造合格新用户或账户。
- **扩张**在现有用户、团队、账户、产品组合、网络或市场内加深价值。

两者相关，是因为同一产品行为可能同时实现二者。分享 Workspace 可以获得一位协作者，也可以提高原账户价值；Marketplace 交易可以强化本地供给，同时扩大地域覆盖。

| 概念 | 核心问题 | 边界 |
| --- | --- | --- |
| 口碑 | 用户为什么自愿讨论产品？ | 它是动机与沟通过程，本身不是产品流程。 |
| 推荐 | 可识别发送者是否创造了合格接收者？ | 没有接收者价值时，邀请与归因注册都不完整。 |
| 病毒传播 | 曝光如何通过参与者或产出扩散？ | 快速传播不能证明留存或网络效应。 |
| 产品循环 | 价值创造的输出能否成为下一项增长输入？ | 循环必须闭合，且每条边都有质量。 |
| 网络效应 | 新增相关参与是否提高其他参与者价值？ | 用户数、数据或集成本身不能建立网络效应。 |
| 扩张 | 现有客户是否获得更广、更深或更大范围的价值？ | 收入扩张属于商业化，本章解释采用机制。 |

留存衡量持续价值，商业化衡量价格、收入与经济价值捕获。本章聚焦价值如何传播和加深。

## 系统模型

```text
已交付价值
├── 分发路径
│   -> 发送者或产品输出 -> 合格接收者 -> 接收者价值
│   -> 留存新参与者 -> 下一次分发机会
└── 扩张路径
    -> 新用户、使用、产品、工作流或市场
    -> 更深客户价值 -> 留存采用
    -> 新输出、证明或商业容量
```

公司可以有推荐但没有扩张、有扩张但没有推荐、两者都有或两者都没有。组合前应分别建模。

## 参与者与价值地图

识别每位参与者、他们获得的价值、需要完成的行为以及承担的风险。

| 角色 | 必要价值 | 常见失败 |
| --- | --- | --- |
| 发送者或倡导者 | 效用、身份、声誉、帮助他人、协作或奖励 | 强迫提示或动机薄弱 |
| 接收者 | 相关上下文与可信价值路径 | Spam、低信任、不相关邀请 |
| 现有客户 | 更深产品或网络价值 | 新用户或使用反而增加协同成本 |
| 付款人或管理员 | 控制、证明、治理与可接受经济性 | 意外成本、权限或合规暴露 |
| 平台或合作方 | 符合政策、双方受益的活动 | 依赖、归因冲突或渠道关闭 |
| 业务 | 高于完整成本和风险的增量留存价值 | 只有总活动，没有因果或经济提升 |

不能通过把成本或风险转嫁给另一方来优化一方。

## 循环模型

```text
价值事件 -> 可分享成果、邀请、互动或证明
-> 合格曝光 -> 接收者行动 -> 接收者价值
-> 留存行为 -> 新价值事件 -> 下一轮循环
```

对每条边定义适用条件、事件、转化、延迟、质量、成本、所有权与护栏。只有最终状态能够可信地创造下一项输入时，图示才构成循环。

## 指标树

```text
来自推荐与扩张的增量留存价值
├── 分发
│   ├── 符合条件并参与的发送者
│   ├── 合格接收者与接收者激活
│   ├── 留存调整传播与循环周期
│   └── 增量留存新用户或账户
├── 扩张
│   ├── 账户广度：用户、团队、工作流、产品、市场
│   ├── 价值深度：频率、用量、成功任务、协作
│   ├── 留存采用与商业扩张
│   └── 增量贡献与回本
└── 护栏
    ├── 信任、许可、疲劳、投诉
    ├── 作弊、滥用、蚕食、补贴依赖
    └── 质量、支持、治理、参与者经济性
```

分子应该是增量结果，质量周期应该匹配产品自然频率。

## 指标字典

| 指标 | 定义 | 解释 |
| --- | --- | --- |
| 合格发送者率 | `符合条件的用户或账户 / 活跃人群` | 记录谁已经获得足够价值并拥有参与许可。 |
| 参与率 | `发送者或扩张者 / 符合条件的人群` | 循环入口；强迫或奖励也能提高它。 |
| 每位发送者合格接收者数 | 每位参与者带来的、符合相关性与身份规则的接收者 | 接收者质量下降时，更多发送并不更好。 |
| 接收者价值转化 | `W 内到达首次价值的接收者 / 合格接收者` | 比打开、点击、安装或注册更强。 |
| 接收者留存率 | `R 时留存的接收者 Cohort / 已激活接收者` | 检验推荐质量是否持续。 |
| 留存调整 K 值 | `参与率 x 合格接收者 x 接收者价值转化 x 留存质量` | 传播诊断估计，不是通用成功阈值。 |
| 循环周期 | 从一个价值事件到下一位产生价值的参与者或输出所需时间 | 缓慢但健康的循环仍可复利；速度需要与质量一起看。 |
| 扩张率 | `W 内加深价值的合格账户 / 合格账户` | 明确定义扩张状态与分母。 |
| 广度 | 每个实体内活跃用户、团队、工作流、产品或市场数量 | 广度可能只增加协同成本。 |
| 深度 | 每个实体内成功价值量、频率或复杂度 | 高活动可能代表重复失败或强制使用。 |
| 增量贡献 | 相对于反事实，由推荐或扩张带来的贡献 | 包括奖励、服务、支持、作弊与基础设施成本。 |

### K 值拆解

```text
总 K 值 = 发送者参与率 x 每位发送者接收者数 x 接收者转化

留存调整 K 值 = 总 K 值 x 接收者留存质量
```

`K > 1` 不能保证可持续增长。循环可能很慢、接收者质量可能很低、重叠可能很高，公司也可能在每轮循环中损失利润或信任。

## 增量与因果衡量

推荐归因经常给本来就会加入的用户记功。扩张也可能只是在产品或席位之间移动活动，并未增加客户总价值。

可用设计包括随机提示 Holdout、奖励 Holdout、发送者实验、接收者实验、市场分批上线、账户分阶段曝光与产品组合级衡量。保留原始分组直到留存和贡献读数。

| 风险 | 必要检查 |
| --- | --- |
| 推荐码捕获自然推荐 | 比较有提示或奖励与没有提示或奖励的合格用户。 |
| 现有用户冒领新用户奖励 | 身份、设备、支付与图谱控制。 |
| 产品 A 增长，产品 B 失去同一用户 | 产品组合级蚕食与总价值。 |
| 席位增加，但没有活跃协作 | 按席位观察活跃角色与价值事件采用。 |
| 收入只因价格上涨而扩张 | 把采用扩张与定价、合同影响分开。 |

## 推荐设计

推荐系统需要与价值一致的发送理由、相关接收者、可信信息、低摩擦路径与留存结果。

| 设计选择 | 核心问题 |
| --- | --- |
| 触发 | 发送者是否已经获得足够价值，可以推荐产品？ |
| 接收者 | 谁更可能受益，如何保护相关性？ |
| Payload | 什么上下文、产物、邀请或证明到达接收者？ |
| 激励 | 奖励是在强化产品价值，还是购买低质量行为？ |
| 落地状态 | 接收者是否带着有用上下文、权限和明确下一步进入？ |
| 闭环 | 接收者价值是否自然创造下一次协作、分享或加深使用的机会？ |

提示时机和文案不能修复薄弱的发送者动机或接收者价值。

## 病毒传播

病毒传播是参与者或产品输出触达并转化更多参与者的扩散模式。它可以内生于沟通、协作、内容、交易或身份，也可以通过提示与奖励制造。

应衡量 Payload、频率、合格触达、接收者转化、留存质量、循环周期、重叠、疲劳与增量。没有接收者价值的病毒触达只是分发噪音。

## 产品增长循环

产品循环把分发或扩张写入正常价值创造。常见输出包括共享产物、邀请、生成内容、交易、集成、数据与公开证明。

```text
输入 -> 创造价值的行动 -> 有用输出 -> 分发或复用
-> 接收者或客户价值 -> 新输入
```

描绘每项转化并识别最弱环节。提高输出量可能降低信任或质量，从而削弱完整循环。

## 网络效应

当新增相关参与通过密度、选择、流动性、数据、声誉、兼容性、信任或工作流协同改善其他参与者价值时，才存在网络效应。

病毒传播描述扩散，网络效应描述价值变化。产品可以只具备其中一项。

### 网络单元与核心互动

定义能够持续产生参与者价值的最小边界。网络单元可以按地区、品类、职业、社会关系、组织或时间划分，再明确核心互动及其成功条件。

总用户数或总供给可能掩盖破碎网络。应在每个单元内衡量可用性、密度、匹配或互动成功、Time to Value、质量、重复行为与留存。

### 冷启动、White-Hot Center 与 Hard Side

从需求强烈、核心互动可以形成密度、产品能够稳定完成任务的窄分群开始。识别限制价值的参与者、供给、专业能力、信任或行为。

Hard Side 会随市场和阶段变化。应根据稀缺性、可用性、质量、动机、经济性与留存诊断，而不能使用固定 Marketplace 标签。

### 强化机制与防御性

明确新增参与如何改善价值的因果机制，再检验竞争者能否复现。Multi-homing、声誉可迁移、切换成本低、频率低、质量差、密度破碎与供给商品化，都可能让大规模网络缺乏防御。

当拥堵、Spam、相关性下降、不信任或参与者经济性恶化时，网络效应也可能反转，让每个新增参与者降低价值。

### AI 时代网络审计

AI Agent 可以创造供给、完成工作、搜索、谈判或交易。需要重新评估网络单元、Hard Side、专有互动数据、工作流位置、声誉、治理，以及人或 Agent 的参与是否仍改善结果。

用户、数据、集成或 AI 活动不能证明防御性。必须找到可观察的价值提升，以及竞争者仍难以复现的机制。

## 协作驱动分发

协作产品可以同时获得被邀请用户并扩张现有 Workspace。邀请应该携带产物、角色、上下文、权限与有意义的任务。

```text
单用户价值 -> 共享产物或工作流 -> 协作者价值
-> 多用户工作流 -> 留存账户价值 -> 更多协作
```

应衡量受邀用户激活、到达协作价值所需时间、活跃角色、重复多用户工作流、账户留存、支持、权限错误与增量账户扩张。大部分席位不活跃时，更多席位不是成功。

## 用户倡导

用户倡导是建立在真实价值和信任上的自愿推荐。客户故事、评论、社区、推荐、公开产出与员工参与可以传播证明。

区分真实倡导、付费推广、脚本式 Testimonials 与奖励驱动行为。衡量合格下游价值与信任，而不是只看 Testimonials 或提及数量。

## AI 辅助 UGC 内容循环

AI 可以生成问题、结构、分类或初始产物，降低贡献成本。合格人工补充能够增加具体性与新鲜度，最终结果重新进入搜索、Feed、分享与召回。

```text
需求或主题信号 -> AI 辅助初始产物
-> 合格人工贡献 -> 有用的可索引或可分享输出
-> 接收者发现并获得价值 -> 新需求、贡献或回访
```

应衡量贡献者参与、被接受贡献质量、页面价值、合格需求、接收者激活、留存价值、更新成本、审核负担与循环周期，并加入原创性、归属、许可、错误信息、Spam、搜索依赖与内容衰减护栏。

AI 输出量不等于循环健康。

## 扩张模型

扩张是在现有关系内改变价值的广度、深度或范围。

```text
留存核心价值
-> 相邻需求或参与者
-> 合格扩张机会
-> 新范围内的首次价值
-> 重复且受治理的采用
-> 留存客户与业务价值
```

定义扩张实体：用户、Workspace、账户、产品、网络或市场。不能把它们混在一个比例中。

## 扩张类型

| 类型 | 价值机制 | 有用证据 | 主要风险 |
| --- | --- | --- | --- |
| 席位 | 更多相关参与者加入团队或账户 | 活跃席位采用、协作、账户留存 | 非活跃席位、权限与协同负担 |
| 用量 | 现有参与者完成更多有价值任务 | 成功价值量、深度、重复使用 | 强制活动、成本增长、重复失败 |
| 工作流 | 产品支持更多重复客户任务 | 工作流激活与跨工作流留存 | 功能膨胀与匹配薄弱 |
| 产品 | 客户采用另一个产品或模块 | 产品激活、Attach、留存多产品价值 | 产品未准备好就交叉销售或产生蚕食 |
| 账户 | 采用扩展到另一个团队、部门或业务单元 | 新单元激活、治理、留存账户广度 | 采购、安全、变更管理摩擦 |
| 地域或网络 | 已验证互动在另一个本地单元复现 | 到达密度所需时间、质量、留存、经济性 | 总规模掩盖薄弱局部网络 |

NRR（净收入留存率）等商业扩张指标验证价值捕获，但不能解释采用本身是否健康。参见[商业化](../monetization/README.zh.md#收入留存与扩张)。

## 扩张资格与顺序

扩张信号应该表明需求、准备度、权限、产品价值与运营容量。产品使用可以揭示机会，却不能证明预算或许可。

1. 确认原范围内存在留存价值。
2. 找到具有相关需求的相邻参与者、任务、产品、单元或市场。
3. 验证前置条件、权限、安全、支持与经济容量。
4. 在把 Attach 或合同称为成功前，交付新范围内的首次价值。
5. 衡量重复采用、客户总价值、蚕食与支持负担。
6. 只有新单元能够独立成立后，才扩展方法。

## 激励与奖励

奖励可以降低协同摩擦、补偿投入，或让与价值一致的收益更明显。它也可能购买低质量传播、扭曲产品信号、吸引作弊，并替代本来会自然发生的行为。

分别衡量总推荐、增量推荐、接收者激活、奖励后留存、奖励与服务成本、作弊损失、蚕食与信任。上线前定义递减与停止规则。参见[激励系统](../incentive-growth/README.zh.md)。

## 信任、滥用、疲劳与治理

| 风险 | 护栏 |
| --- | --- |
| Spam 与通知疲劳 | 频率限制、相关性规则、接收者控制、退订 |
| 强迫分享或权限请求 | 明确许可与最小权限访问 |
| 假账户与奖励作弊 | 身份、设备、支付、图谱与行为控制 |
| 低质量 UGC（用户生成内容）或 AI 输出 | 质量复核、来源、审核与纠错路径 |
| 网络拥堵或恶意参与者 | 排序、声誉、执行与参与者经济性 |
| 账户扩张缺少治理 | 角色、审计日志、数据边界与管理员控制 |
| 产品组合蚕食 | 客户总价值与产品组合级增量 |

信任是系统结果，不是最后一步合规审批。

## 分群与 Cohort 分析

按发送者价值状态、接收者关系、邀请类型、奖励曝光、获客来源、产品角色、账户规模、工作流、市场、网络单元与扩张类型拆分，并比较同等成熟的 Cohort（同期群）。

推荐 Cohort 应在尊重隐私的前提下保留发送者与接收者关联。扩张 Cohort 应记录原始价值状态、扩张起点、新范围，以及原范围是否仍然健康。

## 按周期读取实验

| 周期 | 推荐问题 | 扩张问题 |
| --- | --- | --- |
| 即时 | 合格发送者和接收者是否完成预期转化？ | 合格客户是否进入新范围？ |
| 首次价值 | 接收者是否带着有用上下文激活？ | 新用户、工作流、产品或单元是否获得价值？ |
| 下游 | 增量接收者是否留存并产生贡献？ | 广度或深度是否持续，且没有损害原价值？ |
| 系统 | 信任、经济性、网络质量与治理是否健康？ | 完整成本与蚕食后，账户或市场总价值是否提高？ |

在所需周期内保持发送者、接收者、账户与市场分组稳定。

## 诊断地图

| 可观察模式 | 下一步检查 | 可能的应对 |
| --- | --- | --- |
| 很少合格用户参与 | 发送者价值、触发、动机、Payload | 把触发放到价值之后，或改善有用输出。 |
| 发送多、合格打开少 | 接收者相关性、信任、频率 | 缩窄资格，并改善上下文或许可。 |
| 打开强、激活弱 | 落地状态、权限、接收者任务 | 传递上下文并缩短价值路径。 |
| 归因推荐强、整体提升弱 | 自然重叠、身份、Holdout 结果 | 改变归因并减少或移除奖励。 |
| 总 K 值强、留存弱 | 激励用户、接收者错配、价值过慢 | 优化接收者价值与留存质量。 |
| 席位更多，协作很少 | 角色匹配、邀请、Workspace 准备度 | 销售更多席位前设计首个多用户工作流。 |
| 一个网络单元成立，相邻单元失败 | 密度、Hard Side、本地运营 | 缩小单元并重建冷启动方案。 |
| 产品 Attach 上升，支持与流失恶化 | 资格、实施、蚕食 | 限制扩张并恢复核心价值。 |

这些模式用于缩小调查范围，不能直接证明根因。

## 成熟度模型

| 阶段 | 特征 | 下一项约束 |
| --- | --- | --- |
| 1. 邀请与 Attach | 团队报告发送、推荐码、席位或产品 Attach | 定义参与者价值、资格、首次价值与留存。 |
| 2. 闭合循环 | 发送者到接收者路径和扩张状态可衡量 | 增加增量、循环周期与完整成本。 |
| 3. 留存经济性 | 留存调整传播与扩张经济性决定决策 | 增加滥用、蚕食、信任与网络单元视图。 |
| 4. 组合系统 | 多个循环与扩张路径共享治理和清晰所有权 | 协调账户、产品、市场与平台依赖。 |
| 5. 自适应价值网络 | 机制根据角色、意图、密度、质量和容量变化 | 保持参与者控制、因果有效性与韧性。 |

接收者或扩张实体：

## 建设顺序

1. 定义让推荐或扩张可信的已交付价值。
2. 描绘参与者、实体、动机、接收者价值与风险。
3. 画出从初始价值事件到留存新增或更深价值的每条边。
4. 固定资格、事件、窗口、身份规则、成本与护栏。
5. 在增加提示或奖励前建立发送者或客户动机。
6. 向接收者或新范围交付带有上下文的首次价值。
7. 衡量留存、增量、循环周期、贡献、信任与滥用。
8. 改善最弱环节，而不是提高循环顶部产量。
9. 只有本地价值可以重复后，才扩张到相邻账户、产品或网络单元。
10. 当激励、参与者、产品结构、经济性或平台规则改变时重新验证。

## 常见错误

- 接收者尚未获得价值时，就把邀请或分享称为循环。
- 把病毒传播、网络效应、推荐与口碑当成同义词。
- 使用总 K 值，却不看留存、重叠、循环周期与增量。
- 把购买或不活跃席位当成健康扩张。
- 原价值系统尚未留存时就扩张产品或市场。
- 没有作弊、递减与蚕食控制就上线奖励。
- 在价值、原创性与审核成立前规模化 AI 生成输出。
- 忽略权限、治理、参与者经济性或接收者信任。

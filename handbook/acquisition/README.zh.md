# 获客

> 获客把明确的目标人群和问题连接到可信的产品价值路径。流量是投入，能够留存的增量客户才是结果。

**[English](README.md)｜[简体中文](README.zh.md)**

## 边界

获客从可触达需求开始，到合格用户或账户进入价值路径为止。激活和留存用于验证获客质量，不能与渠道指标割裂优化。

## 获客系统

```text
人群与问题 -> 承诺与证据 -> 渠道 -> 合格到达
           -> 激活 -> 留存 -> 经济性 -> 再投入
```

渠道可以分为自然、私域、付费、合作、产品驱动、销售驱动与产品组合分发。真正重要的是机制、可控性、饱和度和经济性，而不是渠道名称。

## 指标树

| 层级 | 指标 | 决策 |
| --- | --- | --- |
| 触达 | 合格触达、目标需求份额 | 是否触达目标市场？ |
| 响应 | 访问、线索、安装、注册率 | 承诺与证据是否促成行动？ |
| 质量 | 按来源划分的激活与留存转化 | 渠道是否带来能获得价值的用户？ |
| 增量 | Holdout 提升、辅助与增量转化 | 没有渠道时用户是否仍会到来？ |
| 经济性 | 增量 CAC、贡献毛利、回本周期 | 渠道能否持续扩张？ |

`增量 CAC = 增量渠道成本 / 增量获得客户数`

`留存获客率 = 留存获客用户数 / 符合条件的获客用户数`

## 分群与诊断

按人群、意图、渠道、创意或承诺、地区、设备和 Cohort 拆分。归因视图必须与增量估计分开。

| 模式 | 可能的问题 | 下一步检查 |
| --- | --- | --- |
| 点击高，激活弱 | 承诺错位或低意图触达 | 按创意、关键词、人群和落地路径看激活 |
| 激活强，回本弱 | 成本、毛利或销售周期问题 | 按 Cohort 看贡献毛利与回本 |
| 渠道报告转化超过整体增长 | 归因重叠 | Holdout、地区实验或时间增量实验 |
| 初期扩量后快速衰减 | 饱和或素材疲劳 | 边际 CAC、频次、人群重叠 |

## 操作方法

1. 定义人群、问题、价值事件与留存结果。
2. 让渠道机制匹配用户发现和评估方案的方式。
3. 固定来源分类、Cohort 窗口、质量指标和经济性护栏。
4. 扩大投入前先测试承诺、证据、人群和路径。
5. 读取边际表现，并估计增量，而不是只看平均值。
6. 按留存贡献重新分配资源，而不是按平台报告转化。

## 渠道选择矩阵

渠道应该按照公司当前需要做出的决策来选择，而不是按照热门策略的通用排名来选择。

| 维度 | 核心问题 |
| --- | --- |
| 目标 | 当前优先级是学习、规模、成本效率，还是触达特定分群？ |
| 定向 | 渠道能否足够精确地触达目标客户和意图？ |
| 成本 | 需要哪些前置、边际与运营成本？ |
| 投入时间 | 多久可以开始第一项可信测试？ |
| 结果时间 | 激活、留存或收入证据需要多久成熟？ |
| 控制能力 | 能否主动启动、停止并分群控制曝光？ |
| 规模 | 渠道在饱和前能提供多少合格需求？ |

选择最符合目标与约束的一到两个渠道假设。浅层测试很多渠道，可能比深度测试一个渠道获得的信息更少；深度测试需要连接受众、承诺、落地路径、激活与留存经济性。

## Channel-Model Fit

商业模式限制了哪些获客渠道能够成立。较低的单客年收入通常需要低成本、低摩擦渠道；成本更高的销售与合作伙伴渠道，需要足够的留存毛利来支持获客与回本。

```text
单客留存年度贡献
-> 可接受获客成本与回本
-> 可行渠道集合
```

定价与套餐变化可能破坏原本有效的渠道。产品摩擦、ARPU 或 ACV、毛利、转化延迟、回本与资金要求需要一起评估。

## 付费获客

案例证据：[字节跳动与 Ramp 的获客系统](../../case-library/themes/acquisition/README.zh.md#概念证据地图)。

付费获客需要按边际、增量、留存后的经济性管理，而不是按平台归因管理。把人群、承诺、素材、出价、落地路径、激活、留存与贡献毛利放在同一 Cohort 视图中。

`增量回本周期 = 增量获客成本 / 周期增量贡献毛利`

只有边际质量和回本仍在决策边界内时才扩大。素材疲劳、人群饱和、归因重叠和延迟留存会让历史平均表现失真。

### 渠道衰减与付费依赖

随着新鲜感消失、竞争者复制方法、高意图受众饱和和频次增加，渠道表现通常会下降。不能把早期 CTR、CAC 或回本周期当作规模化后的常数。

应按渠道、素材、受众和 Cohort 衡量边际表现，区分单渠道 CAC 与 Blended CAC，定义可接受回本周期并观察渠道集中度。当预算必须持续增加、边际 CAC 恶化，同时没有第二增长引擎时，付费获客会成为结构性风险。

应对方式不是自动停止预算，而是根据经济性限制暴露，改善产品价值与留存，并投资行为机制不同于饱和渠道的分发方式。

## 内容与社媒分发

案例证据：[Ramp、Lovable 与 Duolingo 相关分发证据](../../case-library/themes/acquisition/README.zh.md#概念证据地图)。

内容通过解释问题、演示价值、建立信任或产生可传播资产来创造需求。社媒分发取决于参与者可信度、原生形式、时机、互动，以及从注意力通往产品价值的路径。

| 机制 | 领先信号 | 持续验证 |
| --- | --- | --- |
| 教育内容 | 合格阅读与回访 | 激活并留存的需求 |
| 产品演示 | 相关受众进入产品 | 成功价值事件 |
| 员工或客户分发 | 可信参与与分享 | 增量合格用户 |
| 品牌社媒 | 品牌需求与直接响应 | 留存、转化与信任 |

不要把内容数量或粉丝增长当成客户价值，并应区分自然响应与本来会从其他渠道到来的需求。

## 可规模化 SEO 页面系统

当结构化供给、UGC 或主动生产内容可以满足大量具体意图，同时避免形成单薄页面时，SEO 就成为产品能力。

```text
意图分类
-> 有用的页面模板与内容来源
-> 可索引的未登录体验
-> 合格下一步行为
-> 激活与留存
```

应衡量索引质量、合格流量、入口页转化、下游激活、留存与内容维护成本。页面数量本身不是结果。

## Kindle 与 Fire 策略

早期“Kindle”策略使用人工、狭窄或不可规模化的工作发现留存客户分群，并理解机制为何有效。“Fire”策略把这些学习转化成可重复获客。

只有理解客户、价值、渠道模式匹配与边际经济性后，才适合进入规模化。PMF 前扩张会放大漏损；机制得到证明后仍停留在人工阶段，又会限制复利。

## 有边界的产品与功能发布

发布应该先把产品提供给当前假设所对应的人群，再随着证据增强逐层扩展。

```text
限定受众
-> 触达并过滤
-> 观察定性反应
-> 验证留存产品价值
-> 扩展到下一层人群
```

无差别发布会制造嘈杂反馈、削弱留存证据、损害口碑并消耗用户注意力。在扩大分发前，应定义目标客户、过滤规则、成功信号、核心产品护栏与扩展决策。Publicity 只是触达机制，不是 Product Fit 或 Feature Fit 的证明。

## 常见错误

- 把曝光、点击、安装、线索或注册当成增长。
- 比较成熟窗口不同的渠道。
- 平均 CAC 尚可时忽略边际 CAC 恶化。
- 把归因功劳当成因果贡献。

# 获客

> 获客把明确的目标人群和问题连接到可信的产品价值路径。流量是投入，能够留存的增量客户才是结果。

**[English](README.md)｜[简体中文](README.zh.md)**

## 边界

获客从可触达需求开始，到合格用户或账户进入价值路径为止。激活和留存用于验证获客质量，不能与渠道指标割裂优化。

## 获客系统

```text
人群与问题 -> 承诺与证据 -> 渠道 -> 合格到达
           -> 激活 -> 留存 -> 经济性 -> 再投入
```

渠道可以分为自然、私域、付费、合作、产品驱动、销售驱动与产品组合分发。真正重要的是机制、可控性、饱和度和经济性，而不是渠道名称。

## 指标树

| 层级 | 指标 | 决策 |
| --- | --- | --- |
| 触达 | 合格触达、目标需求份额 | 是否触达目标市场？ |
| 响应 | 访问、线索、安装、注册率 | 承诺与证据是否促成行动？ |
| 质量 | 按来源划分的激活与留存转化 | 渠道是否带来能获得价值的用户？ |
| 增量 | Holdout 提升、辅助与增量转化 | 没有渠道时用户是否仍会到来？ |
| 经济性 | 增量 CAC、贡献毛利、回本周期 | 渠道能否持续扩张？ |

`增量 CAC = 增量渠道成本 / 增量获得客户数`

`留存获客率 = 留存获客用户数 / 符合条件的获客用户数`

## 分群与诊断

按人群、意图、渠道、创意或承诺、地区、设备和 Cohort 拆分。归因视图必须与增量估计分开。

| 模式 | 可能的问题 | 下一步检查 |
| --- | --- | --- |
| 点击高，激活弱 | 承诺错位或低意图触达 | 按创意、关键词、人群和落地路径看激活 |
| 激活强，回本弱 | 成本、毛利或销售周期问题 | 按 Cohort 看贡献毛利与回本 |
| 渠道报告转化超过整体增长 | 归因重叠 | Holdout、地区实验或时间增量实验 |
| 初期扩量后快速衰减 | 饱和或素材疲劳 | 边际 CAC、频次、人群重叠 |

## 操作方法

1. 定义人群、问题、价值事件与留存结果。
2. 让渠道机制匹配用户发现和评估方案的方式。
3. 固定来源分类、Cohort 窗口、质量指标和经济性护栏。
4. 扩大投入前先测试承诺、证据、人群和路径。
5. 读取边际表现，并估计增量，而不是只看平均值。
6. 按留存贡献重新分配资源，而不是按平台报告转化。

## 常见错误

- 把曝光、点击、安装、线索或注册当成增长。
- 比较成熟窗口不同的渠道。
- 平均 CAC 尚可时忽略边际 CAC 恶化。
- 把归因功劳当成因果贡献。

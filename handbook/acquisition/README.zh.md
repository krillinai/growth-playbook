# 获客

> 获客把明确的客户问题和目标人群连接到可信的产品价值路径。触达是投入，能够激活、留存并产生可接受贡献的增量客户才是结果。

**[English](README.md)｜[简体中文](README.zh.md)**

## 定义与边界

获客从识别可触达需求开始，到合格用户或账户进入产品价值路径为止。它包括需求承接、需求创造、分发、渠道运营，以及从外部承诺到自有产品体验的过渡。

| 概念 | 在系统中的作用 | 为什么不能单独代表获客成功 |
| --- | --- | --- |
| 触达 | 让目标人群看到或接触产品 | 曝光可能不相关、重复或没有增量。 |
| 流量或线索 | 产生可衡量的到达 | 到达不能证明匹配、价值或购买意图。 |
| 注册或安装 | 打开产品入口 | 用户可能在首次价值前流失。 |
| 激活 | 检验渠道承诺是否带来价值 | 它验证获客质量，但属于下一生命周期环节。 |
| 留存与贡献 | 验证长期价值和经济性 | 结果成熟更晚，必须回连到来源 Cohort。 |

因此，获客应该优化连接后的结果，而不是渠道后台的局部最大值。

## 获客模型

```text
客户与问题
-> 意图与可触达需求
-> 承诺、证据与 Offer
-> 渠道与分发机制
-> 入口界面与合格到达
-> 激活与留存价值
-> 贡献、回本与再投入
```

每个箭头都可能成为约束。更多触达不能修复错误承诺；正确承诺不能修复断裂的落地路径；低 CAC 不能修复低留存。

## ICP、意图与 Offer 匹配

定义应该获得谁、他们希望取得什么进展、需求何时变得紧迫，以及什么证据能够降低不确定性。只有当 ICP 会改变渠道、信息、路径、经济性或服务设计时，它才具有决策价值。

| 层级 | 核心问题 | 证据 |
| --- | --- | --- |
| 客户匹配 | 谁更可能获得并保留价值？ | 客户任务、约束、留存 Cohort、支付意愿 |
| 意图 | 什么事件或需要让行动变得及时？ | 搜索、触发事件、行为、请求、进入品类 |
| 承诺 | 向用户提供什么结果？ | 信息理解与合格响应 |
| 证据 | 用户为什么应该相信？ | 产品演示、客户证据、声誉与信任 |
| Offer | 用户现在应该做什么？ | 试用、注册、咨询、导入、模板、交易 |

活动可能触达了正确的人口特征，却没有触达正确的客户任务。应把来源与意图一起诊断。

## 渠道组合

渠道应该按照机制、可控性、饱和度、依赖和经济性分类，而不是按照流行标签分类。

| 类型 | 核心机制 | 常见约束 |
| --- | --- | --- |
| 自然发现 | 搜索、答案引擎、目录、应用商店 | 内容质量、权威、索引、意图匹配 |
| 赢得式分发 | 口碑、媒体、社区、公开参与 | 信任、新鲜度、贡献质量、低可控性 |
| 自有分发 | 邮件、自有受众、活动、客户界面 | 受众质量、许可、疲劳 |
| 付费获客 | 购买触达或响应 | 边际 CAC、素材疲劳、归因重叠 |
| 产品驱动分发 | 分享、协作、嵌入产物、集成 | 接收者价值、产品匹配、滥用控制 |
| 合作伙伴 | 捆绑、生态、渠道、联合销售 | 激励一致性、所有权、运营依赖 |
| 销售驱动获客 | 外呼、资格判断、解决方案销售 | ACV、周期、产能、实施成本 |
| 产品组合分发 | 产品或市场之间的交叉推广 | 蚕食、相关性、治理 |

目标不是平均分散，而是建立失败机制不同、整体经济性仍可理解的渠道组合。

## 渠道选择矩阵

渠道应该按照当前决策和约束来选择，而不是按照通用策略排名来选择。

| 维度 | 决策问题 |
| --- | --- |
| 目标 | 当前优先级是学习、规模、效率、市场入口还是战略控制？ |
| 客户入口 | 渠道是否在相关时机聚集了目标客户？ |
| 产品匹配 | 渠道形式和行为能否传达产品价值？ |
| 模式匹配 | 留存毛利能否承担渠道及其运营成本？ |
| 投入时间 | 多快可以开始可信测试？ |
| 证据时间 | 激活、留存与收入需要多久成熟？ |
| 控制能力 | 能否启动、停止、分群并审计曝光？ |
| 规模与衰减 | 饱和前存在多少合格需求？ |
| 依赖 | 引入哪些平台、合作方、政策或数据风险？ |

选择一到两个渠道假设，深度测试到能够连接受众、承诺、入口路径、激活、留存质量与经济性。浅层测试大量渠道通常只会产生动作，不会产生学习。

## Channel-Model Fit

商业模式限制了可行渠道集合。较低的单客留存年度贡献通常需要低成本、低摩擦分发；销售、合作伙伴和重实施渠道则需要足够毛利和现金承受能力来支持获客。

```text
单客留存贡献
-> 可接受 CAC 与回本周期
-> 可行渠道与服务模式
-> 可触达市场与增长上限
```

价格、套餐、转化延迟、毛利、流失、销售产能、实施工作与资金要求需要一起评估。定价或客户分层变化可能让原本有效的渠道失效。

## 指标树

```text
增量留存贡献
├── 增量留存客户
│   ├── 合格触达与到达
│   ├── 按来源和意图划分的激活
│   └── 按来源和 Cohort 划分的留存
├── 单位经济性
│   ├── 边际 CAC
│   ├── 贡献毛利
│   └── 回本与现金需求
└── 护栏
    ├── 信任、投诉与品牌质量
    ├── 平台与政策风险
    └── 集中度、作弊与蚕食
```

渠道局部指标用于定位路径问题，不能替代系统结果。

## 指标字典

| 指标 | 定义 | 有用拆分与解释 |
| --- | --- | --- |
| 合格触达 | 符合已记录相关性规则的目标人群曝光 | 人群、意图、市场、渠道；触达估计可能重叠。 |
| 合格到达率 | `合格到达 / 符合条件的曝光或响应` | 信息、素材、入口界面；需要明确质量规则。 |
| 获客激活率 | `W 内到达首次价值的获客用户 / 符合条件的获客用户` | 来源、意图、活动、市场、Cohort。 |
| 留存获客率 | `R 时仍留存的获客用户 / 符合条件的获客用户` | 使用产品自然频率和成熟 Cohort。 |
| 增量客户 | 相对于反事实，由干预真正带来的客户 | Holdout、地区、时间或随机实验。 |
| 增量 CAC | `增量获客成本 / 增量客户` | 包括媒体、激励、工具、代理、素材和变动人工。 |
| 边际 CAC | 下一批 Cohort 或下一档投入的获客成本 | 扩量时比历史平均 CAC 更有用。 |
| 回本周期 | `获客成本 / 周期贡献毛利` | 使用贡献而非收入，并考虑不确定性和流失。 |
| 渠道集中度 | 最大渠道占合格获客或贡献的比例 | 高集中度增加依赖和议价风险。 |
| 素材疲劳 | 随曝光累积，合格响应或经济性下降 | 按频次、人群、素材、市场和时间拆分。 |

指标定义必须记录适用人群、分子、分母、窗口、来源分类、成本口径、排除项、归因规则与负责人。

## 归因与增量

归因分配可观察功劳；增量估计没有渠道或干预时不会发生的结果。两者回答不同问题。

| 方法 | 适用场景 | 主要局限 |
| --- | --- | --- |
| 末次或首次触点 | 运营路由和简单报告 | 忽略或过度强调其他触点。 |
| 多触点模型 | 描述可观察旅程 | 模型假设不能建立因果。 |
| 平台归因 | 在单个平台内优化 | 自报功劳可能与其他渠道重叠。 |
| 随机 Holdout | 估计因果提升 | 需要统计功效、稳定曝光和溢出控制。 |
| 地区或市场实验 | 衡量无法按用户随机化的渠道 | 市场之间存在差异和污染。 |
| 时间实验 | 测试清晰的启动、停止或切换 | 季节性与同期变化可能混杂结果。 |

保留两种视图：归因帮助运营渠道，增量决定资源分配。不能把平台报告转化相加后称为公司增长。

## 分群与 Cohort 分析

只在结果可能改变决策的维度拆分获客：

- 人群、客户任务、意图与账户类型；
- 渠道、活动、关键词、素材、合作方与入口页；
- 市场、语言、设备、平台，以及新用户与回访用户；
- 激活深度、留存周期、商业化状态与贡献。

比较同等成熟的 Cohort，同时报告比例与绝对贡献。一个渠道可能转化率很强但合格规模太小，也可能规模很大但留存价值很弱。

## 自然与赢得式分发

当有用的产品或内容界面可以持续被发现并满足意图时，自然分发才会复利。赢得式分发取决于独立的人是否主动讨论、分享、引用或演示价值。

| 机制 | 领先信号 | 长期验证 |
| --- | --- | --- |
| 搜索与答案发现 | 合格曝光与访问 | 按查询或界面划分的激活与留存需求 |
| 教育内容 | 合格阅读与回访 | 增量 Pipeline 或产品价值 |
| 产品演示 | 相关受众进入产品 | 成功价值事件与留存 |
| 社区参与 | 有帮助的贡献与可信响应 | 合格关系与持续需求 |
| 品牌社媒 | 品牌需求与直接响应 | 增量获客、信任与留存 |

不能把发布数量、粉丝、提及或排名等同于客户价值。

## 可规模化 SEO 页面系统

当结构化供给、UGC 或主动生产内容可以满足大量具体意图，同时避免形成单薄或误导页面时，SEO 就成为产品能力。

```text
意图分类
-> 有用的页面模板与可靠内容来源
-> 可索引的未登录体验
-> 产品理解与合格下一步行为
-> 激活、留存与维护
```

应衡量索引质量、合格流量、按入口界面划分的转化、激活、留存、更新成本与搜索政策风险。优化真实落地界面，而不只是首页。关于生成式答案系统中的可见性与引用，参见 [GEO](../geo/README.zh.md)。

## 内容与社媒分发

内容可以解释问题、演示价值、建立信任或产生能够被他人分发的资产。社媒分发取决于参与者可信度、原生形式、时机、互动，以及从注意力通往价值的路径。

区分内容角色：品类教育、需求承接、产品证明、客户赋能与公开参与。它们需要不同证据和成熟窗口。把内容和社媒曝光连接到合格到达、激活、留存需求与信任，而不是把粉丝增长当成结果。

## 产品驱动与合作伙伴分发

产品驱动获客通过有用的产品行为产生分发，例如分享产物、邀请、协作、集成、公开输出或交易。接收者必须获得价值，而不只是收到通知。

合作伙伴需要明确受众、价值交换、数据、归因、支持、经济性和终止条件的所有权。从合作方借来的分发既是资产，也是依赖。

## 付费获客

付费获客应该按照边际、增量和留存后的经济性管理，而不能只看平台归因。

```text
人群 -> 承诺 -> 素材 -> 出价与投放 -> 落地路径
-> 激活 -> 留存 -> 贡献 -> 更新资源分配
```

只有边际质量和回本仍在决策边界内时才扩大。使用投入分档或地区实验观察收益递减。成本口径应在重要时包括媒体、素材、工具、激励、代理、作弊和变动运营成本。

### 渠道衰减与付费依赖

随着新鲜感消失、竞争者复制方法、高意图受众饱和和频次增加，渠道表现通常会下降。不能把早期 CTR、CAC 或回本周期当作规模化后的常数。

应按渠道、素材、受众和 Cohort 观察边际表现。当预算必须持续增加、边际 CAC 恶化且没有第二机制时，付费获客会成为结构性风险。应根据经济性限制投入，改善产品价值与留存，并发展失败机制不同于饱和渠道的分发方式。

## 素材系统与疲劳

素材是由客户假设、信息、资产、投放环境与下游结果构成的系统。标注问题、Persona、价值 Pillar、Angle、证据、Format、Creator、市场、行动提示与 Concept 血缘，让学习能够积累。

| 维度 | 必答问题 |
| --- | --- |
| Persona | 正在触达哪种客户环境、动机或分群？ |
| Pillar | 正在表达哪项核心问题或价值？ |
| Angle | 正在测试哪种异议、解释、使用场景或关注理由？ |
| Format | 哪种执行形式承载 Concept？ |
| Hook 与 Proof | 如何获得注意力并让承诺可信？ |
| Concept 血缘 | 这是 Net-new Hypothesis，还是已有 Concept 的迭代？ |

区分 Concept 与它的具体执行：

```text
客户证据
-> Persona x Pillar x Angle 假设
-> Format、Hook、Creator 与 Placement
-> 平台投放与落地路径
-> 新客户与下游结果
-> 更新 Concept 地图
```

执行测试改变 Concept 的表达方式；Net-new Concept 改变客户、问题、价值、异议、Offer 或 Proof 假设。两者都可能有用，但统一称为“更多素材”会隐藏团队究竟学到了什么。

| 失败模式 | 检查 | 应对 |
| --- | --- | --- |
| 频次上升、响应下降 | 人群饱和与素材年龄 | 更新假设或受众，而不只是换视觉。 |
| 点击强、激活弱 | 承诺与产品错位 | 让证据和落地路径对齐真实价值。 |
| 平台转化强、整体提升弱 | 归因重叠 | 运行 Holdout 或市场实验。 |
| 变体更多、学习没有增加 | 分类薄弱或自动化噪音 | 降低产量并提高假设质量。 |

除了单个 Winner，还要 Audit 整体组合覆盖。投入集中可能代表强 Concept、组合探索不足、平台投放偏好或归因偏差，不能证明其他客户需求不存在。没有原创判断、品牌控制、权利与下游衡量的素材产量，只会规模化噪音。

## 落地路径与下游质量

第一个自有界面需要延续渠道承诺、建立信任，并让目标访问者走向价值。按真实入口页诊断，而不只是按活动诊断。

```text
信息连续性
-> 理解与证据
-> 合格行动
-> 首次价值
-> 留存价值
```

把转化与跳出或放弃、理解、激活、错误、支持、留存和信任一起读取。强制 Gate 可能提高注册，同时降低产品理解和留存质量。

## 国际化与市场进入

国际获客不只是翻译。需要诊断市场需求、本地替代方案、渠道、内容供给、语言、文化、监管、支付、设备、网络、信任与运营所有权。

根据证据和战略匹配安排市场顺序。共享基础设施可以标准化分类、实验和报告；本地团队仍需要调整信息、素材、渠道组合、产品路径和合作伙伴执行的权力。

## 平台依赖与渠道组合

对每个渠道记录外部所有者、当前授权、数据访问、政策暴露、议价权、切换成本与恢复方案。依赖临时平台访问的渠道优势应该被视为易逝资产。

只有当第二种机制通过不同依赖触达合格需求时，多元化才有意义。增加多个都依赖同一平台、竞价系统或内容形式的渠道，并没有实质降低风险。

## Kindle 与 Fire 策略

早期“Kindle”工作使用人工、狭窄或不可规模化的方法发现留存客户分群，并理解机制为何成立。“Fire”工作把这些学习转化为可重复分发。

只有理解客户匹配、产品价值、渠道行为与边际经济性后，才适合规模化。PMF 前扩张会放大漏损；机制得到证明后仍停留在人工阶段，会限制复利。

## 有边界的产品与功能发布

发布应该先把产品提供给当前假设所对应的人群，再随着证据增强逐层扩展。

```text
定义受众 -> 控制入口 -> 观察响应
-> 验证留存价值 -> 扩展到下一层受众
```

Publicity 只是触达机制，不是 Fit 的证明。开放分发前应定义受众、过滤规则、成功信号、护栏与扩展决策。

## 按周期读取获客实验

| 周期 | 核心问题 | 常见证据 |
| --- | --- | --- |
| 即时 | 渠道或信息是否产生合格响应？ | 触达、理解、到达、作弊、成本 |
| 首次价值 | 获得的用户是否收到价值？ | 激活、Time to Value、错误、支持 |
| 下游 | 干预是否创造长期增量价值？ | 留存、贡献、回本、增量、信任 |

保留原始分组和来源直到下游读数。主要风险成熟更晚时，不能根据早期代理指标直接扩量。

## 诊断地图

| 可观察模式 | 下一步检查 | 可能的应对 |
| --- | --- | --- |
| 触达高、合格响应弱 | 人群、意图、承诺、证据 | 缩窄客户或改变价值主张。 |
| 到达强、激活弱 | 入口界面、信息连续性、来源结构 | 修复路径或停止错配来源。 |
| 激活强、留存弱 | 客户匹配、产品价值、Cohort 成熟度 | 扩量前重新检查定位和产品约束。 |
| 留存强、回本弱 | 边际 CAC、毛利、周期、服务成本 | 改变渠道、价格、套餐或运营模式。 |
| 报告转化超过整体增长 | 归因重叠与回访需求 | 运行增量实验并统一身份。 |
| 早期扩量后快速衰减 | 饱和、疲劳、竞价压力 | 读取边际分档并建立第二机制。 |
| 一个市场成立，其他失败 | 本地需求、供给、信任、产品准备 | 本地化系统或调整扩张顺序。 |

这些模式用于缩小调查范围，不能直接证明根因。

## 成熟度模型

| 阶段 | 特征 | 下一项约束 |
| --- | --- | --- |
| 1. 活动 | 团队优化触达、点击、线索或安装 | 定义合格客户、激活、留存和成本口径。 |
| 2. Cohort 质量 | 来源 Cohort 连接到激活和留存 | 增加边际经济性与稳定分类。 |
| 3. 增量经济性 | Holdout、贡献与回本决定资源分配 | 管理饱和、素材学习和渠道组合。 |
| 4. 组合系统 | 多种机制共享衡量并设有明确风险限制 | 协调本地适配、自动化与治理。 |
| 5. 自适应分发 | 资源根据意图、价值、经济性和产能变化 | 保持因果有效性、信任、韧性与人工判断。 |

成熟度不取决于渠道或看板数量，而取决于不确定环境下资源分配决策的质量和速度。

## 建设顺序

1. 定义客户、任务、意图、留存价值与经济性边界。
2. 描绘用户如何发现、评估并进入产品。
3. 使用客户、产品和模式匹配选择一到两个渠道假设。
4. 固定来源分类、成本口径、Cohort 窗口、激活、留存与护栏。
5. 扩大产量或投入前测试人群、承诺、证据、入口路径与 Offer。
6. 把获客 Cohort 连接到下游价值，并比较同等成熟的分群。
7. 估计增量，读取边际而非平均经济性。
8. 在明确的饱和、依赖、信任和现金约束内扩张。
9. 只有理解第二机制的角色和失败方式后才加入它。
10. 当产品、价格、市场、渠道政策或客户结构改变时重新验证。

## 常见错误

- 把曝光、点击、安装、线索或注册当成增长。
- 在产品价值和留存可信前扩张获客。
- 比较成本口径或成熟窗口不同的渠道。
- 平均 CAC 尚可时忽略边际 CAC 恶化。
- 把归因功劳当成因果贡献。
- 优化素材与落地转化时忽略下游质量。
- 把文字翻译称为国际化。
- 依赖平台却没有政策、数据或恢复方案。
- 在理解第一种机制前不断增加渠道。

## 核心增长思想

### 1. 区分新客与客户扩张

新客增长和现有客户扩张具有不同约束、负责人和指标。混在一起可能掩盖弱获客或弱产品深度。

### 2. 同时衡量规模与效率

Pipeline、客户或收入规模要与回本周期和贡献毛利一起看。回本周期可以解释投入现金何时返回，但仍依赖假设和 Cohort（同期群）成熟度。

### 3. 必要时使用多个价值指标

单一北极星可能过度简化多边或多产品业务。少量价值与经济性指标可以更好表示系统，但优先级必须清晰。

### 4. 结论性失败

当负向实验解决重要假设时，它具有价值。无结论实验没有改变决策，才是真正浪费。良好设计、埋点、样本量和预先决策标准可以改善学习质量。

### 5. 组织结构服从当前约束

增长团队的组织图不如清晰结果、产品与工程权限、决策速度和跨职能责任重要。

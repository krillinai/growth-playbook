## 增长相关经历

加入 Andreessen Horowitz 之前，Chen 曾负责 Uber Rider Growth，并参与多家风险投资支持的创业公司。这段经历把消费获客与 Marketplace 的核心约束连接起来：乘客需求无法脱离可用、可靠的司机供给独立增长。

他后续把 Marketplace、社交产品、协作工具等多用户产品中的网络增长模式整理成体系。这套框架最适用于产品价值依赖参与者互动的场景；如果单个用户已经能获得完整价值，就不应强行套用网络效应。

## 核心增长理念

### 1. 冷启动问题

网络产品在网络为空时，承诺的价值尚不存在。早期增长需要先在狭窄环境中形成足够多的相关参与者、内容、库存或互动，再尝试大规模扩张。

注册数量不能证明问题已经解决。Marketplace 即使拥有很多注册供给者，如果他们没有在相同地点、品类或时间与需求匹配，网络仍无法运转。

### 2. 从 Atomic Network 开始

Atomic Network 是能够维持核心价值的最小网络。它可能是一座城市、一个校园、一个工作场所、一个社区、一个品类、一条路线或一个创作者群体。

```text
选择狭窄网络
-> 招募 Hard Side
-> 建立足够密度
-> 产生可靠互动
-> 在相邻网络重复方法
```

目标不是人数最少，而是找到在密度、相关性与重复互动上足以运作的最小单元。

### 3. 优先解决 Hard Side

多数网络都有更难获取或留存、并限制另一侧价值的 Hard Side。在 Marketplace 中经常是供给侧，但具体取决于稀缺性、差异化与参与动机。

补贴可以帮助网络形成，但运营能力、产品工具、信任，以及赚钱或创作机会，经常决定 Hard Side 是否留下。

### 4. 网络密度比总规模更重要

总用户数会掩盖网络碎片化。团队需要衡量相关参与者能否在正确的地理、品类、社交或时间边界内找到彼此并完成核心互动。

指标随产品变化，可能包括匹配率、成功互动率、匹配时间、供给可用性、重复互动、局部网络留存和跨边留存。

### 5. 网络效应的阶段

Chen 把网络增长描述成连续阶段，而不是一次病毒传播：

```text
Cold Start -> Tipping Point -> Escape Velocity -> Hitting the Ceiling -> The Moat
```

- **Cold Start：** 建立第一批能运作的网络。
- **Tipping Point：** 形成启动相邻网络的可重复方法。
- **Escape Velocity：** 强化获客、参与和经济效应。
- **Hitting the Ceiling：** 面对饱和、质量下降、竞争或渠道限制。
- **The Moat：** 通过参与者、数据、声誉和互动价值建立防御。

当相关性下降、低质量参与者进入或局部密度丢失时，规模反而会削弱网络。更多用户不自动等于更强网络效应。

### 6. 增长循环必须回到产品价值

邀请、内容创作、Marketplace 交易和协作可以产生新的分发。只有接收者获得价值、留下，并创造下一项有用输入时，它们才成为持续循环。

如果不衡量接收者质量、留存、循环周期、增量和滥用，只看总邀请量或理论 K 值并不足够。

### 7. Marketplace 供给是一套运营系统

Uber 供给增长说明，Marketplace 增长可能需要逐市场运营、激励、预测、引导、流动性管理与产品改进。供给侧不只是另一个付费获客人群。

可以迁移的是识别本地流动性约束和参与者经济性，而不是复制 Uber 的具体策略、规模、劳动环境或城市运营方式。

### 8. Marketplace 商业化建立在流动性与价值之上

Marketplace 指标需要把交易规模连接到成功匹配、重复行为、Take Rate、贡献毛利和参与者健康。过早或过度商业化可能降低供给参与或需求转化，削弱创造收入的网络。

融资指标与运营指标也服务于不同决策。漂亮的总量指标不能替代 Cohort 质量、集中度、流动性和单位经济性。

### 9. 下行周期中的增长战略

资本成本上升时，团队需要更清楚地证明获客能够产生留存价值和可持续经济性。增长战略会从追求总量速度转向强分群、高效循环、留存、回本，以及能够改变资源分配的学习。

这不代表停止增长，而是区分持续机制与依靠阶段性廉价资本购买的增长。

### 10. 留存会复合病毒传播

Chen 在简单 Viral Factor 模型中加入了重复使用。一次符合用户价值的分享可能只触达少数人，但更强留存会创造更多使用 Session，也会让增长循环获得更多运行机会。

```text
长期病毒传播贡献
= X x Y x Z + X x Y x Z^2 + X x Y x Z^3 + ...

X = 参与分享或邀请的比例
Y = 每次行为触达的人数
Z = 两次机会之间的留存
```

这一模型更支持协作、内容分享等原生产品行为，而不是强制通讯录邀请。病毒传播可以放大获客，但不能挽救没有留存的产品。

### 11. 用分布衡量使用频率

Power User Curve 展示用户在一周或一个月内执行关键行为的天数分布。相比混合后的 DAU/MAU，它能区分偶尔使用者、中间用户与高频用户。

时间窗口和行为应匹配产品节奏。工作产品可以围绕工作日关键行为使用 L7，消费网络可以使用 L30。并非所有产品都需要形成“微笑曲线”，低频产品也可能创造并商业化足够价值。

### 12. 渠道效果会衰减

“The Law of Shitty Clickthroughs”描述了常见渠道规律：新鲜感消失、竞争者复制有效方法、规模扩张带来质量更低的受众，用户也会逐渐忽略熟悉的形式。

更新素材可以减慢衰减，却不能消除饱和。团队需要衡量边际渠道表现，持续寻找新分发机制，并建立与产品价值一致的渠道，不能假定早期 CAC 或 CTR 能在规模化后保持不变。

### 13. 避免付费营销依赖

当收入增长依赖持续增加预算，而渠道 CAC 上升、回本周期拉长、自然增长没有同步扩大时，付费获客会形成局部最优。

Chen 用 Dropbox 对比这一问题：付费搜索经济性不成立，促使公司转向文件夹分享和与产品价值一致的推荐机制。结论不是拒绝付费渠道，而是区分单渠道 CAC 与 Blended CAC、观察边际经济性，并在主渠道饱和前建设第二和第三增长引擎。

### 14. 先定义 CAC，再计算 CAC

CAC 文章由 Brian Balfour 撰写并发布在 Chen 的网站，文章区分 Customer Acquisition Cost 与 Cost Per Acquisition。CAC 对应付费客户，CPA 可以对应注册、Lead、试用或已激活的免费用户。

准确 CAC 还需要匹配投入与转化延迟，区分新增和回流客户，并计入工资、管理费用、工具以及免费产品阶段的相关支持成本。若销售或升级周期跨越多个月，用当月投入除以当月客户可能会得到相反的渠道判断。

### 15. 修复生命周期，而不是等待下一个功能

Next Feature Fallacy 指团队相信新增一个功能就会突然让用户开始使用整个产品。多数功能触达人数太少，或者位于新用户和非活跃用户无法跨越的“参与墙”之后。

用于改变增长结果的功能，应从整个生命周期评估 Reach 与 Impact。落地页、新用户引导、首次价值、召回和未登录页面，可能比只服务留存用户的深层功能更有杠杆。

### 16. 围绕约束建立增长团队

Chen 的增长团队方法区分核心产品与增长工作：核心产品深化产品价值，增长团队帮助更多人到达并重复获得价值，但两者边界取决于具体环境。

所有权可以从 Growth-as-a-Service 延伸到自主负责新用户引导、通知、实验、支付或获客系统。组织结构应跟随 KPI、已诊断问题、所需执行方式和维护负担，而不是复制其他公司的组织图。

## 框架总结

| 层级 | 核心问题 | 示例证据 |
| --- | --- | --- |
| Atomic Network | 最小可运作网络是什么？ | 局部密度与成功互动 |
| Hard Side | 哪类参与者限制价值？ | 可用性、创作、响应与留存 |
| Liquidity | 参与者能否可靠完成核心任务？ | 匹配率、匹配时间、Fill Rate |
| Reinforcement | 每次互动是否强化未来价值？ | 重复使用、选择改善、声誉与数据 |
| Expansion | 启动方法能否复制到相邻网络？ | 新网络达到流动性的时间与留存 |
| Economics | 激励与成本之后价值是否成立？ | 贡献毛利、回本、激励后留存 |
| Defense | 为什么网络难以替代？ | Multi-homing、切换、独特供给与信任 |

- 视频发布于 2016 至 2023 年，人物职位、产品、市场与观点可能已经变化。

## 增长相关经历

Winters 的经历横跨 Grubhub Marketplace、Pinterest 消费与国际增长、Eventbrite 产品组织，以及对 Airbnb、Canva、Faire、Reddit、Thumbtack、Tinder、Whatnot 等公司的顾问工作，并延伸到 AI-native 专业网络 SuperMe。

## 核心增长理念

### 1. 留存是判断 PMF 的主要证据

获客可以制造初始增长，却不能证明持续价值。Winters 更重视 Cohort 留存：用户是否按照产品自然频率继续获得足够价值并回来？

留存基准需要匹配品类、使用节奏和用户意图。曲线趋于稳定可以说明存在留存分群，但团队仍需理解留下的是谁、为何留下，以及这些行为能否支撑可行商业模式。

### 2. 新用户引导需要完成 Setup、Aha 与 Habit

Winters 把新用户引导拆成三项任务：

```text
Setup -> Aha -> Habit
```

- **Setup：** 收集让价值成为可能的最少信息、配置或连接。
- **Aha：** 让用户体验核心价值，而不是只完成注册。
- **Habit：** 按照产品自然节奏建立可信的重复价值路径。

团队应优化下游留存，而不是只看引导完成率。删除所有步骤可能提高转化，却让用户没有准备好获得价值。

### 3. 增长模型比收集框架更重要

Winters 反对用流行框架替代业务理解。有效增长模型需要描述特定产品的获客、激活、留存、商业化与增长循环如何相互作用。

模型应暴露当前约束，并说明哪个输入能够改变结果。只有框架术语、没有因果结构，会制造活动却不能形成诊断。

### 4. Marketplace 增长从“供给就是产品”开始

Marketplace 的客户体验取决于供给的可用性、质量、相关性与经济性。Winters 用“产品往往是供给，而不只是软件”概括这一点。

团队需要判断供给应当全面、独家还是精选，并确认当前约束位于需求还是供给。参与者总量不能证明局部或品类流动性。

### 5. Marketplace 扩张需要可重复的市场单元

只有能够在相邻地域或品类重复交付价值后，Marketplace 才适合扩张。扩张会改变密度、运营、参与者经济性与竞争行为。

需要回答：

1. 现有市场是否形成留存与健康经济性？
2. 新品类是否共享需求、供给或运营能力？
3. 能否在不永久补贴的情况下达到流动性？
4. 扩张会强化核心，还是分散注意力与库存？

### 6. 网络效应也会被夺走

网络效应不是永久资产。Winters 使用 Grubhub、DoorDash 与 Uber Eats 说明，竞争者可以改变网络结构、获得更难的一侧、改善物流，或者组合相邻价值。

网络壁垒需要通过参与者行为衡量：独特供给、Multi-homing、流动性、重复交易、服务质量与切换成本。历史规模本身不能保护网络。

### 7. SEO 是产品与供给系统

Pinterest 的方法不把 SEO 简化成关键词写作。团队可以使用结构化供给、UGC 或主动生产内容规模化创建 Landing Page，但每个页面都需要满足真实查询并交付有效价值。

```text
结构化供给或内容
-> 对应特定意图的可索引页面
-> 有用的未登录体验
-> 合格转化路径
-> 激活与留存用户
```

没有页面质量、意图匹配、激活和留存的流量，不是可持续获客。

### 8. 转化优化应从真实入口页面开始

Pinterest 用户经常通过 Pin、Board 与 Topic 页面进入产品，而不是首页。只优化首页会忽略大量可触达用户。

Winters 的案例把定量实验与定性研究结合。转化提升还要保护理解度、用户体验与下游留存。注册墙可能增加注册，却削弱信任或产品理解。

### 9. 可持续获客需要从 Kindle 进入 Fire

早期公司经常使用人工、不可规模化的“Kindle”策略找到留存客户分群。理解机制后，再用“Fire”策略建立可重复、可规模化获客。

PMF 前规模化会放大漏损，永远停留在人工阶段又无法产生复利。转换需要明确客户分群、留存价值、渠道与模式匹配，以及边际经济性能够承受规模的证据。

### 10. 增长团队随约束演进

Winters 描述增长组织如何从通用、集中式团队逐渐发展出专业化所有权。数据能力和决策权与组织图同样重要。

增长团队需要对有意义的结果负责，并掌握推动结果的杠杆。只能建议其他团队执行的增长团队缺少权限；拥有所有界面的团队又可能变成维护组织。

### 11. 产品领导力需要解释并推动判断

产品负责人需要说明不显眼的改进为何重要，在会议前降低决策风险，并把复杂推理表达得简单，但不能隐藏假设。

影响力不只是演示技巧，还需要客户证据、业务语境、取舍、对反对意见的预判，以及听众能够执行的决策。

### 12. AI 知识产品需要信任与同意

SuperMe 讨论如何从写作、对话等内容中表达专业判断。产品依赖来源、署名、审批、删改、反馈，以及默认取得同意。

让专业知识可搜索可能形成网络和数据优势，但前提是用户信任系统如何表达自己的观点。内容数量或病毒传播不能替代忠实、获得许可的知识。

## 运作总结

| 层级 | Winters 的方法 | 诊断问题 |
| --- | --- | --- |
| PMF | 按 Cohort 与分群阅读留存 | 谁持续获得价值，为什么？ |
| 新用户引导 | Setup、Aha、Habit | 首次体验是否为重复价值做好准备？ |
| 模型 | 连接机制与约束 | 哪个输入当前限制增长？ |
| Marketplace | 把供给作为产品的一部分 | 相关供给是否可用且经济性健康？ |
| 扩张 | 重复可运作市场单元 | 新市场会强化还是分散系统？ |
| SEO | 把有用结构化内容转化为入口页 | 搜索流量是否到达价值并留存？ |
| 转化 | 优化真实入口界面 | 更高转化是否保护理解与质量？ |
| 获客 | 从 Kindle 进入 Fire | 机制是否已经具备经济性规模化条件？ |
| 组织 | 让所有权匹配约束 | 团队能否执行自己负责的结果？ |
| AI 信任 | 保留同意与署名 | 专业表达是否忠实且可控制？ |

## 核心增长思想

### 1. 速度是到达已验证影响的时间

快速发布只有在工作到达用户并改变重要结果时才有意义。功能数量可能掩盖返工、缺陷和低价值产出。

### 2. 使用单线程目标

一个明确优先级可以保护团队免受分散请求。高目标能迫使团队寻找不同解法，但需要质量与客户护栏。

### 3. 提供语境，然后授权

团队理解战略、客户问题、约束和决策权时会更快。没有语境的授权使产品碎片化；没有权限的语境只会制造等待。

### 4. 用写作改善决策

书面战略和问题分析会暴露假设、支持异步反馈并保留决策。文档应加速行动，而不是成为仪式化规划。

### 5. 从第一性原理思考

品类惯例是输入，不是要求。复制竞品流程之前，团队应先推导用户问题与约束。

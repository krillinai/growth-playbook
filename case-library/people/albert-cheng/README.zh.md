## 与增长相关的工作

Cheng 对比三款大型消费产品的增长系统，而不是提供一套通用打法。他的框架区分对已知机会的优化和对新界面、新受众与新产品机制的探索。

当成熟实验体系很擅长优化现有漏斗，却不擅长寻找下一条增长曲线时，这个区分尤其有用。

## 核心增长思想

### 1. 平衡 Explore 与 Exploit

Exploit 利用积累的证据改善已知漏斗；Explore 测试新的商业化界面或产品循环等更大不确定性。局部优化终会收益递减，因此组合中需要同时存在两者。

### 2. 根据产品行为选择 Freemium 或 Trial

选择取决于价值出现速度、习惯形成方式、边际成本，以及免费使用能否创造分发。两种模式都不是普遍更优。

### 3. 区分留存与召回

让沉默用户重新回来可以贡献增长，但召回用户应该与连续留存 Cohort 分开衡量。混在一起会掩盖流失并扭曲产品健康度。

### 4. 扩大实验规模时保留战略空间

高实验量需要共用基础设施、明确决策规则和接受负面结果的文化。团队仍要为无法通过小型界面测试判断的大机会保留资源。

### 5. 围绕核心循环建立游戏化

游戏化要强化用户使用产品的根本原因。Cheng 区分核心循环、长期 Meta Game 和身份或 Profile。脱离真实价值的积分与连续记录，可能只创造活动而非长期留存。

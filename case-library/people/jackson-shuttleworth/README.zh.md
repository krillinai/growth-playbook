## 与增长相关的工作

Shuttleworth 负责 Duolingo 留存团队的产品工作，包括记录连续学习天数的 Streak 系统。他的叙述展示了一个简单计数器如何逐步发展为由目标、反馈、恢复、庆祝和提醒组成的系统。

这项工作的价值不是建议所有产品都添加 Streak，而是展示如何让留存机制与用户的重复目标对齐，再通过大量实验持续改善，同时避免机制取代底层学习行为。

## 核心增长思想

### 1. Streak 必须强化产品核心价值

当重复动作本身对用户有价值时，Streak 才有意义。在 Duolingo，机制让每日学习进展变得可见。如果被计数的行为太轻或与用户目标无关，计数器可能提高活动量，却不能改善留存质量。

### 2. 清晰度优先于复杂激励

用户需要理解什么行为可以延续 Streak、一天何时结束，以及自己处于什么状态。清晰状态、一致规则和即时反馈，可以先减少意外中断，再考虑更复杂的奖励与庆祝。

### 3. 恢复机制可以保护长期习惯

Streak Freeze 降低偶尔错过一天的损失，既保留已经积累的投入，也允许现实生活打断日常节奏。恢复机制需要限制，否则会使每日行为失去意义。

### 4. 实验整个系统，而不只是计数器

Duolingo 的 Streak 工作覆盖入口、视觉状态、里程碑、通知、恢复、庆祝和付费功能。Shuttleworth 讨论了超过 600 次相关实验。这个数量说明迭代深度，但不能证明实验数量本身会创造增长。

### 5. 通知是习惯循环的一部分

当时机、情境、文案和用户状态相互连接时，提醒更有效。增加通知可能提高短期回访，也可能造成疲劳或关闭权限，因此参与提升需要信任与通知健康度护栏。

### 6. 商业化不能破坏留存机制

恢复和便利功能可以提供付费价值，但在用户面临损失时收费可能形成胁迫感。商业化实验要保护理解、公平、学习行为与长期留存，而不能只优化即时付款。

### 7. 高频实验需要清晰的产品评审系统

实验速度依赖聚焦的责任边界、可复用基础设施、明确假设、快速评审和学习沉淀。团队仍需判断哪些界面值得持续优化，哪些问题需要更大的产品变化。

## Streak 系统摘要

```text
有价值的重复行为
→ 清晰的每日目标与状态
→ 即时进度反馈
→ 情境化提醒
→ 里程碑与身份
→ 中断后的有限恢复
→ 回到核心行为
```

## 证据与局限

- 产品历史和实验数量来自 Shuttleworth 的参与者叙述。
- 访谈没有提供完整实验设计、效应量、置信区间、Cohort 表或通知关闭数据。
- Duolingo 的学习频率、品牌、受众、产品规模和成熟实验基础设施限制了直接迁移。
- 高投入用户可能更愿意使用 Streak，因此 Streak 与留存可能存在选择相关性；因果贡献仍需通过对照实验和 Cohort 评估。

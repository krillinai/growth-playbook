## 案例摘要

Ramp 描述的方法不是给每位营销人员添加孤立 AI 工具，而是围绕共享 Agent 和名为 Project Glass 的内部系统，重组重复研究、内容、营销活动和运营工作。人继续负责目标、语境、创意判断、验证与协调。

```text
业务目标与可信语境
→ Agent 辅助研究或生产
→ 人工判断与验证
→ 通过产品、品牌与活动分发
→ 实验与业务衡量
→ 更新语境与工作流
```

## 运行原则

### 重构工作流，而不只是单项任务

自动化一个步骤可能把瓶颈移到其他位置。决定 Agent 放在哪里之前，团队应梳理输入、决策、交接、审核、分发和学习。

### 把语境当作基础设施

Agent 需要最新产品事实、受众定义、品牌规则、示例、表现数据与权限。检索质量、来源、访问控制和更新负责人决定可靠性。

### 保持人的结果责任

人定义问题、检查主张、处理歧义、批准品牌风险并负责结果。AI 输出数量不是营销结果。

### 超越末次点击衡量注意力

品牌、实体物品和线下活动可能影响需求，却不产生清晰末次点击路径。衡量可以组合对照、地区或受众实验、直接流量、品牌搜索、客户访谈、Pipeline 质量和长期 Cohort。

### 保留独特品牌判断

AI 让平均内容变得便宜。优势转向专有语境、有用洞察、创意品味、可信分发，以及竞争者无法用相同 Prompt 生成的体验。

## 可以迁移的方法

1. 梳理完整营销工作流及其瓶颈。
2. 定义可信语境、权限、负责人和更新规则。
3. 用测试数据集和人工审核自动化边界明确的步骤。
4. 衡量准确性、周期、成本、返工、品牌质量与下游结果。
5. 为重要决策保留人工回退与日志。
6. 对难以末次归因的渠道使用实验与客户证据。

## 证据与局限

- 当前证据主要来自 2026 年 6 月的一期参与者访谈。
- Project Glass 的架构、模型选择、成本、准确性、采用与业务提升未完整公开。
- 随模型与 Ramp 战略变化，组织方式可能快速演变。

## 案例摘要

Ramp 的产品速度模型从差异化经济承诺开始：帮助客户节省金钱与运营时间。团队获得语境与聚焦目标，用写作暴露假设，并跨越传统职能边界快速建设。

```text
差异化客户结果
→ 聚焦目标与约束
→ 语境与决策权
→ 快速建设与客户反馈
→ 质量、节省与控制检查
→ 扩大或修改
```

## 运行原则

### 让速度服务差异化

快速发布只有在强化独特客户结果时才有意义。广泛财务平台应从已验证问题扩张，而不是累积无关功能。

### 组合消费体验与企业控制

清晰与低摩擦可以形成 B2B 差异化，但财务流程还需要权限、审计、安全、可靠性和政策执行。

### 让团队聚焦一个重要结果

单线程目标降低协调成本并隔离分散请求。领导提供战略、客户语境、约束和决策权，而不是规定每项任务。

### 使用写作与第一性原理

书面思考让权衡可检查并支持异步决策。第一性原理帮助团队挑战品类惯例，同时保留真实监管与客户约束。

### 招聘互补的突出能力

当职位与协作匹配时，异常强项可以提高团队上限。人才密度不代表可以容忍伤害行为、薄弱文档或关键人风险。

## 可以迁移的方法

1. 定义让产品在经济上不同的客户结果。
2. 为每个团队设一个主要结果和明确护栏。
3. 共享客户、战略、技术与监管语境。
4. 在重大产品决策前写下假设。
5. 衡量到客户验证的周期，而不是发布数量。
6. 跟踪缺陷、信任、节省、采用和团队可持续性。

## 证据与局限

- 本案例整合高管与产品负责人的参与者叙述。
- 功能、收入、估值与增长主张不能证明速度造成结果。
- 金融产品控制和 Ramp 的人才市场限制直接迁移。

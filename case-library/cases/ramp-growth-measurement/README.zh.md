## 案例摘要

Ramp 描述的增长系统区分增长规模与增长质量。新客户和账户扩张分别衡量，回本周期与贡献毛利约束投入，增长工程连接产品和 GTM 系统，实验则根据是否产生决策进行评价。

```text
客户价值
→ 获客或扩张假设
→ 经济性与质量约束
→ 产品、数据与 GTM 执行
→ 结论性证据
→ 扩大、修改或停止
```

## 运行原则

### 区分增长动作

新客、激活、使用扩张、交叉销售和留存具有不同分母与时间周期。分开报告可以防止一项强动作掩盖另一项弱动作。

### 同时衡量规模与效率

缺少回本和贡献毛利的收入或客户增长，可能只消耗现金。效率指标还需要 Cohort 成熟度、共享成本规则和敏感性分析。

### 为增长工程定义清晰客户

增长工程可以建设产品界面、数据管道、销售工具与自动化。路线图应连接明确瓶颈和下游结果，而不是成为通用工单队列。

### 结论性失败

实验可以胜、负或没有显著差异。前两者可能产生学习；样本不足或埋点薄弱的测试通常不会。团队应预先定义假设、决策阈值、护栏与下一步行动。

### 使用组合与复盘

渐进优化和探索押注的组合平衡可靠性与发现。复盘把单次结果转化成目标、埋点、流程或战略变化。

## 可以迁移的方法

1. 分开获客、激活、扩张、留存和交叉销售 Cohort。
2. 为每个规模指标配置质量与经济性护栏。
3. 使用贡献毛利而不是总收入计算回本周期。
4. 在核心优化、相邻机会与探索之间分配资源。
5. 实验前写下决策标准。
6. 记录每项重要结果之后真正改变了什么。

## 证据与局限

- 本案例整合 2023 年和 2025 年的参与者叙述。
- 完整内部定义、Cohort、实验日志与经济数据未公开。
- 发布方关于高速增长的主张不作为因果验证。

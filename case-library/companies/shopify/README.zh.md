# Shopify

> 基于 Morgan Brown 担任 Shopify VP of Growth Marketing 时公开分享的商家增长案例。证据连接获客、留存、内容、社区、电商理解与增长系统执行。

**[English](README.md)｜[简体中文](README.zh.md)**

## 增长环境

Shopify 服务需要启动、运营并发展商业业务的商家。因此，商家获客只有在商家完成激活、持续经营、获得客户成功与长期业务价值后才完整。

两期 Morgan Brown 访谈通过他的职位和视角描述 Shopify，不是完整内部增长历史。

## 获客与留存

Brown 直接讨论了获客与留存的平衡。对商家平台而言，获客包括吸引创业者和现有企业，而留存取决于商家是否持续经营并获得价值。

```text
合格商家需求
-> 店铺设置与首次价值
-> 持续商家活动
-> 商家业务成功
-> 留存、扩展与倡导
```

这个顺序是诊断模型，公开资料没有提供所有内部定义或因果估计。

## 内容与社区

Brown 把内容与社区列为电商增长的可能起点。它们可以教育潜在商家、建立信任、发现客户语言，并在付费获客可重复前形成受众。

内容质量需要通过合格商家激活与留存验证，而不是只看发布量、粉丝或社区人数。

## 增长系统

“理解、识别、执行”框架形成纪律性顺序：

1. 理解商家问题、产品价值、分发与经济性。
2. 识别商家旅程中的最大约束。
3. 执行带明确结果和护栏的聚焦干预。
4. 记录学习，并更新产品、营销或流程。

相比具体 Shopify 策略，这个方法更容易迁移，因为它保留公司和市场上下文。

## 衡量问题

| 层级 | 核心问题 |
| --- | --- |
| 获客 | 哪些来源带来更可能获得价值的商家？ |
| 激活 | 哪个里程碑代表商家体验已经有效？ |
| 留存 | 商家 Cohort（同期群）是否持续经营并获得价值？ |
| 内容与社区 | 教育与社交输入是否创造增量合格商家？ |
| 经济性 | 商家增长能否支持持续获客和服务成本？ |
| 质量 | 支持、信任、商家结果与生态健康是否得到保护？ |

## 可以迁移的方法

- 把商家获客连接到后续成功，而不是注册量。
- 根据当前约束平衡获客与留存，而不是使用固定比例。
- 用内容与社区教育和学习，再验证下游价值。
- 使用框架或策略前先理解业务系统。
- 让产品和营销共同看到商家结果。

## 证据与局限

- 来源反映 Brown 当时的公开视角和职位，不是完整 Shopify 运作模型。
- Shopify 的结果来自众多团队、产品、合作伙伴、商家、市场条件与公司决策。
- 公开讨论没有提供完整指标定义、实验结果或因果归因。
- 商家分群、国家、业务成熟度与电商品类限制直接迁移。

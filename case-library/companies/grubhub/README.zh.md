# Grubhub

> 一个围绕餐厅供给、食客需求、配送网络演进、本地密度，以及网络效应如何被不同竞争模式取代的 Marketplace 案例。

**[English](README.md)｜[简体中文](README.zh.md)**

## Marketplace 背景

Grubhub 连接食客与餐厅。Marketplace 价值取决于相关餐厅选择、本地需求、下单可靠性，以及后来支撑配送的运营能力。

Casey Winters 把早期 Grubhub 经历与后续 Marketplace 分析连接起来。这些属于参与者视角，不构成完整公司历史。

## 供给与需求

餐厅库存是产品的一部分。全国餐厅总量很大，不能保证特定地区的食客找到相关且可用的选择。

```text
相关本地餐厅
-> 对食客有用的选择
-> 更多成功订单
-> 更强餐厅价值
-> 更密集的本地 Marketplace
```

每个转化都需要衡量。只有选择数量，没有可用性、订单质量、重复需求或可行餐厅经济性，不能闭合循环。

## 配送改变了网络结构

Winters 讨论了 Grubhub 增加司机应用，以及后来来自 DoorDash 和 Uber Eats 的竞争。配送引入了新的一侧和一套运营系统：司机、调度、时效、履约与服务质量。

竞争者可以通过控制配送供给和改善端到端交易，建立不同的网络效应。当竞争基础转向物流与履约时，餐厅聚合优势可能被削弱。

## 网络效应是有条件的

可用壁垒指标包括：

| 层级 | 示例信号 |
| --- | --- |
| 供给 | 相关餐厅覆盖与独家性 |
| 需求 | 重复食客与本地订单密度 |
| 流动性 | 在合理时间内完成的订单 |
| 履约 | 配送可用性、准确性与可靠性 |
| 经济性 | 餐厅、司机、平台与食客经济性 |
| 防御 | Multi-homing、切换、独特供给与服务差异 |

历史规模并不充分。如果供给同时使用多个平台、需求跟随更好的履约，或竞争者改变价值主张，网络就会减弱。

## 品类与地域扩张

扩张应复用真实优势，而不是只增加名义库存。新市场和品类需要兼容的需求、供给、运营、频率与经济性。

Winters 对 Grubhub 应采取其他战略的回顾不是受控反事实。它适合帮助提出战略问题，不能证明某个替代方案一定成功。

## 证据与局限

- 公开回顾压缩了多年产品、运营、竞争与组织变化。
- 与 DoorDash、Uber Eats 的对比不能隔离市场结果背后的全部因素。
- Grubhub 的结果还来自餐厅、食客、司机、团队、资本、监管、地域与时机。
- 替代战略属于假设，不是已经观察到的反事实结果。

## 案例 1：Facebook / Meta - 留存、增长核算与运营代理指标

**环境。** 早期 Facebook 增长依赖相关社交连接与重复使用。只看获客量会隐藏现有 Cohort 是否留存，以及流失是否抵消新用户。

**衡量系统。** 参与者讲述连接 Cohort 留存、月活跃人数、新增、留存、召回与流失用户，以及作为运营激活代理的早期好友连接行为。

```text
核心社交价值 -> 早期连接行为 -> Cohort 留存
-> 新增 + 召回 - 流失用户 -> 公司级增长方向
```

**保留的证据。** 团队研究留存曲线，从平滑的好友数关系中选择容易记住的运营点，改善新用户引导与好友发现，并偿还关键路径 Data Debt。代理指标创造运营清晰度，但没有建立因果阈值。

**迁移边界。** 月活、好友连接与自然频率都依赖产品和时期。Facebook 的网络效应、身份图谱、规模、埋点与社会外部性不能转化成通用指标 Benchmark。

## 案例 2：Ramp - B2B 增长规模与经济质量

**环境。** B2B 金融增长包括新客户、现有账户扩张、产品采用、销售、回本、贡献毛利与受监管运营成本。合并这些动作会让一项强结果掩盖另一项薄弱结果。

**衡量系统。** Ramp 参与者讲述区分获客与扩张，把规模与效率配对，在业务需要时使用不止一个价值指标，并把实验连接到经济与质量护栏。

```text
客户价值
-> 新客户或扩张动作
-> 激活与留存账户使用
-> 贡献毛利与回本
-> 扩大、修改或停止
```

**保留的证据。** 回本说明投入现金何时返回；贡献毛利检验扣除变动成本后增长是否创造经济价值；实验结论改变资源分配；增长工程连接产品与 GTM（市场进入策略）证据。

**迁移边界。** 公开材料没有提供完整成本定义、共享成本分配、Cohort 成熟度、销售周期处理、实验日志或账户级贡献。金融安全、合规与实施工作影响经济性。

## 案例 3：字节跳动 - 预测 LTV 与产品组合衡量

**环境。** 多产品、多市场公司需要跨渠道、国家、素材、产品以及不同商业化和留存周期分配获客与生命周期资源。

**衡量系统。** 徐洪亮描述预测 LTV 与多种 ROI 定义，而不是一个即时回本数字。共享系统连接预测、自动投放、素材学习、归因、风控、互导与本地市场执行。

```text
已观察 Cohort 行为 -> 按周期预测 LTV -> 预算与市场分配
-> 投放与素材证据 -> 留存价值与贡献
-> 模型校准、风险与产品组合决策
```

**保留的证据。** 长周期价值可以显示延迟回报，但不确定性随周期增加。模型需要回测、Cohort 校准、现金与止损限制、归因边界，以及跨产品蚕食检查。

**迁移边界。** 证据是跨产品和年份的回顾讲述。内部模型特征、校准误差、成本、因果提升、产品组合转移与作弊数据不可得。大型预测无法修复薄弱已观察 Cohort。

## 案例 4：Uber - 本地流动性与双边健康

**环境。** 司机与乘客需要在同一地点和时间出现，并满足可接受等待、价格、可靠性、信任与经济性。总注册数或全球供给可能隐藏破碎本地网络。

**衡量系统。** 公司页按城市或更小 Atomic Network 组织供给、流动性、交易质量、乘客与司机留存、激励、经济性与增量指标。

```text
相关本地供给 -> 匹配与接驾 -> 完成行程与质量
-> 乘客复用与司机回归 -> 利用率与贡献
-> 本地网络扩张决策
```

**保留的证据。** 有用指标包括可用供给、地域与时间覆盖、匹配或履约率、匹配时间、完成、取消、安全、支持、双边留存、利用率、激励成本与激励后行为。

**迁移边界。** 指标定义与优先级会随城市、参与者、产品时期、监管、劳动模式和频率变化。顶层 GMV（成交总额）或交易指标不能建立本地流动性或参与者健康。

## 案例 5：Airtable - 激活与协作使用衡量

**环境。** Airtable 可以支持大量工作流，因此新用户引导完成或通用行为不一定表示用户构建了有用内容。当相关协作者采用同一 Workspace 时，价值可能加深。

**衡量系统。** 客户任务与分群支持产品特定激活事件。早期工作流成功需要通过 Time to Value、留存使用、第四周多用户协作、邀请质量、错误、支持与放弃验证。

```text
客户任务 -> 首个有用工作流 -> 激活质量
-> 活跃协作者价值 -> 重复多用户工作流
-> 留存用户与账户价值
```

**保留的证据。** Lauryn Isford 区分完成界面与到达价值，并把第四周多用户协作描述为有意义下游信号。护栏保护灵活性、质量、单用户使用与后续留存。

**迁移边界。** 协作并非 Airtable 所有分群或产品的正确 North Star。报告引导提升缺少完整分母、指标定义、区间与长期复现，因此不是 Benchmark。

## 跨公司模式

### 实体决定指标

Facebook 衡量社交图谱内的人，Ramp 区分用户、账户与商业动作，字节跳动跨用户、产品、市场与产品组合运营，Uber 衡量本地网络内双方，Airtable 区分个人、Workspace 与账户采用。

混合这些实体会产生无效分母并隐藏约束。

### 结果、输入、诊断与护栏指标服务不同决策

```text
长期客户与业务结果
├── 团队能够改变的输入
├── 解释变化的诊断
└── 检测伤害的护栏
```

Facebook 好友连接、Ramp 回本、字节跳动预测特征、Uber 供给可用性与 Airtable 引导步骤，只有在角色明确并连接到结果时才有用。

### 已观察与预测指标必须分开

字节跳动最明显地展示该区别，但每个案例都包含模型或代理证据。Facebook 激活代理预测留存，Ramp 回本依赖未来贡献假设，Uber 本地流动性预测重复 Marketplace 价值，Airtable 早期工作流预测协作与留存。

预测或代理需要校准、不确定性与后续已观察结果。

### 聚合值隐藏本地失败

公司级增长可能隐藏 Facebook Cohort 流失、Ramp 薄弱扩张、字节跳动产品蚕食、Uber 破碎城市或 Airtable 非活跃 Workspace。只在拆分能够改变决策时分群，同时保留实体、Cohort 与成熟窗口。

## 迁移矩阵

| 产品条件 | 最相关案例 | 可以借鉴 | 不能假定 |
| --- | --- | --- | --- |
| 消费价值依赖重复网络使用 | Facebook / Meta | Cohort 留存、增长核算、运营代理、数据质量 | MAU（月活跃用户）或好友数阈值普遍正确 |
| B2B 增长包含获客与账户扩张 | Ramp | 分开动作，把规模与贡献、回本配对，使用决策护栏 | 一个收入指标代表产品和客户健康 |
| 资源分配依赖跨产品或市场的延迟价值 | 字节跳动 | 分开已观察与预测价值、校准周期、衡量产品组合增量 | 复杂 LTV 预测能够让不确定 Cohort 变安全 |
| 产品价值依赖本地双边交易 | Uber | 定义 Atomic Network、供给、流动性、质量、双边留存、经济性 | 总用户、GMV 或全球供给衡量本地健康 |
| 灵活协作软件需要首次价值定义 | Airtable | 把任务特定激活连接到协作、留存与护栏 | 引导完成或更多席位证明价值 |

## 证据与局限

- 公司证据主要来自参与者访谈、公开演讲、文章与重构公司页，而不是完整内部指标字典和数据集。
- 案例跨越不同产品、时期、实体、频率、商业模式、监管与证据质量。
- 完整定义、分母、窗口、排除、模型误差、成本分配、Cohort 与因果关系不可得。
- 公司结果还来自产品价值、市场条件、战略、资金、运营、领导层与衡量系统之外的多个团队。
- 收录用于记录衡量模式，不代表任何公司或人物认可。

# 增长模型案例

> 五个公司案例，展示客户价值、获客、激活、留存、商业化、传播、扩张与再投资如何形成不同的连接系统。

## 案例 1：Facebook - 连接社交价值、留存与扩张

**环境。** Facebook 需要从密集的早期网络向外扩张，同时保留让人返回的社交价值。

**保留的证据。** Chamath Palihapitiya、Alex Schultz 与 Naomi Gleit 把真实社交连接、好友发现、激活代理、Cohort 留存、增长核算、数据埋点、实验与国际化连接起来。他们的讲述是参与者视角，不是完整公司史。

```text
相关社交连接
-> 早期网络价值
-> 重复沟通与内容
-> 留存活跃用户
-> 口碑、学习与扩张
-> 更多可触达网络
```

**可以支持的增长模型结论。** 早期行为只有明确连接到留存价值与运营决策时才有用。数据与实验强化学习循环，国际化则在核心价值可见后移除扩张障碍。

**缺失证据与迁移边界。** 好友阈值位于平滑留存曲线上，不能证明因果。Facebook 的实名网络、规模、数据、资本、时机与组织都很特殊，增长不能归因于单个人、团队或指标。

## 案例 2：HubSpot - 市场改变时重建模型

**环境。** HubSpot Sales 最初面向个人专业人士，以低摩擦产品、免费套餐、低价订阅、病毒传播与付费获客支持增长。留存证据不能保证它达到预期业务规模。

**保留的证据。** Brian Balfour 描述 Four Fits、趋平留存、市场规模计算、Channel-Model Fit、一个位于两种可行获客动作之间的价格层级，以及后来围绕 HubSpot Mid-market 客户协调修改产品、价格、渠道、销售与合同。

```text
明确市场与问题
-> 低摩擦产品价值
-> 病毒与付费分发
-> 留存使用与收入
-> 市场与经济约束
-> 修改客户、产品、渠道与模式
```

**可以支持的增长模型结论。** 当目标市场或进入市场路径改变时，必须重建模型。产品留存、价格、渠道成本、市场容量、销售动作与收入模式是相互依赖的假设。

**缺失证据与迁移边界。** 记录没有完整市场研究、定价实验、留存经济性、销售数据或反事实路径。HubSpot 后续表现不能归因于单个框架或再定位决定。

## 案例 3：Dropbox - 让分发与产品价值一致

**环境。** Dropbox 的早期产品让文件跨设备同步与存储变得有用。分享与存储奖励能够创造分发，是因为产品产物与激励都连接到这种价值。

**保留的证据。** Sean Ellis 讨论 Must-have Value、激活、留存与推荐；Elena Verna 把产品使用连接到获客与扩张；Andrew Chen 对比产品一致分发与早期付费搜索经济性；Morgan Brown 补充 Dropbox Dash 与显式 AI 产品评估的后期证据。

```text
有用文件或工作结果
-> 分享、协作或存储需求
-> 合格接收者或同事
-> 激活与留存使用
-> 推荐、团队扩张或收入
-> 产品与分发学习
```

**可以支持的增长模型结论。** 只有当产品产物创造合格下一输入，并让接收者获得留存价值时，产品驱动循环才闭合。付费与推荐渠道应该通过边际经济性与留存质量比较，而不是通过偏好标签比较。

**缺失证据与迁移边界。** 证据跨越早期 Dropbox、后来的产品驱动工作与 Dash 时期。公开案例简化了推荐增量、作弊、Cohort 质量、扩张权限、隐私与 AI 评估，不能组成一条经过验证的因果顺序。

## 案例 4：Duolingo - 连接可达性、习惯、收入与再投资

**环境。** Duolingo 的模式组合免费学习产品、重复练习、订阅、广告、国际触达、品牌分发与持续产品实验。

**保留的证据。** Luis von Ahn 连接教育可达性、Freemium、动机、订阅、广告、品牌与产品投资。Gina Gotthilf、Albert Cheng 和 Jackson Shuttleworth 补充自然触达、国际化、订阅杠杆、Explore-Exploit 分配、留存、召回、连续学习与实验。

```text
可达学习价值
-> 广泛自然与国际触达
-> 重复练习与留存习惯
-> 订阅或广告价值
-> 实验与产品再投资
-> 改善触达与学习体验
```

**可以支持的增长模型结论。** 当免费使用、习惯、商业化与品牌受到重复学习价值约束时，它们可以相互强化。收入能够支持下一轮产品建设，但过度商业化也可能削弱可达性或信任。

**缺失证据与迁移边界。** 公开证据无法分离学习结果、品牌、通知、连续学习、定价、广告或市场扩张。消费教育行为与 Duolingo 的品牌许可度不能自动迁移到其他产品。

## 案例 5：Uber - 建立本地跨边流动性

**环境。** Uber 的客户价值依赖相关司机与乘客在相同地域和时间窗口相遇。Marketplace 总注册量不能保证可靠行程。

**保留的证据。** Andrew Chen 描述司机供给、接驾时间、覆盖、履约、利用率、激励、Atomic Network、城市扩张与 Marketplace 经济性。他的文章也解释较低频的“Uber for X”品类为什么无法复制网约车供给经济性。

```text
相关司机可用性
-> 更短接驾与更好履约
-> 乘客价值与重复需求
-> 利用率与收入机会
-> 留存供给与本地扩张
```

**可以支持的增长模型结论。** Marketplace 模型必须在参与者真正获得价值的本地单位上定义。每个跨边转化都需要质量、留存、增量与经济性，汇总规模不能证明网络效应。

**缺失证据与迁移边界。** 来源没有完整城市 Cohort、激励 Holdout、贡献数据、安全结果或监管影响。Hard Side 与活跃约束会随市场和时期变化。

## 跨公司模式

### 价值与留存锚定模型

Facebook 用社交连接代理留存社交价值；HubSpot 用趋平留存作为一项匹配信号；Dropbox 依赖有用同步与分享；Duolingo 依赖重复学习；Uber 依赖可靠本地交易。没有留存价值，分发无法闭合模型。

### 产出创造不同的下一输入

Facebook 可以创造社交邀请与学习，HubSpot 可以再投资收入与渠道知识，Dropbox 可以创造接收者与团队采用，Duolingo 可以再投资订阅与广告价值，Uber 可以改善跨边密度。不是每个产出都是循环，每个转化都需要证据。

### 经济性可以逆转看似有前景的机制

HubSpot 的定价与渠道匹配、Dropbox 的早期付费搜索比较、Duolingo 的可达性与商业化权衡、Uber 的激励与利用率经济性，都说明规模不能证明模型可扩张。

### 模型随增长单位改变

相关单位可以是活跃个人、账户、团队、市场层级、学习者、订阅、城市、乘客或司机。改变单位会改变资格、价值、渠道、留存、商业化与扩张假设。

## 迁移矩阵

| 产品条件 | 最相关案例 | 可以借鉴 | 不能假定 |
| --- | --- | --- | --- |
| 消费网络需要清晰激活模型 | Facebook | 从留存行为推导早期代理，并投资数据与学习 | 著名阈值或 MAU 是正确指标 |
| SaaS 产品有匹配但规模经济薄弱 | HubSpot | 一起测试产品、市场、渠道、价格、销售动作与市场容量 | 定价变化或进入高端市场可以被孤立处理 |
| 产品使用能够暴露或邀请他人 | Dropbox | 衡量接收者价值、激活、留存、奖励成本与扩张质量 | 分享自动创造增量病毒传播 |
| Freemium 消费产品需要可持续收入 | Duolingo | 用学习护栏连接可达性、重复价值、订阅、品牌与再投资 | 更多参与或自然关注证明长期价值 |
| Marketplace 需要解决冷启动 | Uber | 定义 Atomic Network、Hard Side、本地流动性、质量与参与者经济性 | 汇总供给或永久激励创造网络效应 |

## 证据与局限

- 模型是基于参与者讲述、文章、演讲与公司页的编辑重构，不是经过验证的内部系统图。
- 五个案例来自不同时期、产品、市场结构与结果定义；除非明确有更强证据，箭头只代表假设。
- 公司结果还来自产品质量、资本、市场时机、竞争、监管、运营与这里没有覆盖的团队。
- 模型用于暴露假设和选择证据；逻辑一致本身不能证明假设。
- 收录用于记录可迁移证据，不代表公司或参与者认可。

## 案例 1：Facebook / Meta——Magic Moment 与网络价值

早期 Facebook 用户可以注册，却没有获得社交价值。激活假设关注帮助用户形成足够多相关连接，让网络变得有用。

```text
新用户与社交意图
-> 发现相关人物
-> 形成有意义连接
-> 获得重复社交价值
-> 留存并加深网络参与
```

留存用户分析帮助识别候选早期行为，埋点与实验支持执行。好友阈值是从留存关系中选择的运营目标，不是产品价值本身。

**主要边界：**相关性不能证明连接数量本身造成留存。产品设计、相关性、网络密度、内容、市场与历史环境都很重要。

## 案例 2：Pinterest——从入口界面到 Setup、Aha 与 Habit

Pinterest 用户经常从单个 Pin、Board 和 Topic 进入，而不是主页。因此，激活从实际入口界面和访问者发现意图开始。

```text
相关未登录内容
-> 理解产品与用途
-> Setup 兴趣与上下文
-> 通过有用发现或收藏到达 Aha
-> 通过重复相关发现形成 Habit
```

案例把定量转化工作连接到定性研究、理解、用户满意、激活与留存。Signup Wall 可以增加注册，却降低信任或产品理解。

**主要边界：**Pinterest 的视觉内容供给、SEO（搜索引擎优化）入口模式、个性化与消费节奏不同于企业、交易和低频产品。

## 案例 3：Airtable——意图、模板与协作价值

Airtable 的灵活性意味着用户带着不同任务进入，也可能面对高投入空白状态。激活问题是帮助用户创建有用工作流，而不是强迫所有用途经过同一条配置路径。

```text
客户意图
-> 相关模板、示例或 Setup 路径
-> 第一个有用工作流或成果
-> 协作或 Workspace 重复使用
-> 留存账户价值
```

模板、默认值、渐进 Setup 与分群引导可以降低 Time to Value。后续协作可能代表更强价值，但团队需要检验协作造成留存，还是只识别高意图账户。

**主要边界：**一个模板或协作阈值无法代表 Airtable 不同工作流、角色、账户规模与自然频率。

## 案例 4：Duolingo——习惯前的首次学习价值

Duolingo 激活必须从有意义学习开始，而不是 App 活动。可及课程、即时反馈、可见进度与重复练习帮助用户理解产品价值并建立下一项自然行动。

```text
学习意图
-> 完成有意义首次课程
-> 获得反馈与可见进度
-> 理解下一项学习行动
-> 重复练习并留存
```

Streak、提醒、League 与恢复属于价值之后的强化。如果用户只优化积分而没有学习，活动增加，激活质量却下降。

**主要边界：**Duolingo 是高频消费学习产品，拥有特殊使命、品牌与实验规模，其激活节奏不能直接迁移到低频或高风险产品。

## 案例 5：Dropbox——Must-have 效用与已存价值

Dropbox 早期产品价值围绕可靠文件同步与存储。激活要求用户体验该效用，而不只是安装产品或创建账户。

```text
文件或连续性需求
-> 存储或同步真实文件
-> 可靠检索或访问
-> 累积已存价值
-> 分享、协作或重复使用
```

用户理解 Must-have 价值后，推荐循环才合理。额外存储与产品价值一致，但完成邀请或奖励仍位于接收者激活与留存使用上游。

**主要边界：**公开材料组合早期 Dropbox、后续 PLG（产品驱动增长）与 AI 产品时期，不是完整内部激活历史。

## 跨公司模式

### 首次价值由产品决定

- Facebook 需要相关网络价值。
- Pinterest 需要有用发现与产品理解。
- Airtable 需要有用工作流或成果。
- Duolingo 需要有意义学习。
- Dropbox 需要可靠文件效用。

注册、完成向导、邀请、生成内容或发布供给不能替代客户结果。

### Setup 可以支持或延迟价值

| 产品模式 | 必要 Setup | 主要风险 |
| --- | --- | --- |
| 社交网络 | 身份与相关连接 | 收集连接却缺少相关性 |
| 发现产品 | 兴趣、上下文与有用内容 | 理解前出现 Signup Wall |
| 灵活 Workspace | 数据、模板、结构与工作流 | 空状态投入或过度配置 |
| 学习产品 | 水平、课程上下文与可及行动 | Setup 或游戏机制替代学习 |
| 已存价值工具 | 真实文件、设备、权限与可靠性 | 技术完成却无法成功检索 |

只有结果质量与下游留存不受损时，删除 Setup 才有价值。

### 激活是对留存的假设

每个案例都识别早期事件，并要求后续验证。候选行为可以预测留存，是因为它创造价值，也可能只是高意图用户更容易执行。实验与路径分析帮助区分两种解释。

### 使用正确实体与入口状态

Meta 与 Pinterest 主要观察个人消费者；Airtable 与 Dropbox 需要用户、协作者、Workspace 与账户视角；Duolingo 需要学习者与订阅状态。受邀、自主进入、回访与辅助客户需要不同资格规则。

## 迁移矩阵

| 情况 | 优先建设 | 避免 |
| --- | --- | --- |
| 注册高但首次价值弱 | 承诺匹配、入口意图、前置条件、错误与客户路径 | 理解价值前继续增加获客 |
| 灵活或空状态产品 | 意图、模板、示例、导入、默认值与渐进 Setup | 所有用途使用同一通用向导 |
| 网络或 Marketplace | 相关供给或连接、互动成功、质量与信任 | 把供给、连接或注册数当成价值 |
| 习惯产品 | 有意义首次行动、反馈、下一次机会与质量 | 价值出现前增加 Streak |
| B2B（企业对企业）协作产品 | 用户、Workspace、账户、购买者证明与独立重复价值 | 把用户激活当成唯一成功层级 |
| AI 产品 | 成功任务、质量、验证、重试、成本与可重复性 | 把 Prompt 或生成产物数量当成激活 |

## 证据与局限

- 证据主要来自参与者访谈、演讲与实践者分析，不是完整内部激活实验和 Cohort 数据。
- 实践跨越不同时期，不代表当前所有 Onboarding 系统。

- 发布方增长、用户、收入、客户或实验量主张不用于因果证明。
- 公司结果不能归因于某个激活事件、流程、团队或实践者。

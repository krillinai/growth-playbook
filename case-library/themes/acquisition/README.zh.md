# 获客

## 案例 1：Pinterest - 把 SEO 入口页做成产品系统

**环境。** 潜在用户通常通过单个 Pin、Board 和主题页发现 Pinterest，而不是从首页进入。因此，搜索获客直接进入未登录内容界面。

**机制。** 结构化内容供给与 UGC（用户生成内容）围绕具体意图聚合。增长工作把真实搜索入口页连接到产品理解、合格注册路径、Setup、首次价值与回访行为。

```text
搜索意图 -> 相关结构化页面 -> 产品理解
-> 合格注册 -> 激活 -> 留存
```

**保留的证据。** 页面数量与流量只是中间信号。公司页记录了真实入口界面的测试、Gate 与理解之间的取舍、Setup-Aha-Habit 模型，以及不同市场的内容差异。

**迁移边界。** 搜索政策、内容质量、品牌、当前索引行为与产品未登录价值决定系统能否迁移。历史 Modal 或 Gate 方法不是建议。

## 案例 2：Airbnb - 产品化接入已有需求

**环境。** Airbnb 需要把新房源供给连接到已经寻找住宿的人群。Andrew Chen 的历史分析描述了帮助符合条件的房东把房源信息发布到 Craigslist 的流程。

**机制。** 分发被嵌入房东工作流。产品减少跨发工作，适配外部发布结构，并建立从外部发现回到专用产品和交易路径的通道。

```text
房东房源 -> 辅助外部发布 -> 合格房客发现
-> Airbnb 落地与信任路径 -> 咨询或预订 -> 留存 Marketplace 价值
```

**保留的证据。** 有用的衡量单位不是跨发数量，而是从符合条件的房源到发布、回流质量、交易与参与者持续价值的完整顺序。产品、工程与渠道策略共同构成机制。

**迁移边界。** 该分析没有提供 Airbnb 增长的因果估计。任何集成都必须检查当前授权、平台规则、用户许可、数据处理、可靠性、归因与品牌风险。

## 案例 3：字节跳动 - 把付费获客做成共享基础设施

**环境。** 多产品、多市场的产品组合需要跨渠道、素材、国家和不同价值周期分配获客资源。

**机制。** 已记录系统连接预测 LTV、多种 ROI（投资回报率）周期、自动投放、素材标注与生产、归因、风控、共享服务、产品组合分配和本地执行。

```text
留存产品价值 -> 价值预测 -> 预算与市场分配
-> 自动投放与素材学习 -> 归因与风控
-> 本地执行 -> 更新 Cohort 与模型
```

**保留的证据。** 付费获客被当作有边界的决策系统，而不是媒体采购看板。长周期模型需要回测和现金限制；自动化需要稳定目标、日志、覆盖与人工调查；素材产量需要假设质量与下游衡量。

**迁移边界。** 案例主要来自前参与者对多年、多产品工作的回顾。数据量、资金、推荐质量、市场时机、监管与内部能力无法从运营系统中剥离。

## 案例 4：Duolingo - 自然触达、品牌与国际化

**环境。** Duolingo 把广泛可访问的学习产品与国际触达、鲜明品牌表达、社媒参与、订阅商业化和习惯型产品系统组合起来。

**机制。** 产品效用和公开沟通创造自然发现。国际化扩大可触达受众，而激活、学习留存、订阅转化和市场级经济性检验注意力能否成为长期价值。

```text
可访问学习价值 -> 自然与社媒发现 -> 市场进入
-> 学习激活 -> 习惯与回访 -> 订阅质量
```

**保留的证据。** 自然触达需要连接到合格访问、首次学习价值、留存行为、付费转化与品牌护栏。市场扩张包括需求、本地渠道、文化、支付条件、设备与运营重点，而不只是文字翻译。

**迁移边界。** 可识别品牌或社媒账号不能替代产品价值。Duolingo 的使命、品类、使用频率、Freemium 模式、内容供给和国际基础共同塑造机制。

## 案例 5：Lovable - 让赢得式分发越过发布注意力

**环境。** 一个快速变化的 AI 产品处于公开关注强烈、功能变化迅速、客户产出可见，同时长期渠道与推理经济性仍不确定的品类。

**机制。** 创始人和员工参与、产品演示、客户证明、发布与公开学习可以创造赢得式分发。只有注意力进入可信产品体验并继续走向激活和留存使用时，路径才成立。

```text
公开参与与证明 -> 合格注意力 -> 产品体验
-> 成功任务 -> 留存工作流 -> 新的赢得式证据
```

**保留的证据。** 公司证据支持 earned-first 分发、信任、发布节奏、直接客户接触与把品牌视为产品责任。随着渠道成熟，还需要在内容、社区、集成、合作伙伴、SEO 与生成式发现之间建立更广组合。

**迁移边界。** 创始人影响力、员工可信度、品类新鲜度、客户热情与发布时机不能被假定。公司报告增长不用于证明某一项策略造成结果。

## 跨公司模式

### 获客质量在下游成熟

五个案例都要求越过响应读取结果。Pinterest 搜索流量必须激活与留存；Airbnb 回流必须创造健康交易；字节跳动投入必须产生留存贡献；Duolingo 注意力必须成为学习价值；Lovable 发布必须越过新鲜度。

```text
触达 -> 合格到达 -> 激活 -> 留存 -> 贡献
```

成熟窗口与衡量实体不同，但没有任何案例支持把渠道报告转化当成最终结果。

### 分发可以成为产品能力

Pinterest 创建有用的可索引界面，Airbnb 把渠道入口嵌入房东工作流，字节跳动建设可复用的投放与素材服务。渠道不一定只归营销团队所有；产品、工程、数据、内容、运营与风控都可能成为机制的一部分。

### 最强机制也带有依赖

搜索依赖搜索系统和内容质量；Airbnb 的历史流程依赖外部 Marketplace；付费获客依赖竞价、平台、数据与素材供给；赢得式分发依赖自愿信任与注意力。每种机制都需要明确依赖与恢复方案。

### 国际规模需要本地证据

Pinterest、字节跳动和 Duolingo 都说明语言只是扩张的一部分。供给、需求、渠道、监管、文化、产品理解、支付和团队权力都可能按市场变化。

## 迁移矩阵

| 产品条件 | 最相关案例 | 可以借鉴 | 不能假定 |
| --- | --- | --- | --- |
| 结构化内容能够满足大量具体意图 | Pinterest | 意图分类、真实入口页优化、下游质量 | 更多页面或更强 Gate 会产生长期搜索增长 |
| 产品参与者能够触达匹配的外部需求 | Airbnb | 产品化授权分发并衡量交易质量 | 平台套利长期可用或符合规则 |
| 高投入和高素材量带来运营复杂度 | 字节跳动 | 多周期价值、有边界自动化、素材学习、风控 | 长周期预测或规模能让弱 Cohort 变健康 |
| 消费价值可以通过身份与公开参与传播 | Duolingo | 把自然触达连接到习惯、市场准备和订阅质量 | 品牌注意力证明学习或商业健康 |
| 可见产品可以通过证明与发布赢得注意力 | Lovable | Earned-first 参与、演示、信任、渠道组合 | 新鲜度、创始人影响力或报告增长能够复制 |

## 证据与局限

- 公司证据主要来自参与者访谈、公开演讲和外部分析，而不是完整内部因果历史。
- 案例覆盖不同年代、产品、市场、商业模式与证据质量，不能按照公司报告结果排名。
- 每个案例都缺少完整增量、成本、Cohort、实验与反事实数据。
- 公司整体结果来自产品价值、市场条件、资金、竞争、时机、多个团队和未在此记录的机制。
- 收录用于描述运营模式，不代表认可绕过平台规则、激进 Gate、不可持续投入或任何单项公司决策。

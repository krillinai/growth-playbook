## 案例 1：Airtable - 灵活产品的意图路径

**环境。** Airtable 可以支持大量工作流。创造价值的产品灵活性也会削弱单一新用户引导路径：用户带着不同目标、经验、起始资料与学习方式进入。

**保留的证据。**

    留存超级用户工作流
    -> 任务、语言与起始条件
    -> 简短意图问题
    -> 相关模板或引导路径
    -> 首个有用工作流
    -> 后续协作与留存

参与者讲述把客户成功反馈和超级用户研究连接到定位、模板与分群引导。只有答案能改变推荐路径并改善下游价值时，问题才有用。

**迁移边界。** 报告的新用户引导结果没有提供完整分群定义、分配规则、样本量、置信区间或各路径长期效果。协作可以表示更深采用，但不是所有个人工作流的正确结果。

## 案例 2：HubSpot - 分群选择改变完整增长系统

**环境。** HubSpot Sales 最初聚焦使用低摩擦 Freemium 模式的一线销售人员。留存价值存在，但市场计算与渠道经济性限制了目标规模。

**保留的证据。**

    一线销售用户
    -> 重复邮件追踪价值
    -> 相邻的关系型专业人士
    -> 市场与支付意愿证据
    -> 广泛低 ARPU 路径或 HubSpot Mid-market 路径
    -> 协调改变产品、价格、渠道、销售与合同

Balfour 的讲述区分了用户角色与账户级目标市场。选择 HubSpot 已有的 Mid-market 客户不只是改变企业属性标签；产品、套餐、获客动作、销售流程与经济性需要一起变化。

**迁移边界。** 证据是对 2014 至 2016 年的回顾，并使用近似数字。完整分群 Cohort、客户研究样本、转化、服务成本与反事实结果不可得。

## 案例 3：Lovable - 区分 ICP 信号与 Hobbyist Noise

**环境。** AI 产品可以吸引广泛好奇、试验与社交关注。这些信号可能淹没真正拥有重复问题、组织意图、支付意愿与长期使用路径的客户证据。

**保留的证据。**

    广泛 AI 兴趣
    -> 产品与账户行为
    -> 重复有价值工作流
    -> 商业意图与准备度
    -> Product-Led Sales 或 Self-serve 路径
    -> 留存使用与商业化

Verna 的参与者视角强调区分 ICP 信号与 Hobbyist Noise，并把产品行为连接到激活、留存、商业化与销售。公开证明与自然分发应吸引相关需求，而不是只优化关注。

**迁移边界。** 公开材料没有披露完整 ICP 定义、账户资格、行为阈值、路由规则、PLS 转化、分群留存或服务成本。Hobbyist 仍可能创造产品学习或分发价值，不能被统一视为低质量。

## 案例 4：字节跳动 - 市场、设备与产品组合条件

**环境。** 跨国家与产品组合运营的消费产品会面对不同设备、渠道、支付条件、监管、竞争、价格敏感度与用户动机。

**保留的证据。**

    市场与设备条件
    -> 目标分群与受限行为
    -> 主产品、Lite 产品或本地化执行
    -> 资格、素材、渠道与激励差异
    -> 留存价值与经济性
    -> 产品组合蚕食与风险检查

徐洪亮的讲述描述本地团队调整优先级与执行，Lite 产品触达低配置设备或价格敏感群体，以及使用人群隔离降低主产品用户被补贴迁移的风险。只有客户条件与体验差异足以抵消重复运营时，独立路径才合理。

**迁移边界。** 证据跨越多个产品、市场与年份，缺少完整分群规则、本地 Cohort、成本或对照比较。设备、收入与地区可能成为敏感代理，分群需要隐私、公平与监管审查。

## 案例 5：Duolingo - 按市场、状态与意图划分消费用户

**环境。** Duolingo 组合广泛免费使用、国际市场、学习行为、订阅、留存、召回与广告。一个公司级转化或活跃数字可能隐藏不同客户状态与市场条件。

**保留的证据。**

    市场 + 渠道 + 学习意图 + 产品状态
    -> 合格获客与激活
    -> 留存学习行为
    -> 免费、付费、留存、召回或流失状态
    -> 订阅价值与服务经济性

公司案例建议按市场、渠道、免费与付费状态、学习意图划分获客、激活、转化、留存与收入。国际化也需要本地需求、文化、支付、设备与运营证据，而不只是翻译。

**迁移边界。** 该案例是对不同时期和职能参与者讲述的编辑综合。完整分群定义、本地 Cohort、学习结果、Paywall 处理效果、归因与订阅经济性不可得。

## 跨公司模式

### 分群必须改变决策

Airtable 改变新用户引导，HubSpot 改变完整市场与 GTM 系统，Lovable 改变资格与销售路由，字节跳动改变产品和本地执行，Duolingo 改变市场、生命周期与商业化处理。

描述性切片只有在改变决策后，才成为运营分群。

### 单位、角色与状态必须分开

HubSpot 区分个人用户与目标账户，Airtable 区分用户意图与 Workspace 采用，Lovable 连接产品行为与账户准备度，字节跳动跨用户、产品与市场运营，Duolingo 区分生命周期和付费状态。

混合这些层级会产生无效 ICP 与误导性分母。

### 识别必须发生在结果之前

留存与扩张可以在事后揭示强分群，但路由需要处理前可获得的信息。早期意图、工作流、账户、市场、设备或行为信号，需要通过后续价值验证。

### 扩张会重新打开分群问题

进入更高市场层级、进入新国家、创建 Lite 产品、改变免费与付费设计，或增加新 AI 客户类型，都会改变价值实现与获取的条件。

## 迁移矩阵

| 产品条件 | 最相关案例 | 可以借鉴 | 不能假定 |
| --- | --- | --- | --- |
| 灵活产品服务大量工作流 | Airtable | 只询问能改变路径的意图问题，并通过下游价值验证 | 更多个性化问题或协作一定改善激活 |
| B2B 产品正在考虑市场层级迁移 | HubSpot | 围绕目标分群重建产品、价格、渠道、销售与经济性 | 只用公司规模定义新 ICP |
| AI 产品关注度广泛，但商业需求不确定 | Lovable | 路由前组合产品行为、账户准备度、留存与商业化 | Hobbyist 没有价值，或一个使用阈值证明 Sales-ready |
| 消费产品组合跨设备与市场 | 字节跳动 | 创建独立路径前使用本地条件、资格与产品组合检查 | 历史 Lite 产品或激励策略可以直接迁移 |
| Freemium 消费产品跨市场与生命周期状态 | Duolingo | 按市场、渠道、意图、免费付费状态与留存行为分群 | 一个全球 Paywall、节奏或学习模式适合所有分群 |

## 证据与局限

- 公司证据主要来自参与者访谈、演讲、文章与重构公司页，而不是完整内部分群字典。
- 案例跨越不同产品、时期、单位、市场、商业模式与证据质量。
- 识别规则、重叠、未知比例、Confusion Matrix、分群 Cohort、经济性、实验与失败路径不完整。
- 公司结果还来自产品价值、市场条件、竞争、资金、运营、领导层与分群之外的多个团队。
- 收录用于记录分群模式，不代表任何公司或人物认可。

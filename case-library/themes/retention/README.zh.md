# 留存：跨公司案例

> 留存衡量用户是否在产品自然使用周期内持续获得价值，并区分连续留存、召回、扩张与流失行为。

**[English](README.md)｜[简体中文](README.zh.md)**

关于留存模型、Cohort 指标与诊断，参见主手册[留存](../../../handbook/retention/README.zh.md)。

## 公司证据

| 公司 | 重复价值 | 强化机制 | 验证与边界 |
| --- | --- | --- | --- |
| [Duolingo](../../companies/duolingo/README.zh.md) | 持续学习进步 | Streak、恢复、通知、身份 | 学习行为与召回；机制不能替代学习价值 |
| [Facebook / Meta](../../companies/facebook/README.zh.md) | 重复的相关社交连接 | 更密集网络与持续社交内容 | Cohort 留存与增长核算；网络效应限制迁移 |
| [Airtable](../../companies/airtable/README.zh.md) | 持续存在于工作中的有用流程 | 成果、协作者与团队重复使用 | 后续协作与工作区行为；频率随用途变化 |

## 对比与迁移

Duolingo 强化个人习惯，Facebook 复合网络价值，Airtable 把工作流嵌入团队运营。可见机制不同，但都需要把回访连接到重复价值，而不是只依赖提醒。

迁移前需要确认：重复价值是习惯、网络、工作流还是低频任务；没有通知和奖励时用户为何回来；协作是否真正深化价值；召回是否与连续留存分开报告。

## 证据与局限

- 公司证据主要来自参与者访谈和公开材料，不是完整内部因果评估。
- 公司规模、市场、产品频率、监管与数据能力限制直接迁移。
- 发布方公布的增长与收入数字不用于证明单项方法有效。

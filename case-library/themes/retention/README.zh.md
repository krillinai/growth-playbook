# 留存

> 留存衡量用户是否在产品自然使用周期内持续获得价值，并区分连续留存、召回、扩张与流失行为。

**[English](README.md)｜[简体中文](README.zh.md)**

关于留存模型、Cohort 指标与诊断，参见主手册[留存](../../../handbook/retention/README.zh.md)。

## 公司证据

| 公司 | 重复价值 | 强化机制 | 验证与边界 |
| --- | --- | --- | --- |
| [Duolingo](../../companies/duolingo/README.zh.md) | 持续学习进步 | Streak、恢复、通知、身份 | 学习行为与召回；机制不能替代学习价值 |
| [Facebook / Meta](../../companies/facebook/README.zh.md) | 重复的相关社交连接 | 更密集网络与持续社交内容 | Cohort 留存与增长核算；网络效应限制迁移 |
| [Airtable](../../companies/airtable/README.zh.md) | 持续存在于工作中的有用流程 | 成果、协作者与团队重复使用 | 后续协作与工作区行为；频率随用途变化 |
| [Shopify](../../companies/shopify/README.zh.md) | 商家持续成功与平台价值 | 产品能力、教育、社区与生命周期支持 | 商家持续使用和业务结果；获客量本身不充分 |
| [Dropbox](../../companies/dropbox/README.zh.md) | 文件价值、协作和 AI 辅助知识获取 | 已存资产、协作、扩展和重复成功检索 | Cohort 使用与任务质量；上线 AI 功能不等于形成留存 |
| [Pinterest](../../companies/pinterest/README.zh.md) | 重复发现并收藏相关灵感 | 个性化内容、兴趣与重复用途 | Setup 和 Aha 后的 Cohort 留存；搜索转化本身不充分 |
| [HubSpot](../../companies/hubspot/README.zh.md) | 了解潜在客户何时与被追踪沟通互动所产生的重复价值 | 快速反馈与持续外部销售活动 | 早期 HubSpot Sales 回顾中的平稳留存曲线；具体内部 Cohort 未公开 |

## 对比

Duolingo 强化个人习惯，Facebook 复合网络价值，Airtable 把工作流嵌入团队运营，Shopify 依赖商家持续成功，Dropbox 连接已存价值与协作，Pinterest 依赖重复且相关的发现。可见机制不同，但都需要把回访连接到重复价值，而不是只依赖提醒。

## 迁移问题

1. 重复价值是习惯、网络、工作流还是低频任务？
2. 没有通知和奖励时，用户为什么回来？
3. 协作是否真正深化价值，还是只增加活动？
4. 召回是否与连续留存分开报告？

## 证据与局限

- 公司证据主要来自参与者访谈和公开材料，不是完整内部因果评估。
- 公司规模、市场、产品频率、监管与数据能力限制直接迁移。
- 发布方公布的增长与收入数字不用于证明单项方法有效。

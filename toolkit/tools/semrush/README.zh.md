# Semrush

> 覆盖 SEO（搜索引擎优化）、AI 搜索、PPC（按点击付费广告）、社媒、内容与相关市场研究的可见性和竞品研究平台。它适合形成与监控假设，最终结果仍需用第一方产品和收入数据验证。

**[English](README.md)｜[简体中文](README.zh.md)**

| 字段 | 内容 |
| --- | --- |
| 官方网站 | [semrush.com](https://www.semrush.com/) |
| 主要任务 | 关键词与需求研究、竞品分析、技术站点审计、外链研究、可见性追踪、PPC 与社媒研究 |
| 增长阶段 | 获客与衡量 |

## 工作流位置

```text
市场与竞品研究
-> 搜索与主题假设
-> 技术和内容诊断
-> 执行排序
-> 可见性监控
-> 第一方激活、留存与收入验证
```

Semrush 最适合用于执行前和执行过程中：理解可发现需求、比较公开竞品信号、发现技术或内容缺口，以及监控可见性。它不能判断 PMF（产品市场匹配），也不能证明排名、提及或点击造成留存增长。

## 核心任务

### 1. 搜索需求与关键词研究

- 发现查询、主题、意图模式、估算需求与竞争难度。
- 按客户问题、旅程阶段和目标页面组织关键词，而不是把每个词当成独立目标。
- 比较品牌与非品牌需求，判断获客是否依赖已有认知。

### 2. 竞品与市场研究

- 比较公开搜索可见性、主题、付费词、外链与内容模式。
- 用结果发现缺口和假设，而不是生成需要复制的页面清单。
- 区分直接产品竞品，以及竞争同一查询的媒体、Marketplace、社区和聚合网站。

### 3. 技术站点审计

- 发现平台能够识别的抓取、索引、内链、Metadata、性能与其他技术问题。
- 按受影响的重要页面和可能客户影响排序，而不是按警告数量排序。
- 必要时使用搜索引擎工具、服务器日志、浏览器测试与工程调查确认。

### 4. 外链与权威研究

- 研究引用域名、目标页面、Anchor 模式与竞品权威来源。
- 寻找值得联系或投入原创工作的可信媒体、社区、合作方、数据集和资源。
- 不要把链接数量等同于权威，也不要进行操纵性获取。

### 5. AI 搜索与品牌可见性

- 在产品支持的 AI 搜索范围内监控品牌、主题与竞品出现方式。
- 结合 GEO 衡量框架比较提及、引用、Prompt 与来源模式。
- 保留 Prompt 集、日期、市场以及模型或产品环境，因为 AI 回答会持续变化。

### 6. 付费搜索与社媒研究

- 查看估算付费词、素材模式、受众或主题信号与竞品活动。
- 把信号转化成承诺、人群、落地路径与经济性的测试假设。
- 使用广告平台实验、可行时的 Holdout，以及留存 Cohort（同期群）经济性验证。

## 衡量地图

| 工具信号 | 可以回答的问题 | 必须补充的验证 |
| --- | --- | --- |
| 估算搜索量 | 可发现需求是否值得进一步调查？ | Search Console 展现、合格访问、客户研究 |
| 排名或可见性 | 目标主题是否更容易发现？ | 增量流量、激活、留存、辅助转化 |
| 竞品流量估算 | 哪些竞品或主题值得调查？ | 公开证据与第一方市场研究 |
| 站点审计问题 | 哪项技术约束可能阻碍发现或体验？ | 搜索引擎诊断、日志、浏览器与工程检查 |
| 外链信号 | 哪些来源可能带来发现或权威？ | 链接相关性、推荐质量、引用、品牌信任 |
| AI 提及或引用 | 品牌如何出现在支持的 AI 回答中？ | 重复 Prompt 抽样、来源检查、下游行为 |
| 付费词或素材 | 哪类人群与承诺值得测试？ | 增量转化、CAC（客户获取成本）、回本与留存质量 |

## 实际工作流

1. 定义人群、问题、市场、渠道与业务结果。
2. 建立稳定项目、竞品集合、地区、设备与主题分类。
3. 记录可见性、技术问题、内容覆盖、权威与付费活动基线。
4. 把发现转化成包含负责人、机制、指标与护栏的排序假设。
5. 执行内容、技术系统、分发或 Campaign 改动。
6. 监控可见性变化，但不把它当成业务结果。
7. 使用第一方获客、激活、留存、收入与增量数据验证。

## 局限

- 搜索量、流量、竞品、权威与成本数字属于第三方估算，覆盖和方法会随市场与产品变化。
- 被发现的问题、关键词缺口、外链、排名或 AI 提及不能证明业务因果影响。
- 功能、数据库、名称、限制与价格可能变化，需要查看最新官方文档。
- 数据访问不能替代定位、客户价值、编辑判断、工程排序或实验。
- 遵守搜索引擎政策、平台条款、隐私要求与适用法律。

# Sensor Tower

> 覆盖移动应用、使用行为、受众、广告、Web、游戏与相关竞争趋势的数字市场情报平台。它适合用于市场基准和假设形成，最终决策仍需通过第一方商店、产品、广告与财务数据验证。

**[English](README.md)｜[简体中文](README.zh.md)**

## 工具概览

| 字段 | 内容 |
| --- | --- |
| 官方网站 | [sensortower.com](https://sensortower.com/) |
| 主要任务 | App 市场规模、竞品基准、下载与收入估算、使用和留存研究、广告情报、受众与跨平台研究 |
| 增长领域 | 获客、留存、商业化、国际化、付费媒体与市场研究 |
| 常见分析单位 | App、Publisher、品类、国家、平台、受众、广告主或素材 |
| 数据角色 | 第三方估算与 Panel，不能替代第一方分析 |

## 工作流位置

```text
市场与品类研究
-> 竞品和受众基线
-> 获客、留存或商业化假设
-> 产品与 Campaign 决策
-> 第一方验证
```

当内部数据只能解释自己的产品，而团队需要比较品类、竞品、国家、Publisher、广告或受众行为时，Sensor Tower 更有价值。它不能揭示竞品完整内部经济性，也不能证明市场变化的原因。

## 核心任务

### 1. 市场与品类规模

- 比较不同市场的估算下载、收入、活跃使用、参与度与品类变化。
- 判断变化更可能属于单个公司、整个品类、季节、地区还是平台。
- 使用多个时间窗口，避免发布、促销、节假日与报告变化扭曲短期结果。

### 2. 竞品 App 表现

- 比较估算下载、收入、排名、评分、评论、关键词和 Publisher 产品矩阵。
- 比较增长形态、市场结构、发布时间与品类集中度，而不是复制单个功能或 Campaign。
- 把可观察公开事件与推测原因分开。

### 3. 使用、参与和留存研究

- 比较平台能够提供的活跃用户、Session、使用时长、留存用户、流失用户与使用强度估算。
- 用数据选择值得深入调查的竞品、Cohort、国家或行为。
- 自己的留存与价值事件仍需用产品分析验证；外部参与度本身不能解释客户价值。

### 4. 广告与素材情报

- 研究平台覆盖的广告主、Publisher、广告网络、SOV、素材与 Campaign 模式。
- 发现值得测试的持续信息、形式、市场和可能的 Campaign 变化。
- 不能只根据广告出现情况推断投放效率、增量、转化或回本。

### 5. 受众与跨平台研究

- 比较受众参与、App 互动、广告曝光，以及平台支持的 Web-to-App 模式。
- 寻找受众重叠、相邻需求、媒体行为与可能的跨渠道旅程。
- 把 Panel 和模型化重叠当成方向性证据，覆盖会随市场和分群变化。

### 6. 国际化与市场选择

- 比较国家级品类规模、竞品强度、使用、商业化、排名与广告活动。
- 用结果排序本地化研究，而不是假设产品价值和 GTM 能够自动迁移。
- 进入市场前补充语言、支付、监管、设备、分发、文化与本地竞品证据。

## 衡量地图

| 工具信号 | 可以回答的问题 | 必须补充的验证 |
| --- | --- | --- |
| 下载估算 | 获客或品类需求是否变化？ | 商店后台安装、来源质量、激活与增量 |
| 收入估算 | 哪些产品或市场可能具有强商业化？ | 第一方收入、退款、平台费、毛利与付款 Cohort |
| 活跃用户或 Session 估算 | 哪些 App 可能维持使用？ | 产品价值事件、Cohort 留存与频率 |
| 留存或流失用户估算 | 哪些产品的参与质量可能不同？ | 内部 Cohort 定义与自然使用周期 |
| 排名和关键词信号 | App 在商店发现中的可见性如何？ | 商店展现、页面转化与合格安装 |
| 广告 SOV 或素材 | 哪些信息和市场获得持续投入？ | 平台花费、实验、CAC、回本与留存质量 |
| 受众重叠 | 哪些相邻受众可能相关？ | 客户研究、CRM、产品与 Campaign 测试 |

## 实际工作流

1. 定义市场问题、决策、品类、国家、平台、竞品与时间窗口。
2. 记录指标定义，避免比较不兼容的产品、市场或成熟阶段。
3. 建立获客、使用、商业化与广告的品类和竞品基线。
4. 把观察结果与因果假设分开记录。
5. 排序产品、本地化、渠道、素材或市场实验。
6. 使用 App Store 后台、产品分析、广告平台、CRM、收入和定性研究验证。
7. 方法、覆盖、品类定义或平台政策变化时重新检查外部估算。

## 局限

- 下载、收入、活跃、留存、广告、受众与竞品数字可能是估算或模型结果，不是经审计的公司数据。
- 不同产品对活跃用户、收入、流失、市场与品类的定义可能不同。
- 广告、排名、下载、使用与收入之间的相关关系不能证明因果。
- 官方页面描述的数据来源包括消费者 Panel 与 App Store API；需要检查目标产品和市场的具体覆盖与方法。
- 功能、产品名称、数据库权限、价格与条款可能变化。
- 遵守平台条款、隐私要求、竞争法与适用监管。
